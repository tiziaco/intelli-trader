<objective>
Close the never-raise contract hole in `_add_strategy_verb` (`itrader/strategy_handler/lifecycle/manager.py`)
with a **zone-based guard**, per the LOCKED Option B design.

Purpose: `STRATEGY_COMMAND` is externally admitted (D-10, `LiveTradingSystem.add_event`). Today a routine
bad operator payload raises out of a boundary whose docstring promises "Every rejection below is a LOUD
no-op ... NEVER a raise into the queue". In live mode that escape reaches `ErrorPolicy.on_handler_error`
-> `record_failure` -> the failure-rate tripwire -> `halt()`, and `HALTED` has no legal exit except
operator `reset_halt()`. Routine bad input can latch live trading into HALT.

Output: a two-tier guard on zone 1 only, three regression tests, and the backlog todo marked resolved.

**This is an explicitly NON-golden-master, behaviour-changing fix** (a currently-raising path becomes a
logged no-op). That is intended and approved. Backtest is unaffected — STRATEGY_COMMAND routes to an
empty list there — and the oracle is re-confirmed as a gate.
</objective>

<design_locked>
**Do not re-litigate or propose alternatives. This design is settled with the owner.**

The verb has two zones:

- **Zone 1** = untrusted payload -> live object. Exactly one call: `build_strategy(rec, catalog=...)`
  at `manager.py:392`. MUST never raise.
- **Zone 2** = register / persist / emit — `self._managed.add_strategy`, `_persist_strategy`,
  `registry_store.add_portfolio_subscription`, `global_queue.put`. MUST stay loud (D-19 fail-loud
  depends on it). **DO NOT TOUCH ZONE 2.**

Guard **only zone 1**, two-tier:

1. Known validation kinds — `UnknownStrategyTypeError`, `StrategyConfigError`, `UnknownParamError`,
   `MissingParamError`, `ValueError` -> `self.logger.warning(...)`, loud no-op, `return`.
   **PRESERVE the existing message form exactly**: it names the error KIND via `type(exc).__name__`
   and NEVER payload values (the P8 declared-fields-only precedent).
2. Any other `Exception` -> `self.logger.error(..., exc_info=True)`, STILL a loud no-op, `return`.

**Why a bigger tuple is not the fix (verified, do not "simplify" back):** `build_strategy` ->
`cls(**params)` runs `_apply_params` -> `validate()` -> `_run_init()`, and `_run_init` calls
`self.init()` (`base.py:212, 382`) which is ARBITRARY USER-AUTHORED strategy code from
`my_strategies/`. The set of exceptions escaping that call is unbounded by construction. No finite
tuple can be complete. `TypeError` needs no separate decision — the zone guard subsumes it.

**The "never a bare except" doctrine is NOT violated.** The existing comment ("a store/driver fault
must not be silently eaten") governs **zone 2**. Zone 1 contains no store call. The executor MUST
write that reasoning into the code comment so a future reader does not "fix" it back.
</design_locked>
