# Quick Task 260608-a59: Demote By-Design Signal-Rejection Logs Summary

Demoted two by-design signal-rejection log calls in `order_manager.py` from `error` to `warning` (severity flip only), so clean backtest runs no longer pollute logs / trip alerting on expected business-rule rejections; all genuine engine-fault logs remain at `error`.

## What Was Done

### Task 1: Demote the two by-design rejection logs to warning

Changed exactly two `self.logger.error('%s for %s %s', ...)` calls to `self.logger.warning('%s for %s %s', ...)`:

1. **`_reject_unsized_signal` helper** (was line 1067): the first log call after `error_msg = f"{error_prefix}: {reason}"`. Every caller of this helper is an audited business rejection (direction gate LONG_ONLY/SHORT_ONLY, increase gate, max_positions gate, invalid-price guard, SizingPolicyViolation), so this single flip covers all of them — no new method parameter needed.
2. **`except InsufficientFundsError` block** (was line 404, `triggered_by="cash_reservation"`): the log call after `error_msg = f"Cash reservation failed: {e}"`.

Only the method name `error` -> `warning` changed. Format strings, arguments, and surrounding code untouched. TAB indentation preserved (order_manager.py uses tabs per CLAUDE.md).
