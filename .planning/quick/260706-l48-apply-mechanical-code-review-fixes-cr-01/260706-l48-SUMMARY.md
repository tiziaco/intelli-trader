# Phase quick-260706-l48 Plan 01: Apply Mechanical Code-Review Fixes (CR-01, WR-03) Summary

Two drop-in phase-6 code-review fixes: OkxDataProvider.unsubscribe() now clears the per-symbol reconnect-supervisor state (down-flag + attempt count), and universe_poll_cadence_s is validated `gt=0.0` at config construction.

## What Was Done

### Task 1 — CR-01 + WR-03 (commit e08424d2)

**CR-01 (blocker)** — `itrader/price_handler/providers/okx_provider.py`, `unsubscribe()`:
After the existing `task.cancel()` block, added `self._streams_down.discard(symbol)` and `self._reconnect_attempts.pop(symbol, None)`. This prevents a stale down-flag from permanently pinning `is_streaming_healthy()` to False (which wedges the live NEW-submission resume gate `_all_venue_streams_healthy`) and a stale attempt count from tripping the D-20 retry ceiling on a later re-subscribe. Verified `_streams_down` is a set (`.discard`) and `_reconnect_attempts` is a dict (`.pop`). Docstring preserved unchanged.

**WR-03** — `itrader/config/system.py` line 71:
Changed `universe_poll_cadence_s: float = 60.0` to `universe_poll_cadence_s: float = Field(default=60.0, gt=0.0)` so a 0/negative value fails loud at config construction rather than letting the live poll timer busy-spin the queue. `Field` was already imported.
