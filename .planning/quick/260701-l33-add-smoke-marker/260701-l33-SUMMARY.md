# Quick 260701-l33: Add smoke pytest marker Summary

Registered a `smoke` PURPOSE-axis pytest marker (orthogonal to the folder-derived
TYPE axis), tagged the three existing run-path smoke files, added a `make test-smoke`
target, and documented a durable hand-tagging rule in `tests/README.md` and
`tests/conftest.py`.

## What was built

- **pyproject.toml**: one new entry in the `[tool.pytest.ini_options] markers` list
  after `e2e`, documenting `smoke` as a hand-applied PURPOSE axis. This is the single
  `--strict-markers` registration home.
- **tests/integration/test_okx_smoke.py**: converted the existing single `pytestmark`
  (skipif) into a list `[skipif(...), pytest.mark.smoke]` — skipif reason preserved
  verbatim, so the credential-gated auto-skip still holds.
- **tests/integration/test_backtest_smoke.py** and
  **tests/unit/reporting/test_plots_smoke.py**: added `import pytest` +
  module-level `pytestmark = pytest.mark.smoke`.
- **Makefile**: `test-smoke` added to `.PHONY` (line 6) and a target after `test-e2e`
  running `poetry run pytest tests/ -v -m "smoke"` (recipe lines are HARD TABS, matching
  neighbouring targets).
- **tests/README.md**: a PURPOSE-axis paragraph under `## Markers`, a `make test-smoke`
  line in the `## Running` block, and the durable rule (new smoke tests MUST be tagged
  `@pytest.mark.smoke`).
- **tests/conftest.py**: module-docstring note that `smoke` is manual and intentionally
  absent from `pytest_collection_modifyitems`. Docstring only — the hook is unchanged.

## Verification

- `poetry run pytest tests -m smoke --collect-only -q` → **7 selected / 1502 deselected**
  across exactly the 3 smoke files (test_backtest_smoke×1, test_okx_smoke×1,
  test_plots_smoke×5). No `--strict-markers` error.
- `poetry run pytest tests -m "integration and smoke" --collect-only -q` → **2 selected**
  (the two integration smoke files only; not the plots unit file).
- `poetry run pytest tests/ -v -m "smoke"` → **6 passed, 1 skipped** (okx auto-skips on
  absent creds), 1502 deselected.
- `poetry run pytest tests --collect-only -q` (full suite) → **1509 collected**, no marker
  error (registration valid; regression check).

## Deviations from Plan

None — plan executed exactly as written.

## Environment Caveats

- `make test-smoke` could not be exercised directly in the worktree: the `Makefile`
  begins with `include .env` and aborts on the missing `.env` (`Makefile:2: .env: No
  such file or directory` — a known worktree condition). The target selection was
  verified via the exact equivalent command it wraps, `poetry run pytest tests/ -v -m
  "smoke"`, which produced the expected 6-passed/1-skipped result. Run `make test-smoke`
  in the main checkout (where `.env` is present) to confirm the Make wrapper itself.
- All `poetry run pytest` invocations were run with `PYTHONPATH="$PWD"` prepended to
  defeat editable-install shadowing of worktree edits (known worktree gotcha).
