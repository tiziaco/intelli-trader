# Quick 260623-f80: Fix perf coverage instrument A over-sell (bracket-only recycle) Summary

Removed coverage instrument A's discretionary `FractionOfCash`-sized exit so it recycles via its OCO bracket alone — A-only full-window fills now spread Feb-Jun (213 post-Jan) instead of freezing after January, closed positions jumped 0/61 baseline -> 125, and final equity is a sane $76k on a $100k start (no ~$10M phantom).

## What Changed

`perf/strategies/a_bracketed_momentum.py` (4-space indented, Decimal end-to-end):

- **Deleted** the two-line discretionary exit branch in `generate_signal`
  (the downward `macd_hist` zero-cross that returned a cash-sized `self.sell(ticker)`).
  After the bracket-buy on the upward crossover, the next statement is now `return None`.
- **Dropped** the now-unused `crossunder` import from the `primitives` import line
  (kept `crossover` and `is_above`).
- **Updated** the module docstring and the `generate_signal` comments (decision-anchored
  style) to explain WHY the discretionary exit was removed and to note that a separate
  engine-level over-sell anomaly is being investigated under `/gsd:debug` — this change
  does not attempt to fix the engine.

Entry logic, `_SL_PCT`/`_TP_PCT` (0.02 / 0.03), `sizing_policy`, `max_positions` (default 1),
`direction`, indicator windows, and the engine/sizing layers were all left untouched.

## Root Cause (as fixed)

A is `LONG_ONLY` with `sizing_policy = FractionOfCash(Decimal("0.95"))`. The discretionary
`self.sell(ticker)` exit was sized off CASH (the instrument's sizing policy), NOT off the held
position quantity. So each exit sold more than the position held, building a net-short inventory
that the spot `LONG_ONLY` engine mislabeled `LONG` with a positive `market_value` (phantom ~$10M
equity on a $100k start). The net inventory never returned to 0 -> the long never closed -> with
`max_positions=1` every later entry was blocked -> fills froze after January. This is the direct
analog of the 260623-bmg recycle fix applied to B/C/D.
