<objective>
Fix the perf coverage instrument A (`perf/strategies/a_bracketed_momentum.py`) over-selling bug
by removing its cash-sized discretionary exit branch, so it recycles via the OCO bracket alone.

Strategy A is a LONG_ONLY coverage instrument. It bracket-buys on (uptrend filter AND macd_hist
crossover), declaring an OCO bracket (sl 2% / tp 3%). It ALSO has a discretionary exit on
macd_hist crossunder that calls `self.sell(ticker)`. That sell is sized off
`FractionOfCash(Decimal("0.95"))` (CASH), NOT off the held position quantity - so A over-sells,
building a net-short inventory mislabeled LONG with positive market_value (phantom ~$10M equity on
a $100k start). The net inventory never returns to 0, the long never closes, and with
max_positions=1 every later entry is blocked - fills FREEZE after January despite ~400 entry +
~400 exit signals/month through June.

This is the direct analog of the prior 260623-bmg recycle fix applied to B/C/D.

Purpose: Make A recycle correctly so the W1 benchmark exercises its bracket/OCO path across the
full window instead of stalling in January.
Output: Edited `a_bracketed_momentum.py` (discretionary exit removed, comments updated) +
verified-sane A-only full-window backtest.
</objective>
