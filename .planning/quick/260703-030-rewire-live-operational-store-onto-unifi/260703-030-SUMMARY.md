# Quick Task 260703-030: Rewire Live Operational Store onto Unified SqlSettings Summary

Deleted the legacy `SYSTEM_DB_URL` D-live stopgap and rewired the live operational store gate onto the unified `SqlSettings` `ITRADER_DATABASE_*` credential surface — the same source Alembic migrations already use — so migrations and the live composition root read one canonical credential source.

## What Changed

### Task 1 — Rewire the live store gate and delete SYSTEM_DB_URL (commit 8947cc27)
- **`itrader/trading_system/live_trading_system.py`** (4-space, matched):
  - Deleted the module-level `_SYSTEM_DB_URL = os.getenv("SYSTEM_DB_URL", "")` read and its comment block; rewrote the comment to describe the new env-presence mechanism on `ITRADER_DATABASE_PASSWORD`/`ITRADER_DATABASE_URL`. `os` stays imported (still used by the new in-method `os.getenv` reads).
  - Replaced the store-selection gate: reads `pg_password = os.getenv("ITRADER_DATABASE_PASSWORD", "")` and `pg_url = os.getenv("ITRADER_DATABASE_URL", "")` inside `__init__`; gates on `if not (pg_password or pg_url):` → in-memory fallback with an updated WR-10 warning naming the new vars. The env-presence check gates BEFORE constructing `SqlSettings` (which raises `ValidationError` via `_require_pg_credentials` when no credential is present — no construct-and-catch).
  - Postgres arm: removed `from pydantic import SecretStr` and the `url=SecretStr(_SYSTEM_DB_URL)` argument; now builds `SqlBackend(SqlSettings(driver=SqlDriver.POSTGRESQL_PSYCOPG2))` with no explicit `url=`, mirroring `migrations/env.py:76`. SQL imports stay lazy inside the branch (inertness gate preserved). `CachedSql` composition and `_system_db_backend` disposal wiring unchanged.
  - Updated the two remaining `SYSTEM_DB_URL` comment references (signal-store sink comment; `start()` reconcile-guard comment).
- **`itrader/price_handler/store/sql_store.py`** (4-space, matched): rewrote the "Single canonical credential source" docstring paragraph — the live runtime now reads the same unified `SqlSettings` `ITRADER_DATABASE_*` surface; the string `SYSTEM_DB_URL` no longer appears.

### Task 2 — Env-gated store-selection tests + component-var proof (commit 71db5acb)
- **`tests/integration/test_store_live_drive.py`** (4-space, matched):
  - Converted `test_unset_system_db_url_falls_back_to_in_memory` → `test_unconfigured_db_env_falls_back_to_in_memory`: `monkeypatch.delenv` both `ITRADER_DATABASE_PASSWORD`/`ITRADER_DATABASE_URL`, kept in-memory type assertions, added a `caplog.at_level(WARNING)` assertion that the WR-10 warning fires (substring `ITRADER_DATABASE_PASSWORD`).
  - Converted `test_live_system_wires_cached_sql_when_system_db_url_set` → `test_live_system_wires_cached_sql_from_database_url`: `monkeypatch.setenv("ITRADER_DATABASE_URL", pg_url)` (verbatim escape hatch), kept CachedSql wrapper assertions.
  - Added `test_live_system_wires_cached_sql_from_component_vars`: decomposes `pg_url` via `sqlalchemy.make_url`, sets `ITRADER_DATABASE_HOST/PORT/USER/PASSWORD/NAME`, deletes `ITRADER_DATABASE_URL`, asserts both CachedSql wrappers wire AND `system._system_db_backend.engine.url.database == u.database` (proves component assembly, no verbatim-URL read).
  - Added `import logging`; updated module docstring off `SYSTEM_DB_URL`.
