OLD check_timeframe (midnight-relative / date-anchored):
  - converts to UTC and zeroes sub-minute: time.astimezone(pytz.utc).replace(second=0, microsecond=0)
  - seconds = (time - time.replace(hour=0, minute=0, second=0, microsecond=0)).total_seconds()
  - fires when: seconds % timeframe.total_seconds() == 0

Why this is correct: anchor is midnight-OF-THE-DAY (in UTC), so any timeframe fires on every
midnight regardless of unit (weekly -> any midnight, 7h -> midnight). For the daily golden bars
at 00:00 UTC, seconds-since-midnight == 0, so `0 % 86400 == 0` -> True, identical to the epoch
grid for daily. Daily firing is therefore unchanged.

Current (WRONG) seam in time_parser.py:127-153:
