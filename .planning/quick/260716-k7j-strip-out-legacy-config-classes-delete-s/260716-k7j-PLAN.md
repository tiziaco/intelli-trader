<objective>
Strip the two legacy, production-dead config classes — `Settings` (config/settings.py) and
`SystemConfig` (config/system.py) — and re-home the still-needed fields onto the live frozen
root `ITraderConfig`.

Facts established by the orchestrator (do NOT re-derive):
- The process singleton is already `config = ITraderConfig()` (itrader/__init__.py:11). NO
  composition root constructs `SystemConfig`; only tests call `SystemConfig.default()`. It is
  dead in production.
- `Settings`'s ONLY live consumer is `config/__init__.py:76` deriving the module-level
  `TIMEZONE` from the CLASS-DEFAULT `Settings.model_fields["timezone"].default` ("Europe/Paris").
  No `config.runtime.*` instance read exists anywhere in itrader/.
- `TIMEZONE` is oracle-critical: it MUST stay exactly "Europe/Paris" or the byte-exact oracle
  (tests/integration/test_backtest_oracle.py — 134 trades, frozen-golden CSV compare, no
  tolerance) breaks.
- The logger reads `ITRADER_LOG_LEVEL`/`ITRADER_DISABLE_LOGS` straight from `os.environ` and MUST
  NOT construct any config/settings instance at import (Pitfall 8). Its behavior is untouched;
  only its stale docstrings are refreshed.
- `ITraderConfig` already carries `name`/`version`/`environment`/`debug_mode`/`data_dir`/… , the
  mutable sub-models, the `order` field, and the lazy `sql` cached_property — keep all of them.

Design note on the log_level/disable_logs re-home (user decision 4 + 5, jointly honored):
`ITraderConfig` is a plain frozen `BaseModel`, not `BaseSettings`, so a plain field default cannot
env-parse. The mechanic is a slim `RuntimeSettings(BaseSettings, env_prefix="ITRADER_")` carrying
ONLY `log_level`/`disable_logs`, mounted as a NEW field. Because decision 5 says drop the field
named `runtime`, the new field is named **`logging`** (type `RuntimeSettings`) — NOT `runtime`.
Net: the legacy `runtime: Settings` field is gone (decision 5) and env-parsing is preserved via
`config.logging` (decision 4). `Settings.environment` is redundant (ITraderConfig already has an
`Environment` enum base field) and is simply dropped.

Purpose: shrink the config surface to one live root, eliminate two dead classes, keep the backtest
oracle + import-inertness byte-for-byte intact.
Output: `RuntimeSettings` in a new `config/runtime.py`; `timezone` on the frozen base; `logging`
sub-model field; `settings.py`/`SystemConfig` deleted; every consumer/barrel/test/docstring
repointed with zero dangling references.
</objective>
