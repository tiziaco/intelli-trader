<objective>
Make transaction processing atomic via validate-first ordering (D-09/D-12): all checks before
any mutation, so no rollback machinery is needed. Establish the one error contract — raise
typed, return None (D-10). Delete the never-worked saga (`TransactionContext`,
`TransactionState`, pending-dict, `_handle_transaction_error`, the unreachable `return False`),
the `Portfolio.cash` setter, and `apply_transaction_delta` (D-05/D-11). Route settlement cash
through a new full-precision CashManager fill flow with one ledger entry per fill carrying a
`fee` field (D-06), making the ledger LIVE deterministically (Pitfall 5).

Purpose: M4-02 (findings #16, #23). The settlement funds check survives as an INVARIANT GUARD
against `balance` — never `available_balance` (D-10, Pitfall 2) — making FILL dispatch order
irrelevant to the reservation lifecycle that plan 05-06 wires.

Value-preserving: the new cash math must equal today's `apply_transaction_delta` delta exactly
(byte-exact oracle at every commit).

Output: reordered Portfolio settlement, shrunk TransactionManager, deterministic live ledger.
</objective>
