<objective>
Wire the full reservation lifecycle onto the trade path (D-01..D-04, Critical #22 / M4-01):
synchronous check-and-reserve in `OrderManager`'s admission path via the PortfolioReadModel
Protocol (BUYs only; reserve = price × quantity + estimated commission); reserve failure →
audited PENDING→REJECTED via the Phase 4 state-change path, nothing emitted; idempotent
release on every terminal reconciliation in `OrderManager.on_fill`. Settlement debit/credit
already flows via FillEvent (plan 05-05) — the queue stays the source of truth for facts;
the sync call is only the pre-trade gate (D-02, NOT queue-mediated — TOCTOU).

Purpose: every trade now routes cash through CashManager — reservation gate at admission,
full-precision debit/credit at settlement, live ledger/audit. Completes M4-01.

Value-preserving (Pitfall 3): the golden run is long-only, fees 0, sizing 0.95 × cash —
reserve never rejects and reservations are fully released before the next admission. D-14
mandates a trace PROVING this; this plan ships it as an integration assertion.

Output: admission gate + terminal release wired; golden-run inertness test.
</objective>
