Where CrossvalResult is a small dataclass this module defines (the orchestrator will read
these attributes — keep names stable):

    @dataclass
    class CrossvalResult:
        engine: str                       # "nautilus"
        reconciled: bool                  # True only if the engine ran end-to-end
        reason: str | None                # populated when reconciled is False: human-readable degrade cause
        trade_log: "pandas.DataFrame | None"   # columns: entry_date, exit_date, side, realised_pnl
        equity_curve: "pandas.Series | None"   # equity by bar timestamp

UNIFORM ORCHESTRATOR CONTRACT (consumed by 08-07 exactly like the gating engines): this
module ALSO exposes a thin `run` wrapper so 08-07 can call every engine identically:

    def run(
        prices: "pandas.DataFrame | None" = None,      # golden BTCUSD frame (orchestrator-injected) or None
        indicators: "pandas.DataFrame | None" = None,  # sma_short/sma_long/macd_hist columns, or None
    ) -> "tuple[pandas.DataFrame, pandas.Series]":
        """Call run_nautilus(...) (mapping prices/indicators into its params). If the result
        is reconciled, return (result.trade_log, result.equity_curve). If degraded, RAISE
        RuntimeError(result.reason) — 08-07 wraps every engine run() in try/except and records
        the message as the 'Nautilus: not reconciled — {reason}' status (D-12). run_nautilus
        itself NEVER raises (degrade-safe for standalone use); only the run() wrapper raises so
        the orchestrator's uniform try-guard sees the same shape across all three engines."""

The normalized trade_log columns are `entry_date, exit_date, side, realised_pnl` (matching
iTrader's tests/golden/trades.csv shape + 08-07's reconcile helpers).

D-01 force-match config (mirror scripts/run_backtest.py exactly):
  - starting cash $10,000; fees 0; slippage 0
  - 95%-of-equity sizing, long-only, allow_increase=False (single position from flat)
  - next-bar-open fills (signal on completed bar T fills at bar T+1 open)
  - SMA short=50, long=100; MACD FAST=6, SLOW=12, WIN=3 (ta.trend.SMAIndicator / ta.trend.MACD.macd_diff)

The SMA_MACD quirk (SMA_MACD_strategy.py lines 88-94) — replicate verbatim:
  - ENTRY:  short_sma[-1] >= long_sma[-1]  AND  macd_hist[-1] >= 0  AND  macd_hist[-2] < 0  -> BUY
  - EXIT:   short_sma[-1] >= long_sma[-1]  AND  macd_hist[-1] <= 0  AND  macd_hist[-2] > 0  -> CLOSE
  - The SMA filter gates BOTH entry and exit — a held long is NOT exited on a MACD
    down-cross when the filter is False (the exit elif is nested inside the filter if).
  - Warm-up: no signal until len(bars) >= max(long_window, 100).
