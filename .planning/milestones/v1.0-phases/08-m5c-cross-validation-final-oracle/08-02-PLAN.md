<objective>
After plan 08-01 retypes `Portfolio.total_market_value` / `total_equity` /
`total_unrealised_pnl` / `total_realised_pnl` / `total_pnl` from `float` to
`Decimal` (and cleans the same-file `MetricsManager` coercions and
`EnhancedOrderValidator` cash checks), every site that consumed those members as
`float` can break: `mypy --strict` flags `float + Decimal` mixed arithmetic and
float-expecting passes, and `filterwarnings=["error"]` can surface a dtype/cast
warning as a hard test failure. This plan is the **caller fan-out + propagation
sweep** (RESEARCH risk #4 "Decimal retype fan-out", D-06 / D-13).

It fixes the cross-file consumers that 08-01 does NOT own — the end-of-run
summary in `backtest_trading_system.py` and the oracle generator
`scripts/run_backtest.py` — and decides where the Decimal→float serialization
boundary sits (keep money Decimal end-to-end, convert at the serialization edge
with the existing `float()` / `FLOAT_FORMAT="%.10f"` pattern). The gate is the
program definition-of-done's structural half: `make typecheck` clean and the
full live suite green under strictness.

This plan is the **sweep ONLY**. It does NOT regenerate or re-freeze the oracle —
that is plan 08-03 (`make backtest` diff vs `tests/golden/*` + conditional
`REFREEZE-M5C-DECIMAL`). Here, `make backtest` is invoked only to prove the run
still executes end-to-end and serializes artifacts without error; no diffing or
re-freeze happens.

Purpose: close the type-propagation fan-out so the golden path is Decimal-clean
end-to-end and the toolchain (mypy --strict, pytest strictness) is green, which
is a hard precondition for the 08-03 re-freeze and the D-13 DoD gate.
Output: a green `make typecheck` + `make test`, and a `make backtest` that runs
and serializes without error, against the 08-01 Decimal numbers.
</objective>
