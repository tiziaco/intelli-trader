# Phase 4 Plan 03: Fill De-Mutation & Replace-In-Book Matching Summary

The execution layer performs zero event mutation: FillEvent.new_fill is construct-complete with keyword-only executed price/quantity/commission plus a UUIDv7 fill_id and strategy_id linkage (D-12), and MatchingEngine.modify replaces the resting OrderEvent via dataclasses.replace instead of mutating it — REFUSED/CANCELLED values byte-identical, suite + both oracle layers byte-exact at every commit.

## What Was Built

- **FillEvent (event.py, TABS):** new fields `fill_id: uuid.UUID = field(default_factory=uuid_compat.uuid7)` and `strategy_id: Optional[StrategyId] = None`. Factory reshaped to `new_fill(cls, status, order, *, price: float, quantity: float, commission: float)` — executed values are keyword-only constructor inputs; inside, `fill_id=uuid_compat.uuid7()` (exchange generates at fill construction, D-12), `strategy_id=order.strategy_id`, `order_id=order.order_id` (explicit, no getattr). `FillStatus(status)` enum parse kept. `from uuid import uuid4` (dead) replaced by `import uuid` + `import uuid_utils.compat as uuid_compat`.
- **SimulatedExchange (simulated.py, TABS):** `_emit_fill` passes `price=executed_price, quantity=fill_quantity, commission=float(commission)` into the factory; the two post-construction mutations (`fill_event.price = ...` / `fill_event.quantity = ...`) are DELETED. The `float(commission)` Decimal→float boundary coercion and its comment kept exactly (D-04). `_emit_rejection` and both CANCELLED sites pass the order's OWN `price=event.price, quantity=event.quantity, commission=0.0` — byte-identical values, zero numeric drift.
- **MatchingEngine (matching_engine.py, SPACES):** `modify` is replace-in-book — `dataclasses.replace(order, price=..., quantity=...)` with None-guarded conditional kwargs, stored back under the same key; docstring records that `replace` deliberately PRESERVES `order_id` (and `event_id` once events carry one): a MODIFY amends terms, not identity (RESEARCH Open Question 2 resolution). Stale int annotations fixed: `_resting: dict[OrderId, OrderEvent]`; `cancel`/`modify`/`has_order`/`get_order` take `OrderId`; internal `candidates`/`chosen` maps and `_pick_bracket_winner` typed `OrderId`.
- **Event annotations (event.py):** `OrderEvent.order_id`/`parent_order_id` and `FillEvent.order_id` retyped `Optional[OrderId]` — ids have been UUIDv7-backed since M2; required so the OrderId-keyed book typechecks against its callers under mypy strict.
- **Tests:** `test_fill_event_schema.py` extended 2→6 cases: `fill_id.version == 7`, fill_id uniqueness, strategy_id propagation, executed values land without mutating fill or originating order. `test_matching_engine.py` +3 cases: replace-in-book stores a new object and never mutates the submitted event, order_id/parent_order_id identity preservation, None-guarded partial modify leaves the other field unchanged (unknown-id False already covered).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Old new_fill signature used by four test files outside the plan's file list**
- **Found during:** Task 1
- **Issue:** `tests/unit/order/test_order_manager.py`, `tests/unit/portfolio/transaction/test_transaction_init.py`, `tests/unit/events/test_event_immutability.py`, `tests/unit/events/test_events.py` call `FillEvent.new_fill(status, commission, order)` — the reshaped keyword-only signature breaks them
- **Fix:** call sites updated to `new_fill(status, order, price=..., quantity=..., commission=...)` passing the order's own values (identical results); stale "rewritten by the simulated exchange / MatchingEngine.modify" docstrings in `test_event_immutability.py` corrected to "structurally mutable until the 04-04/04-05 freeze"
- **Commit:** bd723c4

**2. [Rule 3 - Blocking] mypy strict rejected `dataclasses.replace(order, **changes)` with a `dict[str, float]`**
- **Found during:** Task 2 typecheck gate
- **Issue:** mypy checks `**kwargs` against each typed field of the generated `replace` signature — 8 arg-type errors
- **Fix:** single `replace` call with None-guarded conditional-expression kwargs (`price=order.price if new_price is None else new_price`), same semantics
- **Commit:** 3359b1f

**3. [Rule 3 - Blocking] OrderId-keyed book fails mypy against int-annotated event ids**
- **Found during:** Task 2 (anticipated during read phase, confirmed by gate)
- **Issue:** `submit`/`cancel`/`modify` callers feed `OrderEvent.order_id` (annotated `Optional[int]`, stale since M2) into `dict[OrderId, ...]`
- **Fix:** `OrderEvent.order_id`/`parent_order_id` and `FillEvent.order_id` retyped `Optional[OrderId]` — annotation-only, zero runtime change; ripple contained (these fields' only typed consumers are the execution layer; order_manager reads via getattr/Any)
- **Commit:** 3359b1f
