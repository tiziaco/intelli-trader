## Pattern Assignments

### `admission_manager.py` — the short-increase gate (the ONE source edit)

**Analog:** the LONG INCREASE gate in the same file (`:577-591`). Mirror it byte-symmetrically for the SHORT case.

**Mirror target — long gate (`:577-591`)** — copy this shape exactly, swapping `LONG`→`SHORT`:
```python
if open_position is not None and open_position.side is PositionSide.LONG:
    # INCREASE case (D-10): BUY for an already-open long.
    if not signal_event.allow_increase:
        return self._reject_unsized_signal(
            signal_event,
            f"position increase not allowed by strategy "
            f"(allow_increase=False) for {signal_event.ticker}",
            triggered_by=OrderTriggerSource.ADMISSION_INCREASE,
            operation_type=OrderOperationType.SIGNAL_ADMISSION,
            error_prefix="Signal rejected at admission",
        )
    # allow_increase=True: fall through to entry sizing — the
    # FractionOfCash arm reads CURRENT available_cash (remaining-cash
    # semantics) and the check-and-reserve gate covers the cash check.
    return None
```

**Code to LIFT — current unconditional short rejection (`:537-556`)**, inside the `if signal_event.action is not Side.BUY:` block:
```python
if (snap is not None and snap.side is PositionSide.SHORT):
    return self._reject_unsized_signal(
        signal_event,
        f"position increase not allowed: short increase is out of "
        f"v1 scope (D-09) for {signal_event.ticker}",
        triggered_by=OrderTriggerSource.ADMISSION_INCREASE,
        operation_type=OrderOperationType.SIGNAL_ADMISSION,
        error_prefix="Signal rejected at admission",
    )
return None
```
The unconditional `snap.side is PositionSide.SHORT` rejection becomes conditional on `not signal_event.allow_increase` (D-01). When `allow_increase=True` it must `return None` (fall through to sizing) instead of rejecting.

> **Planner discretion (CONTEXT :107-110):** either (a) lift inline at `:547-556` by adding the `allow_increase` check, or (b) fold the SHORT case into the side-dispatch block alongside the long gate (`:577-592`) so the two read symmetrically. The SELL-vs-open-SHORT branch currently lives EARLY (`:537-556`, before the `portfolio_handler is None` guard at `:557` and the snapshot dispatch at `:577-594`) because it keys off `snap` directly. **The LONG gate at `:577-591` MUST stay byte-exact either way** — do not touch its lines.

**Where the admitted add lands (`:800-806`)** — no edit, just the fall-through destination. A SELL-add that passes the lifted gate reaches:
```python
return self.sizing_resolver.resolve_entry(

```
This is the SAME arm the long add uses. Direction is not passed; it never was (D-02).

**Reservation-side check (D-06, criterion 4b):** the planner MUST verify in planning (not change here) that the check-and-reserve gate reached after `resolve_entry` reserves `notional / leverage` on the correct (reserve) side for a SELL-add — this is a correctness check on the existing path, not a new edit.

### `sizing_resolver.py` — `resolve_entry` (REUSED UNCHANGED — do NOT add a short arm)

**Analog:** itself. Direction-agnostic policy dispatch, `assert_never`-closed (D-02). Adding a short arm would break the one-resolver invariant.

**The two cash-vs-margin bases (D-03/D-04)** the short add can reach unchanged:
```python
case FractionOfCash():
    available = self._read_model.available_cash(portfolio_id)      # :115
    qty = (policy.fraction * available) / to_money(price)          # :116 (compounding, D-04)
...
case LeveredFraction():
    equity = self._read_model.total_equity(portfolio_id)           # :136 (margin-aware, D-03)
    qty = (policy.fraction * equity) / to_money(price)             # :137 (NEVER touches cash)
case _:
    assert_never(policy)                                           # :138-139 (one-resolver invariant)
```
`FractionOfCash` = spot/collateralized short (compounding fraction-of-remaining-cash); `LeveredFraction` = levered short (notional off equity). The strategy already declares the basis via policy. **The phase introduces NO new arm.**
