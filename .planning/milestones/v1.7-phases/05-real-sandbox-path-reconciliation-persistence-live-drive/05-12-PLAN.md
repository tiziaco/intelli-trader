|-----------|----------|-----------|-------------|-----------------|
| T-05-12-01 | Tampering / Elevation | live order submission routing (sandbox flag) | mitigate | Every live test asserts `system._okx_connector.sandbox is True` BEFORE any order submission (`_assert_sandbox_routed`); the demo host (`wspap.okx.com` / REST x-simulated-trading) is what makes the demo keys real-money-free (T-05-04). | T-05-12-03 | Information Disclosure | credential handling in the suite | accept | No new secret handling — credentials cross only via the existing `OkxSettings`/`SecretStr` layer (`.get_secret_value()` at the ccxt edge); the suite reads `os.environ.get` presence only for the skipif and never logs the triple. |
| T-05-12-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed — test authoring over already-vendored ccxt.pro / pytest / the existing OKX stack. |
