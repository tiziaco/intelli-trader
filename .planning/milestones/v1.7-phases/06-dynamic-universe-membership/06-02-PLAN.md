<objective>
Grow `OkxDataProvider` from a single wiring-time symbol to dynamic, per-symbol candle
subscribe/unsubscribe (D-05): a `{symbol: asyncio.Task}` subscription registry, an idempotent
`subscribe(symbol)` that spawns a supervised `candle{tf}` coroutine on the connector loop, an
`unsubscribe(symbol)` that cancels via the connector's existing cooperative-cancel teardown, and
**per-symbol supervisor keys** (replacing the single `"candles"` literal so one symbol's drop no
longer marks all streams down / one symbol's payload no longer resets all budgets — Pitfall 2).

This is the Arm B data-plane mechanism. It carries NO membership knowledge — the registry is
mechanical socket state derived from `UniverseUpdateEvent`s (wired in plan 05); the provider
never decides membership. The `confirm='0'` snapshot-on-subscribe is already dropped by
`_process_row`; this plan adds NO new dedup logic and documents the warmup-before-subscribe
ordering contract (Pitfall 6) that plan 05's consumer honors.

Purpose: give plan 05 a per-symbol subscribe/unsubscribe surface to drive from membership.
Output: the registry + subscribe/unsubscribe + per-symbol keys, async unit tests (mocked connector).
</objective>
