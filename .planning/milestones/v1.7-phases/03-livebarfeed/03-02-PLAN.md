<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| provider `ClosedBar` push → `LiveBarFeed.update()` | Every incoming closed bar is untrusted venue data crossing into engine state (the ring + indicator-driving BarEvent stream). The feed's monotonic guard is the integrity checkpoint. |
| asyncio (producer) thread → engine thread | `update()` runs on the connector asyncio thread; only `global_queue.put` crosses to the engine thread (D-02/D-19). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-REPLAY | Tampering | `update()` guard | mitigate | Monotonic-forward-only guard: stale (`t<L`) rejected, duplicate (`t==L` same) dropped, revision (`t==L` differ) forward-only WARN+drop with NO state mutation — an out-of-order/replayed bar cannot rewind indicator state (D-06/D-07, V5 input validation). |
| T-03-02-LOOKAHEAD | Info-disclosure (look-ahead) | `Bar.time` / `window()` | mitigate | Bar `time` is the venue bar-open `ts` (tz-aware, never wall-clock); `window()` enforces the rule-4 completed-bars cutoff — the forming bucket is invisible (LX-08, 7-rule contract in the feed only). |
| T-03-02-MEM | Denial of Service | ring buffer | mitigate | Bounded `deque(maxlen=cache_capacity())` — the ring is memory-bounded regardless of stream volume; a flood of closed bars cannot grow it unboundedly. |
| T-03-02-RACE | Tampering | ring/guard writes | mitigate | All `update()` calls stay on the single connector asyncio thread (single-writer ring/guard, RESEARCH Thread hand-off); only the MPSC-safe `queue.Queue.put` crosses threads (D-19 preserved) — no two-writer race, no lock needed. |
| T-03-02-SILENTDROP | Repudiation | rejected/dropped bars | mitigate | Stale/revision/duplicate are logged (WARN/debug), never silently swallowed; `EventHandler._dispatch` raises on any unrouted BarEvent downstream. |

No high-severity web threats apply — the feed handles no credentials, no direct network, no user
input (the socket + auth are the Phase-2 connector's concern).
</threat_model>
