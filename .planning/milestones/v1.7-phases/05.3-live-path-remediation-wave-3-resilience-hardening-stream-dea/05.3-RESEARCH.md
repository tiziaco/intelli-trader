### Pitfall 1: Trusting CONTEXT / 05.1 line numbers blind
**What goes wrong:** editing `_escalate_connector_halt` at `okx.py:578` (now `:594`), `_dispatch_live` at
`:727` (now `:877`), or `add_event` at `:1323` (now `:1600`). The 05.1 research is TWO PRs stale (#77, #78).
**How to avoid:** use the Re-Anchor Map above; re-grep the symbol before every edit.
**Warning signs:** an edit lands on a docstring or an unrelated method.

### Pitfall 2: Indentation hazard (tabs vs spaces)
**What goes wrong:** the D-16/D-22 files (`venue_correlation.py`, `reconcile_manager.py`, `okx.py`,
`portfolio.py`) are **TABS**; the D-19/D-20/D-21 companions (`cached_sql_storage.py`, `halt_record_store.py`,
`portfolio_handler.py`, `live_trading_system.py`, `venue.py`, `simulated.py`) are **4 SPACES**. A mixed
diff breaks a tab file's import.
**How to avoid:** match the file (see the complete table above); never normalize.

### Pitfall 3: Blocking SQL on the asyncio loop thread (D-21 — the WHOLE point)
**What goes wrong:** `record_halt` → `engine.begin()` runs on the connector loop thread via
`_escalate_connector_halt` → `halt()`. That stalls the loop and every stream sharing it.
**How to avoid:** flag-only escalation on the loop; do the blocking write on the engine thread (mirror the
pause/resume flag pattern). This is Pitfall 9 from the ARCH register.

### Pitfall 4: `connector.call()` from the loop thread self-deadlocks (D-17)
**What goes wrong:** `_backfill_gap` on the loop thread → `connector.call(...).result()` → 30s stall → livelock.
**How to avoid:** loop-thread code awaits the client coroutine DIRECTLY; never bridge through `call()`. The
engine-thread `warmup` path is safe and stays unchanged.

### Pitfall 5: Backtest oracle must stay byte-exact (all D)
**What goes wrong:** D-19 (touches `transact_shares`/settle path), D-22 (touches `reconcile_manager`), and
D-23 (touches `portfolio_handler.rehydrate` ungate) all sit on hot money paths. Any equity-curve shift
fails the oracle.
**How to avoid:** all fixes live-path-only / oracle-dark — the backtest uses `in_memory` storage
(no `save_account_state`, no `rehydrate`, no `venue_trade_id`) so the getattr-guards make every change a
clean skip. Re-run `poetry run pytest tests/integration/test_backtest_oracle.py` (`134 / 46189.87730727451`,
`check_exact=True`) per task.

### Pitfall 6: mypy-strict on all but two files
**What goes wrong:** D-19/D-22/D-16/D-13/D-11 edits land on strict files (`cached_sql_storage.py`,
`reconcile_manager.py`, `venue_correlation.py`, `okx.py`, etc.) — an untyped local or `Any` leak fails the gate.
**How to avoid:** `live_trading_system.py` + `trading_interface.py` are the ONLY `ignore_errors` files
(`pyproject.toml:104-105`). Everything else must keep `mypy --strict itrader` clean.

### Pitfall 7: D-19 seam is TWO commits on ONE engine
**What goes wrong:** assuming position + cash persist are far apart or on different engines.
**How to avoid:** both `set_position` (`_store.set_position:100`) and `save_account_state`
(`engine.begin():234`) go through `self._store.engine` — a single engine. Threading one connection through
both is mechanically feasible; the seam is real, not architectural.

### Pitfall 8: `ReconcileManager` has no portfolio transaction store (D-22)
**What goes wrong:** trying to seed `_applied_trade_keys` from `order_storage` — the `venue_trade_id`
history lives on the PORTFOLIO state storage, read via `PortfolioHandler.rehydrate:950`.
**How to avoid:** drive the seed from `PortfolioHandler.rehydrate` into a `ReconcileManager` seed hook (via
`OrderManager`), or give `ReconcileManager` durable-transaction read access. Reuse the `_record_applied_trade:107`
primitive; do not re-read the history twice with divergent keying.

### Pitfall 9: Secret scrub on halt egress (D-11, D-21)
**What goes wrong:** logging `str(exc)` on a connector-fatal escalation leaks request context / creds.
**How to avoid:** preserve the existing scrub — `_escalate_connector_halt:603` logs `type(exc).__name__`,
never `str(exc)`; the halt reason is the fixed literal `'connector-fatal'`. D-11/D-21 must keep this (V7).
