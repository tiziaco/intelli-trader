# Phase 05.3 Plan 10: Gate drop_pending on the single-channel spot leaf Summary

Gated `VenueAccount.drop_pending` to a no-op on the wired single-channel spot leaf so the `_pending` overlay is held from ack until terminal release — closing the CR-01 buying-power OVER-statement the unconditional D-15 drop-on-ack introduced — while keeping the drop-on-ack valid on the derivative leaf, with a production-premise `market_type='spot'` regression test (D-27).

## What was built

- **D-24 / CR-01 fix (`venue.py`):** added an intent-revealing read-only flag `_cash_stream_nets_holds` (`market_type != "spot"`, set in `__init__`). `drop_pending` now early-returns (negation-first guard-clause) when the flag is False, so on the spot leaf the overlay is HELD until terminal `release`. On the spot leaf `_write_balance_stream` is positions-only and `_venue_balance` is re-baselined solely by `snapshot()` (D-01), so the `_pending` overlay is the SOLE tracker of a resting order's cash hold — dropping it on ack snapped `available_balance` back to the full settled balance while the order was still open, admitting a second order against already-committed cash. The derivative leaf is unchanged (D-15 drop-on-ack stays valid, dormant-valid for a future stream-refreshed cash channel).
- **`release` untouched:** terminal pop stays unconditional on BOTH leaves — the spot hold is released at terminal, never leaked.
- **Option (b) NOT built:** no venue `used`/`free` netting into `_venue_balance` (rejected — would reintroduce the dual-channel cash authority D-01 killed). Confirmed by grep: the only `_venue_balance =` write remains inside `snapshot()`.
- **D-27 regression tests (`test_venue_account_drop_pending.py`):** new `test_spot_ack_holds_pending_overlay` constructs `VenueAccount(market_type='spot')`, snapshots a settled balance, reserves + acks, and asserts `available_balance` stays reduced by the hold while the order is open (RED on pre-fix code). Added spot + derivative terminal-`release`-still-pops control arms; reframed the existing ack-drops test as the derivative control arm proving the guard branches (not a blanket disable).
