CR-01: the WR-02 warm-verify MISS marks a symbol FAILED AFTER `absorb_warmup` has already appended to the
ring, and composes with the CR-02 next-poll FAILED-retry. Neither re-warm path is idempotent against a
re-delivered overlapping warmup window:
- `absorb_warmup` (`live_bar_feed.py`) does an unconditional `ring.append(bar)` with no timestamp dedup.
- `Strategy.update` (`base.py`) unconditionally increments `_bar_counts`, appends to `_recent_closes`, and
  advances the O(1) indicator recurrences.

Reachability is CONFIRMED (no exception required): any first warmup window shorter than the strategy
`min_period` -> FAILED -> next-poll retry re-appends the same bars -> the count crosses `min_period` OFF
DUPLICATES -> READY + subscribe -> the symbol becomes tradeable in LIVE on a duplicate-corrupted ring +
inflated/garbage indicator state, driving live order decisions off bad state.

The decided design is Option B "unified monotonic idempotency", Level 2 (`Bar.time` IS the ts_event — NO
new timestamp field). Exactly FOUR changes, all LIVE-ONLY and inert on the byte-exact backtest oracle:
1. FEED — `absorb_warmup` honors the existing `_last_delivered` cursor (reject `bar.time <= cursor` before
   `ring.append`). Zero new state; `_deliver` untouched; cursor stays `pd.Timestamp`.
2. STRATEGY — add a per-symbol `_last_bar_time: dict[str, datetime]` cursor; reject `bar.time <= last` in
   `update()` before any state mutation; clear it in `reset()` and `_reset_ticker()`.
3. DROP SEMANTICS (both cursors) — `==` (duplicate) SILENT drop; strict `<` (out-of-order) `warning` + drop.
   Reject is `<=`, never `<`.
4. RETRY POLICY (Level 2) — `on_poll` cadence-gates FAILED re-warms to one per bar interval; a warning fires
   after 3 consecutive failed re-warms; NEVER auto-drop / quarantine (Level 3 is explicitly OUT).

Purpose: eliminate the garbage-warmed-tradeable failure mode reached automatically via CR-02 auto-retry,
WITHOUT disturbing the byte-exact backtest oracle. Output: the two hardened live seams + the Level-2 retry
hygiene + a RED-FIRST regression proving the corruption is unreachable after the fix.

EXPLICITLY OUT OF SCOPE (do NOT touch): de-pandas-ifying the feed time model (the feed cursor STAYS
`pd.Timestamp` — see `livebarfeed-depandas-time-model-datetime.md`); OKX markets-map freshness / delisting
detection; Level 3 hard-ceiling / quarantine-drop / auto-drop; the review's Option A ring-reset
(`reset_symbol`) remediation — the decided design is the timestamp-guard idempotency (Option B), NOT ring
reset; any change to matching / portfolios / execution / the backtest feed.
