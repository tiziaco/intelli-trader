<objective>
Build `OkxConnector` — the thin authenticated session/transport primitive the three OKX arms
share. Async is bottled here: one asyncio loop on a daemon thread, one ccxt.pro client, one
sandbox knob, a call/spawn scheduling bridge, and lifecycle. No venue operations, no domain events.

Purpose: This is the genuinely-new code of the phase (loop-on-thread + coroutine bridging). Every
arm (02-03/02-04/02-05) receives this session injected and calls through its one client, so the
async containment (CONN-04) and single-bool sandbox routing (CONN-03) must be exactly right —
a sandbox misroute placing a live order is the phase's highest-severity threat.

Output: `itrader/connectors/okx.py` (`OkxConnector`), barrel export, and unit tests proving
sandbox->wspap routing, loop/bridge behavior, and the no-domain-import discipline.
</objective>
