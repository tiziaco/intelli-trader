# Phase 05.1 Plan 07: Per-market-type Venue-Truth Adapter — Spot Base-Balance Positions (D-03) Summary

**Made `VenueAccount` venue-type-aware so the wired BTC/USDC SPOT pair derives real per-symbol position truth from the base-currency balance total (`total[BASE]`) instead of the permanently-empty `fetch_positions` derivatives channel — ending the structural drift/orphan blindness (V17-04 truth arm) while the derivative channel and the SMA_MACD oracle stay byte-exact.**

## Accomplishments
- **Per-market-type truth channel (`venue.py`):** `VenueAccount.__init__` gains `market_type: Literal["spot", "derivative"]` (default `"derivative"`, backward-compatible) and `symbol`. `_base` is derived from the symbol's left leg (`"BTC/USDC"` → `"BTC"`).
- **Spot position derivation (`_extract_spot_position`):** reads `total[BASE]` from the ccxt balance payload, guarding None/missing keys BEFORE the `to_money(str(x))` edge (never `Decimal(float)`), keys the holding under the wired `symbol`, and surfaces a zero/absent base total as an empty map (flat = no row).
- **Balance-stream-borne spot truth:** `snapshot()` derives spot positions from the balance payload (skips the `fetch_positions` RPC, which is `[]` on spot); `_stream_account` derives spot positions from each `watch_balance` push; `start_streaming` does NOT spawn `_stream_positions` for spot (a `watch_positions` `[]` would clobber the base-derived holding).
- **Derivative channel unchanged:** derivative market types keep the exact `fetch_positions`/`watch_positions` + `_extract_positions` path — asserted by the SWAP regression test (long 0.5 BTC/USDT position row still surfaces).
- **Gates green:** new drift-cluster spot assertions GREEN (positions == `{"BTC/USDC": 0.5}` from `total[BTC]`; flat = `{}`; exact Decimal; SWAP regression intact); `mypy --strict` clean (228 files); SMA_MACD oracle byte-exact (`134 / 46189.87730727451`, `check_exact=True`).

## Decisions Made
- **Default `market_type="derivative"`:** all 12 existing `VenueAccount(...)` call sites (cache, conformance, supervisor-catchall, wiring, integration, e2e) construct without the new kwargs and keep the unchanged fetch_positions channel — zero ripple, backward-compatible.
- **Base from symbol, not a separate param:** the single wiring input (the traded pair) drives both the position key and the `total[BASE]` read — no split-brain between `base` and `symbol`.
- **Spot skips the positions stream/RPC:** spot has no positions channel; `watch_positions`/`fetch_positions` return `[]`, so spot position truth rides the balance path exclusively (spawning `_stream_positions` for spot would clobber the derived holding to `{}`).
