# Phase 05.1 Plan 05: Latched SystemStatus State Machine (D-05) Summary

**Delivered D-05 (ARCH-4 Layer 1): HALTED is now a latched state. Added `VALID_STATUS_TRANSITIONS` (HALTED maps to an empty set) next to `SystemStatus`, made `_update_status` the single enforced mutation seam, routed `halt()`'s direct write through it, made `start()` refuse RUNNING-from-HALTED post-reconcile, and added an operator-only `reset_halt()` as the sole off-table exit. The A3 reconcile-halt-latch test turned GREEN; the SMA_MACD backtest oracle stayed byte-exact and no lifecycle test regressed.**

## Accomplishments

- **Latch table (Task 1)** — `VALID_STATUS_TRANSITIONS: dict[SystemStatus, set[SystemStatus]]` in `core/enums/system.py`, mirroring `VALID_ORDER_TRANSITIONS`. `HALTED` maps to `set()` (the terminal analog of `OrderStatus.FILLED: []`); every non-terminal state may transition to `HALTED`. Exported through the `core/enums` barrel. 4-space indentation (matches the target file; the tab-indented donor shape was transcribed, not copied).
- **Single-seam enforcement (Task 2)** — `_update_status` is now the ONE point that writes `self._status` for a lifecycle transition. It rejects any transition not in `VALID_STATUS_TRANSITIONS[current]` (log-and-refuse, status unchanged), treats a same-state call as an idempotent no-op, sets `halt_reason` atomically with the flip, clears `halt_reason` when leaving HALTED, and returns whether the status actually changed.
- **halt() through the seam** — `halt()` no longer writes `self._status` directly; it routes through `_update_status(HALTED, ...)` and emits the single CRITICAL `ErrorEvent` only when it was the winning caller (WR-01 preserved). The halt-reason secret-scrub is intact (the reason is a fixed literal, never `str(exc)`; the refusal log binds only enum literals — T-05.1-10 / ASVS V7).
- **start() refusal** — after `reconciler.reconcile()` and BEFORE the thread spawn, `start()` checks `_is_halted()` and refuses to enter RUNNING: it does not spawn the processing loop (whose first action was the unconditional `_update_status(RUNNING)` clobber at `:993`), leaves the engine HALTED, and returns `False`. `_update_status`'s HALTED->RUNNING refusal is the second line of defence.
- **Operator reset_halt()** — the SOLE sanctioned HALTED exit (deliberately outside the table, via `force=True`). Returns the engine to STOPPED and clears `halt_reason` so a subsequent `start()` re-runs reconciliation + the baseline guard (verify-then-trust, F/U-9). A no-op returning `False` when not HALTED.

## Grep Gate

`self._status = ` assignments in `live_trading_system.py` are now exactly two: line ~102 (`__init__` construction — no prior state to transition from) and line ~549 (inside `_update_status`, the single seam). `halt()`'s former direct write (`self._status = SystemStatus.HALTED`) is gone; `reset_halt()` uses `force=True` through the seam rather than a direct write. > Note on the plan's `-> 0` grep-gate literal: it is unachievable as written — `_update_status` itself must contain `self._status = new_status` (that IS the single writer), and `__init__` must set the initial state before any transition exists. ## Deviations from Plan

- **[Rule 3 — Blocking fix] Same-state transitions treated as idempotent no-op.** `__init__` ends with `_update_status(SystemStatus.STOPPED)` while `self._status` is already STOPPED, and STOPPED->STOPPED is not in the (self-exclusive) transition table. Without handling, that construction call would spuriously log an "illegal transition" refusal on every `LiveTradingSystem()`. `_update_status` short-circuits `new_status == old_status` to a silent no-op (returns `False`), mirroring `VALID_ORDER_TRANSITIONS` (which also omits self-loops). This preserves the never-started `get_status()['status'] == 'stopped'` lifecycle test.
- No other deviations. F/U-8 and F/U-9 were open discretion resolved here (see key-decisions), not deviations.
