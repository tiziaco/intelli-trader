## Accomplishments

- **D-03 quote-wiring arm (Task 1, `live_trading_system.py:400`):** `VenueAccount` is now constructed with `quote_currency=_OKX_STREAM_SYMBOL.split('/')[1]` (USDC), `market_type='spot'`, and `symbol=_OKX_STREAM_SYMBOL` ‚Äî the single wiring input drives both the settlement currency (no USDT default) and the base-balance position truth (`total[BASE]`). This ends the V17-04 `quote_currency="USDT"` default at the root.
- **D-04 baseline guard (Task 1):** new `_run_session_baseline_guard()` runs strictly AFTER `reconciler.reconcile()` and BEFORE the `_is_halted()` RUNNING refusal / thread spawn. It compares the venue base-asset holding for the wired symbol against each active portfolio's post-reconcile believed position within the per-instrument dust epsilon (F/U-6, `_drift_precision` + `is_within_single_unit_tolerance`); any residual mismatch calls the latched `halt('baseline-residual')`. Quote-side cash is NOT asserted; divergence is NEVER auto-adopted; the halt reason is a fixed literal (ASVS V7 / T-05.1-10). Runs unconditionally when a VenueAccount is wired (not store-gated).
- **D-04 spurious-halt band (Task 2, `portfolio_handler.py:_compare_symbol_drift`):** added a signed `just_applied_fill_qty` param. On the immediate on-fill compare, when the venue truth still matches the pre-fill holding (`engine_qty - just_applied_fill_qty`) within the band, the skew is exactly the fill the venue snapshot hasn't streamed yet ‚Äî absorb it (log at INFO) and defer to the periodic sweep. `on_fill` passes the signed delta (`+qty` BUY / `-qty` SELL); the per-bar sweep and the baseline guard pass `None`, so genuine venue-ahead divergence still halts.
