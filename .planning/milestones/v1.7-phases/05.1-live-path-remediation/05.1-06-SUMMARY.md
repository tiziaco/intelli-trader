# Phase 05.1 Plan 06: Account Contract Enforcement (D-02) Summary

**Enforced the account contract at the type + runtime level: re-typed `Portfolio.account` to the `Account` ABC, replaced both bare `cast(SimulatedMarginAccount, ...)` narrowings with a single `_require_margin_account` isinstance guard that raises a typed `StateError` BEFORE any mutation (closing the V17-14 partial-mutation-on-margin hazard), and made `mypy --strict` see the ABC-vs-concretion live wiring via a typed conformance module — so account-contract drift can never ship silently again. Oracle stays byte-exact; the 5 `test_venue_account_cache.py` REDs turn GREEN.**

## Accomplishments

- **`Portfolio.account` re-typed to the ABC (`portfolio.py:104`):** field annotation moved from `SimulatedCashAccount` to `Account`, so the live `VenueAccount` leaf (assigned at `live_trading_system.py:565`) is type-valid on the field. No behavior change — every read Portfolio makes off `self.account` (`balance`, `available_balance`, `reserved_balance`, `assert_funds_invariant`, `apply_fill_cash_flow`) is on the 7-member ABC from 05.1-04.
- **Both margin casts replaced by an isinstance guard (`portfolio.py:438`→`_process_transaction_margin`, `:834`→`_accrue_short_carry`):** new `_require_margin_account(operation)` helper raises `StateError(portfolio_id, type(account).__name__, required_state="SimulatedMarginAccount", operation=...)` when the leaf is not a margin account, BEFORE any position/cash mutation. A bare `cast` was a type-level no-op that let a venue/cash leaf reach a margin-only method (`lock_margin`/`accrue_borrow_interest`) and `AttributeError` *mid-settlement, after the position already moved*; the guard now fails loud at the boundary.
- **Conformance test extended (Task 1):** added `test_margin_op_on_venue_account_raises_before_mutation` — a venue-linked + margin-configured portfolio driven through a margin BUY raises `StateError` with the position and transaction ledger asserted untouched (the V17-14 no-partial-mutation proof).
- **F/U-4 mypy visibility (`account/conformance.py`, NEW):** a never-imported, strict-checked module mirrors `_link_venue_account_to_portfolios` — `_assert_leaf_conforms` reads the ABC surface on each leaf and `_assert_live_wiring` assigns cash/margin/venue leaves onto `Portfolio.account`. If the field were re-narrowed to a concretion, `portfolio.account = venue` fails `mypy --strict` here. `mypy --strict itrader` now covers **228** files (was 227) and is clean.
- **7 cache-test renames:** the 7 `.available` assertions in `test_venue_account_cache.py` renamed to `available_balance` (tracking the 05.1-04 ABC rename) — the 5 deferred REDs turn GREEN.
- **Permanent conformance gate:** `test_account_conformance.py` exercises all three leaves through admission read + reserve/release + BUY settle + SELL atomicity + serialization, plus the new margin-guard case — **16 passed**.
