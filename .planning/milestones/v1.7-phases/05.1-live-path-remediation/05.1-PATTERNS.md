### Money edge (all waves)
**Source:** `venue.py:123` / `core/money.py`. `to_money(str(x))` at every venue-float ingress;
NEVER `Decimal(float)`. Guard None/missing keys BEFORE the edge.
**Apply to:** every VenueAccount/reconcile/venue-payload touch (D-01/03/08/09/16).

### Typed exceptions (D-02/04)
**Source:** `core/exceptions/` — `StateError(entity_id, current_state, required_state, operation)`
(see `venue.py:228-233`), `InsufficientFundsError(required_cash, available_cash)`,
`ConfigurationError`. Raise typed, never bare `Exception`/bool. Fail-fast in backtest; publish-and-
continue in live (run-mode policy).

### Transition-table-at-one-seam (D-05)
**Source:** `core/enums/order.py:81` `VALID_ORDER_TRANSITIONS` enforced conceptually like the order
mirror. One writer, one rule (V17-03 root cause = multiple writers, no rules).

### SqlBackend live wiring (D-07/D-10)
**Source:** `live_trading_system.py:257-285` — lazy SQL import, `SqlBackend(SqlSettings(...))`,
`CachedSql*` store-first wrapper, retain backend for `stop()` dispose, factory `'live'` arm.
**Apply to:** the 5 portfolio storage sites + the durable halt record.

### Secret-scrub on halt/error egress (D-11)
**Source:** `okx.py:569/581` — log `type(exc).__name__` + fixed cause, NEVER `str(exc)`; halt
reason is a fixed literal (`'connector-fatal'`). Security V7 — preserve.
