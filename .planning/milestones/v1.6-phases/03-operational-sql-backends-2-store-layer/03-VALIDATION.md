## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | pytest 8.4.2 (+ pytest-cov 7.1.0), testcontainers Postgres (Phase-1 fixture) |
| **Config file** | `pyproject.toml [tool.pytest.ini_options]` (`testpaths=["tests"]`, `filterwarnings=["error",...]`, `--strict-markers`, `--strict-config`) |
| **Quick run command** | `poetry run pytest tests/integration/storage/test_sql_<concern>_storage.py -x` |
| **Full suite command** | `make test` (or `poetry run pytest tests` in a worktree — see MEMORY: make-test `.env` aborts) |
| **Estimated runtime** | full suite ~ (existing) + container spin-up; per-concern file ~seconds once container is warm |

*Tests live under `tests/integration/storage/` — a package-less dir. Do NOT add `__init__.py` (test-dir collision note).*

---

## Sampling Rate

- **After every task commit:** concern's quick `pytest ... -x` + `poetry run mypy itrader`
- **After every plan wave:** `poetry run pytest tests/integration/storage -x` + oracle test
- **Before `/gsd:verify-work`:** full suite green + `mypy --strict` clean + oracle byte-exact (134 / `46189.87730727451`) + W1/W2 no-regression vs v1.5 baseline (15.7 s / 152.8 MB)
- **Max feedback latency:** per-concern file is the quick loop; Postgres-only money cases require Docker (skip gracefully without it)
