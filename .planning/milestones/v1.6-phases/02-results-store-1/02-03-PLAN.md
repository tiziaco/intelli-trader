<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → top_runs/top_portfolios | a `metric` string drives an `ORDER BY` (column name cannot be bound) |
| stored blob → reader | gzip/JSON `run_artifacts` blob is decompressed + parsed on read (untrusted if the DB is shared) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering / SQL Injection | `top_runs` / `top_portfolios` ORDER BY | mitigate | Resolve the ORDER BY through a `MetricName`→`Column` dict (bound Column objects), never an f-string/`text()`; `MetricName` `Literal` + `mypy --strict` block any free string; a grep gate asserts no f-string order_by. |
| T-02-05 | Tampering / DoS | gzip/JSON deserialization of `run_artifacts` blobs | mitigate | Decompress with stdlib `gzip` (single-shot `decompress`) + `pd.read_json(orient="split")` on the store's OWN written bytes; no `pickle`, no eval; round-trip tests fix the codec. The ephemeral SQLite file is process-local (not a shared/multi-tenant DB this phase). |
| T-02-06 | Tampering | idempotent `create_all` clobbering data | mitigate | `create_all(checkfirst=True)` (D-12) appends; never DROP — re-runs accumulate, no data loss. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages (gzip/io/uuid stdlib; pandas/sqlalchemy already deps) — no install task. |
</threat_model>
