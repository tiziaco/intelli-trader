<objective>
Build `CachedSqlSignalStorage` — the live-only decorator wrapper for the strategy/signal seam (D-04) — composing
the untouched Phase-3 `SqlSignalStorage` (system of record) with an in-memory `InMemorySignalStore` working set.
This is the **simplest** of the three wrappers: signals are append-only and NEVER purged (D-02's purge gate
applies to orders/positions, not signals), so there is no terminal gate and no read-through — the cache is a pure
full mirror. `add` is store-first then cache-mirror; `get_all` / `by_strategy` / `by_ticker` serve from the full
cache mirror; an optional `rehydrate()` repopulates the mirror from the store. Wire the factory `'live'` arm to
return the wrapper.

Per **D-01**, this wrapper is **built + component-tested only** — the live composition root's hardcoded
`'backtest'` signal store (`live_trading_system.py:113`) stays untouched (N+4). Resolves **A5**: the new module
enters `mypy --strict` (no override).

Purpose: complete the `InMemory` / `Sql` / `CachedSql` per-concern triple for the signal seam, keeping the
gate-passed `SqlSignalStorage` untouched (D-04). Output: `cached_sql_storage.py`, the wired factory `'live'` arm,
and the testcontainers Postgres integration suite.
