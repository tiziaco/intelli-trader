<objective>
Add the strategy→signal authoring surface for per-intent ENTRY price and order_type (SIG-01/SIG-02).
This is the ONLY genuinely-new surface in Phase 5: the order/admission/matching machinery that consumes
`signal_event.order_type` + `signal_event.price` is ALREADY wired (`admission_manager._build_primary_order`
dispatches MARKET/LIMIT/STOP; `MatchingEngine._evaluate` fills limit-or-better / pessimistic-stop). This
plan only feeds those existing consumers per-intent values instead of strategy-fixed ones.

Purpose: A single strategy can specify MARKET/LIMIT/STOP and a limit/stop entry price per intent, instead
of a fixed-per-instance order_type hardwired to the decision-bar close.

Output: Two new `SignalIntent` fields, four factory methods on the strategy base, the retirement of the
per-instance `Strategy.order_type` class attr, the per-intent handler fan-out, the `SignalRecord` audit
fields, and a Wave-0 factory unit test. The existing oracle stays byte-exact (134 / 46189.87730727451)
because the reference SMAMACD uses plain `buy()`/`sell()` which stay MARKET-at-close.
</objective>
