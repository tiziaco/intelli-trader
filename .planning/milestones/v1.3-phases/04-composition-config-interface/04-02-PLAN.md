Collapse the fat `TradingSystem.__init__` + `run()` into the D-04 factory/holder shape and fold the composition-root cleanups (W4-02/03/05/06/07). The direct-construction path (oracle + integration tests) must keep working through the renamed class — only the class name changes for those sites (Wave 4 swaps them); the spec-driven path is the NEW factory.

The D-15 late-binding property is ORACLE-DARK (the golden run pins fees at 0, so the byte-exact gate can never catch a stale-fee regression). Task 2 therefore carries an explicit late-binding behavior test: after a fee-model hot-swap the FeeModelCommissionEstimator must return the NEW model's estimate, proving the adapter reads `exchange.fee_model` in `__call__` and never captures it at `__init__`.

Purpose: deliver the engine-level composition API and the construction-time ExchangeConfig threading that replaces the Phase-7 D-14 post-construction re-init seam, byte-exact.
Output: compose.py, backtest_runner.py, slimmed backtest_trading_system.py, reporting/summary.py, the symbol-seeding integration test, the commission late-binding unit test.
