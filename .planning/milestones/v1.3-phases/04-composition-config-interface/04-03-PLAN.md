<objective>
Roll the canonical `update_config(dict) -> None` contract onto the five config-model handlers (COMP-02): `PortfolioHandler`, `Portfolio`, `SimulatedExchange`, `ExecutionHandler`, and `OrderManager`/`OrderHandler` (via the Wave-2 `OrderConfig`). Standardize the three existing inconsistent forms (`PortfolioHandler` Dict→bool, `Portfolio`/`SimulatedExchange` **kwargs→None) onto one: `deep_merge(self.config.model_dump(), updates)` → `Config.model_validate(merged)` → atomic-swap, wrapping pydantic `ValidationError` into `core.ConfigurationError` (D-07/D-08/D-09). Promote `PortfolioHandler._deep_merge` to a shared `config/merge.py` helper (WR-04 sibling-preservation — do NOT re-derive per handler). Each handler reproduces its existing post-swap cache re-derivations (Pitfall 1). Fix the `SimulatedExchange.configure()` Protocol coupling (Pitfall 2).

These methods are oracle-dark by construction (the golden run never fires `update_config`), so correctness is proven by direct unit tests + the byte-exact gate holding trivially.

Purpose: deliver the uniform runtime config-update surface COMP-02 requires on every config-model handler, single web-catchable error contract.
Output: shared merge helper + 5 migrated handlers + per-handler contract unit tests.
</objective>
