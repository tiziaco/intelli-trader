<decisions>
## Implementation Decisions

### Convention Documentation Home
- **D-01:** Document the four conventions in **both** `.planning/codebase/CONVENTIONS.md`
  (authoritative, full write-up) **and** root `CLAUDE.md` (concise pointer — it is what
  Claude reads every session). CONVENTIONS.md is the formal home; CLAUDE.md carries a
  short cross-reference so the convention survives in the per-session context even though
  CONVENTIONS.md is regenerated by map-codebase.
- **D-02:** The four conventions to document:
  1. **config-enum exception (W2-13)** — the seven `str, Enum` config-domain enums
     (`FeeModelType`, `SlippageModelType`, `PortfolioType`, etc. in
     `config/exchange.py`, `config/portfolio.py`, `config/system.py`) live in `config/`
     not `core/enums/` **by design** — they are consumed only by their co-located Pydantic
     models; relocating would invert the core→config dependency. Adjudicated acceptable.
  2. **broad-`except` run-mode policy** — backtest is fail-fast (`EventHandler._on_handler_error`
     re-raises); live is publish-and-continue (`LiveTradingSystem` emits `ErrorEvent`,
     keeps draining). Note: CLAUDE.md already describes this — make sure it is stated as
     an intentional policy, not an inconsistency.
  3. **tab/space indentation hazard** — tabs in handler modules; 4 spaces in `config/`,
     `core/`, `price_handler/feed/`, events package. Already partially in CLAUDE.md;
     reinforce "match the file, never normalize."
  4. **dual-layer validator overlap (W4-04)** — `order_validator.py:176-260` /
     `simulated.py:375-434` price/qty validation is **justified-by-decision**
     (defense-in-depth; the exchange must validate independently because `create_order`
     and live paths bypass the domain validator). Document, do **not** remove.

### Stale-Doc Edit Style
- **D-03:** **Trim to truth + terse traceability ref.** These are current-state docs, not
  append-only logs — annotate-and-keep would accumulate resolved-item cruft.
  - **CONCERNS.md:** remove the obsolete `screener_event_handler` "known bug" entry
    outright (file deleted in M2-11 — the concern can no longer exist).
  - **ROADMAP.md 999.5-(d):** the line already reads FL-01/FL-02 "were marked **done**";
    the remaining fix is to drop the now-redundant self-referential
    "stragglers … corrected in v1.2 Phase 1 / DEAD-02" forward-pointer (it becomes
    self-stale the moment DEAD-02 lands) and leave one tight factual closure line:
    FL-01/FL-02 closed in v1.1 (quick 260610-sjp). **Net reduction, not growth.**
  - Do NOT do a broader ROADMAP/CONCERNS prune — out of scope (scope creep).

### Importer-Update Scope
- **D-04:** **Full sweep + suite gate** — "zero importer breakage" requires real importer
  edits because the targets are NOT importer-free:
  - `OrderBase` is **inherited** by `OrderHandler(OrderBase)` in `order_handler.py:17` and
    **re-exported** in `order_handler/__init__.py` (lines 11, 19). Make `OrderHandler`
    standalone (no base / `object`) and drop the `__init__` re-export. Keep `OrderStorage`
    (live ABC in the same file).
  - The 3 ABCs share `portfolio_handler/base.py` with the **live** `PortfolioStateStorage`
    (imported by 4 managers — cash/position/transaction/metrics). This is a **partial-file
    edit**: delete only the 3 ABC classes + the orphan `get_last_close`, keep
    `PortfolioStateStorage`.
  - **False alarm cleared:** `tests/unit/events/test_bar_event_ohlc.py:62`
    (`test_no_accessor_methods_exist`) name-references `get_last_close` but asserts
    `not hasattr(bar, name)` on a **`Bar`** object — unrelated to the ABC method. Deleting
    the ABC does NOT break it; do not touch this test.
  - Verify with grep (no remaining symbol references), full suite green, `mypy --strict`
    clean.

### Dead-Code Scope Discipline
- **D-05:** **Touched-path cleanup, logged** (per `CLEANUP-STANDARD.md`). Remove now-dead
  artifacts in files already being edited — e.g. imports/typing left orphaned in
  `base.py` after ABC removal (`abc.ABC`/`abstractmethod`, unused `typing` names). Anything
  larger than the touched files gets **logged to FIX-LIST.md**, not folded in. No
  repo-wide dead-code hunt.

### Claude's Discretion
- Exact wording/placement of the convention write-ups within CONVENTIONS.md sections
  (Code Style / Error Handling / Type Hints / Comments & Docstrings are the natural homes).
- Plan/wave decomposition (deletions vs doc edits as separate or combined plans) — planner's call.
