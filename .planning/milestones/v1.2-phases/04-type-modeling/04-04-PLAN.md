<objective>
Bring the order-domain core vocabularies onto the canonical class-based enum form (TYPE-04 + TYPE-03)
and freeze the bracket DTO (TYPE-01), all on the FRAGILE reconciliation/reservation-release path:
D-01 convert `OrderStatus`/`OrderCommand` from the functional auto-int form to class-based string-valued
enums; D-04 add `OrderOperationType`/`OrderTriggerSource` (value-equal to the current string literals),
retype the `OperationResult.operation_type` field/factory-params from `str` to `OrderOperationType`, and
swap every `operation_type=`/`triggered_by=` call-site in `order_manager.py`/`order.py`; D-07 add
`slots=True, kw_only=True` to the already-frozen `_PendingBracket`.

Purpose: closed canonical vocabularies replace functional/stringly-typed forms. The int->string `.value`
flip is byte-inert (D-02 audit: status serializes via `.name`, never `.value`); the D-04 swap is
value-equal (the carrier type changes, the value does not), so audit records, logs, and comparisons stay
byte-identical. ⚠ The reconciliation/reservation-release LOGIC is FROZEN — this is a pure
annotation + value-equal literal swap, NOT structural code-motion.

⚠ DEPENDENCY/WAVE NOTE: this plan is `wave: 2` and `depends_on: ["04-01", "04-02", "04-03"]` (the
ROADMAP marks Wave 2 as blocked-on-Wave-1). The explicit depends_on enforces the between-wave oracle
re-run gate that protects this FRAGILE order-domain path — 04-04 must NOT run concurrently with Wave 1.
The `operation_type` field retype is co-located HERE (NOT in Plan 04-01) on purpose: `OrderOperationType`
is DEFINED in this plan (Task 2), so retyping `OperationResult.operation_type` to it must live in the same
plan that introduces the enum — putting it in 04-01 (Wave 1) would create a backward dependency on a

(frozen/slots/kw_only + tuple fields); this plan only changes the `operation_type` annotation on top of
that frozen shape. The shared file `operation_result.py` is touched by 04-01 (Wave 1) then 04-04 (Wave 2)
— the dependency chain (04-04 depends_on 04-01) keeps that sequencing acyclic.
Output: core/enums/order.py + operation_result.py + 2 order-domain modules + the enum unit tests; oracle
byte-exact, mypy --strict clean, e2e 58/58.
