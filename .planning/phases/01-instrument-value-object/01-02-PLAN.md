<objective>
Build the symbol->`Instrument` resolution layer in `universe/` and wire it through the
backtest composition root. This delivers INST-02 (the declared -> inferred(guarded,
string-read, 8dp cap) -> default precision ladder, with BTCUSD always on the declared
branch) and the INST-03 `ExchangeLimits` demotion (`SimulatedExchange` resolves
`min_order_size` Instrument-first, ExchangeLimits-fallback).

for symbol->Instrument resolution (D-03, no parallel registry). A `Universe` read-model
(D-06) composed over the existing pure functions (D-07) becomes the injectable seam that
the exchange (now) and the margin code (later phases) consume.
Output: `universe/instruments.py` (new), `universe/universe.py` (new), updated
`universe/__init__.py`, demoted `config/exchange.py::ExchangeLimits`, Instrument-aware
`SimulatedExchange`, wired `backtest_runner.py` + `compose.py`, plus three unit tests.

**Byte-exact discipline (CRITICAL):** This plan must hold the SMA_MACD oracle
byte-for-byte (proven in plan 01-03). Three load-bearing guards:
1. BTCUSD declares price 8dp + quantity 8dp and leaves `min_order_size` UNDECLARED (None)

2. Inference (D-09) is NEVER invoked on BTCUSD (declared wins, D-10) ‚Äî it is covered by a
   SYNTHETIC non-oracle symbol fixture only.
3. The Trap-4 wiring order in `backtest_runner.py` (membership derive -> feed.bind ->
   ping-grid -> precompute) is preserved exactly; `feed.bind` receives the SAME
   set-derived `list[str]` it does today (Pitfall 4).
