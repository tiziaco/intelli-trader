<objective>
Wire `LiveBarFeed` into the `LiveTradingSystem` composition root and prove the recurring milestone
gate. Four moves: (1) swap the `BacktestBarFeed` placeholder for a lazy-imported `LiveBarFeed`;
(2) register the D-13 `RawBarConsumer(required_history_depth = max(strategy.warmup))` on the LIVE
feed so `cache_capacity()` derives to 100 (without this the ring + warmup size to 1 and SMA_MACD
never warms → zero trades → oracle fails — RESEARCH Pitfall 1, the phase's single most likely
correctness failure); (3) inject the real OKX provider into the feed via `self.feed.set_provider(
self._okx_data_provider)` inside the okx arm (03-02's public setter — it writes the PRIVATE
`self._provider` that `warmup()`/gap-backfill read); (4) wire `set_bar_sink(feed.update)`,
`bind(global_queue, members)`, and a pre-`start_stream` warmup so `LiveBarFeed` replaces
`TimeGenerator`'s driver role (FEED-05) while keeping the dormant TIME route a valid callable (D-05).

Then lock the gate: extend the inertness probe's `_FORBIDDEN` to forbid `live_bar_feed` on the
backtest import path, and confirm the oracle stays byte-exact.

Purpose: FEED-05 + D-13 + the recurring milestone gate (oracle byte-exact + LiveBarFeed inert on
the backtest hot path).
Output: wired composition root + provider-injection construction test + route-order integration test
+ extended inertness probe.
