## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-DOUBLEDELIVER | Tampering (correctness) | reconnect backfill vs resumed stream | mitigate | Backfill is composed with the resumed stream so a re-sent bar hits the `update()` duplicate branch (D-06) — no double-delivery; the boundary check gates on completed-bar open-time, not outage duration (D-08). |
| T-03-03-STARVE | Denial of Service (correctness) | warmup depth | mitigate | Warmup depth is `cache_capacity() + margin` (derived, D-09/D-10) so indicators reach readiness — the Pitfall-1 zero-trades failure is guarded by an explicit is_ready integration assertion. |
| T-03-03-PARITYDRIFT | Tampering (parity) | replay path | mitigate | Warmup + reconnect route through the identical `update()` guard — no bulk `warmup_from` fast-path exists (LX-09), so there is no second state-building path to diverge from the live path. |

No high-severity threats: backfill is a REST read owned by the trusted provider; the feed only
replays through its own already-hardened guard.
