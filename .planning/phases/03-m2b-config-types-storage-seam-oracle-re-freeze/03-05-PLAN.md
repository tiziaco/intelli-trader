<objective>
Config collapse to Pydantic v2 (M2-06, #12/#13 — completes the #34/TD2 dual-config span). Replace
the 3,380-line `config/` package + the flat `itrader/config.py` shadow with ~600-900 lines of
Pydantic v2 models and a minimal `pydantic-settings` layer (D-01/D-02/D-03).

Three sub-steps (one task each):
1. Build the Pydantic models + `Settings(BaseSettings)` with fail-loud secrets, and move reference-
   data literals to `core/constants.py` (fixing the implicit string-concat bug in FORBIDDEN_SYMBOLS).
2. Absorb the flat `itrader/config.py` shadow (the REAL source of `TIMEZONE`/`FORBIDDEN_SYMBOLS`,
   loaded via importlib in `config/__init__.py:60-76` — RESEARCH Pitfall 1, NOT in CONTEXT.md):
   `TIMEZONE`→`Settings`, `FORBIDDEN_SYMBOLS`→`core/constants.py`, then DELETE the flat module + the
   importlib shim; rewire the `config.TIMEZONE`/`FORBIDDEN_SYMBOLS` readers.
3. Clean-break rewire: delete the `config/__init__.py` getters + the registry/provider/validator/
   schema machinery; rewire the ~4 in-scope consumers to construct models directly. `mypy --strict`
   catches any missed site.

Security-relevant change (the one in this phase): secrets (DB URL, API keys) are declared as
required-no-default `SecretStr`/`Optional` so they FAIL LOUD if a live path ever instantiates
`Settings` — satisfying M2-06's explicit "no working secret defaults" criterion WITHOUT wiring
DB/exchange auth (that is D-live, deferred).

Purpose: Replace hand-rolled config machinery with standard Pydantic idioms; the round-trip and
fail-loud secrets come for free. Behavior-preserving on the backtest path.
Output: collapsed Pydantic config, flat shadow deleted, consumers rewired, mypy --strict clean.
</objective>
