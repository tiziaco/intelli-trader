<objective>
Restructure the config hierarchy into the load-bearing foundation of the runtime-config platform (D-05/D-06/D-07/D-08/D-09). Rename the existing `SystemConfig` aggregator into a NEW top-level `frozen=True` `ITraderConfig` root whose immutable base params (`rng_seed`, `environment`, identity/dirs) live directly on the frozen base (D-04) and whose domain sub-models are mutable overlays with `validate_assignment=True` (D-13). Demote `SystemConfig`'s lifecycle knobs into a `SystemSettings` sub-model (D-08), fold `MonitoringSettings`'s two used fields into a new `UniverseConfig` sub-model, delete the verified-0-ref `PerformanceSettings`/`MonitoringSettings` (D-09), and move `rng_seed`/`universe.*` read sites.

This touches the config the BACKTEST reads, so it is the phase's oracle- + inertness-gated decision. There is NO separate `RuntimeConfig` wrapper (D-05 owner override of design spec §6c) — the single imported `config` singleton IS the runtime config.

Purpose: give the mutation path (Wave 2) a frozen-base allowlist structure and a mutable overlay to route into, while keeping the backtest byte-exact.
Output: `ITraderConfig` aggregator, `SystemSettings`/`UniverseConfig` sub-models, the flipped import singleton, moved read sites, and the RTCFG-01/04 unit test.
</objective>
