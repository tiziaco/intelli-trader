<objective>
Build the runtime-mutation core on the LIVE engine (live-only, backtest-dark): a scoped `ConfigUpdateEvent(scope, key, value)` on the CONTROL plane (D-23) and an engine-thread `ConfigRouter` that runs validate → persist → apply → push (D-15) with default-deny scope→owner dispatch (D-11/D-21), Pydantic `validate_assignment` validation (D-13), a venue-kind predicate (D-14/RTCFG-05), and deduped WARNING-`ErrorEvent` rejection surfacing (D-16). Wire the `CONFIG_UPDATE` CONTROL route into the pre-declared empty slot via `LiveRouteRegistrar`.

There is NO standalone allowlist artifact (D-11) — the mutation boundary is the compounding structure: the frozen base blocks immutable keys (RTCFG-04, Plan 01), the router's scope→owner map is the routable-key set, and `validate_assignment` validates. No copy-on-write/atomic-swap snapshot machinery (D-03) — the config is a plain engine-thread-owned mutable object, single writer + single live reader.

Purpose: deliver RTCFG-02/04/05 mutation semantics with a unit-testable router; Wave 3 wires the real stores + ingress + restart layering around it.
Output: `ConfigUpdateEvent`, `ConfigRouter`, the wired CONFIG_UPDATE route, and the router unit test.

Self-gated planner contributions (recorded per phase brief): ai-integration/API-coverage = no-op (the CONFIG_UPDATE ingress is an internal event-admission boundary, not a new external API/SDK); assumption-delta = no-op (`venue:{name}`/`portfolio:{id}` are already-plural existing entities); schema-gate = no-op (SQLAlchemy + Alembic, not a push-based ORM).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CONTROL-bus event → engine-thread router | a dequeued `ConfigUpdateEvent` crosses into the single-writer engine thread |
| router → durable store | the coerced value is persisted before it is applied live |
| config value → venue execution arm | a fee/slippage mutation reaches an exchange only after the venue-kind predicate |

| T-9-venue-kind | Tampering | venue-scope apply path | high | mitigate | Venue-kind predicate (D-14) rejects fee/slippage mutation for a live venue — real-venue fees come from actual fills, not engine config (RTCFG-05) |
