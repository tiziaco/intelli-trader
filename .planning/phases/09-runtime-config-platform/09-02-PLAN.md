<objective>
Build the runtime-mutation core on the LIVE engine (live-only, backtest-dark): a scoped `ConfigUpdateEvent(scope, key, value)` on the CONTROL plane (D-23) and an engine-thread `ConfigRouter` that runs validate ‚Üí persist ‚Üí apply ‚Üí push (D-15) with default-deny scope‚Üíowner dispatch (D-11/D-21), Pydantic `validate_assignment` validation (D-13), a venue-kind predicate (D-14/RTCFG-05), and deduped WARNING-`ErrorEvent` rejection surfacing (D-16). Wire the `CONFIG_UPDATE` CONTROL route into the pre-declared empty slot via `LiveRouteRegistrar`.

There is NO standalone allowlist artifact (D-11) ‚Äî the mutation boundary is the compounding structure: the frozen base blocks immutable keys (RTCFG-04, Plan 01), the router's scope‚Üíowner map is the routable-key set, and `validate_assignment` validates. No copy-on-write/atomic-swap snapshot machinery (D-03) ‚Äî the config is a plain engine-thread-owned mutable object, single writer + single live reader.

Purpose: deliver RTCFG-02/04/05 mutation semantics with a unit-testable router; Wave 3 wires the real stores + ingress + restart layering around it.
Output: `ConfigUpdateEvent`, `ConfigRouter`, the wired CONFIG_UPDATE route, and the router unit test.

| Boundary | Description |
|----------|-------------|
| CONTROL-bus event ‚Üí engine-thread router | a dequeued `ConfigUpdateEvent` crosses into the single-writer engine thread |
| router ‚Üí durable store | the coerced value is persisted before it is applied live |
| config value ‚Üí venue execution arm | a fee/slippage mutation reaches an exchange only after the venue-kind predicate |
