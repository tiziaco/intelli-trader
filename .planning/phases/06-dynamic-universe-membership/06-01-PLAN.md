<objective>
Build the pure, backtest-safe FOUNDATION seam for dynamic universe membership: the
`UniverseUpdateEvent` (msgspec `Event` subclass), its `EventType.UNIVERSE_UPDATE`
discriminator, an explicit-empty `_routes` entry, and `Universe.apply(desired) -> UniverseDelta`
that mutates `_members` in place (preserving the feed's by-identity bind, Pitfall 4) plus a
leaving-set surface for the later remove-policy admission gate.

This plan implements D-03 (Universe owns mutation, stays queue-free) and D-04 (the
`UniverseUpdateEvent` seam, distinct from `ScreenerEvent`). It ships NO consumers and NO side
effects — the route is deliberately empty; live consumers are wired live-only in plan 05. The
empty-delta fast path is the oracle-dark guarantee: single-symbol SMA_MACD yields `desired ==
current`, so `apply` returns an empty delta and nothing is emitted.

Purpose: give plans 03/04/05 a settled event + mutation contract to build against.
Output: the new event type, the `Universe.apply`/`UniverseDelta`/leaving-set surface, unit tests.
</objective>
