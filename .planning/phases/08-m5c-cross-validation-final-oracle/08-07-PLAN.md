<objective>
Build the cross-validation orchestrator `scripts/cross_validate.py` and a pure
reconciliation module `scripts/crossval/reconcile.py`, then emit the committed
evidence artifact `tests/golden/CROSS-VALIDATION.md`.

The orchestrator (a) precomputes the shared `ta` SMA(50)/SMA(100)/MACD-hist(6,12,3)
arrays ONCE from the golden BTCUSD CSV and passes the identical arrays to every
engine (D-03 — collapses indicator-library divergence to zero); (b) runs each
force-matched engine module from 08-05 (backtesting.py + backtrader) and the
optional 08-06 Nautilus module behind a try-guard; (c) reads iTrader's frozen
`tests/golden/*` for the iTrader side; (d) recomputes the headline metric set for
EVERY engine through iTrader's own `itrader/reporting/metrics.py` so the comparison
is apples-to-apples (D-04, RESEARCH risk #5 — never trust engine-native annualized
metrics); (e) builds the D-02 trade-level-primary + D-04 metric-level-secondary
reconciliation tables; (f) emits `tests/golden/CROSS-VALIDATION.md` with both tables
plus a per-divergence section stub that 08-08 fills with root-cause dispositions.

Purpose: produce the durable committed evidence that iTrader's SMA_MACD numbers are
trustworthy across engines (M5-10). This plan produces the REPORT + TABLES only; the
per-divergence ROOT-CAUSE trace and any bug-fix re-freeze is the separate 08-08.

Output: `scripts/cross_validate.py`, `scripts/crossval/reconcile.py`,
`tests/golden/CROSS-VALIDATION.md`.
</objective>
