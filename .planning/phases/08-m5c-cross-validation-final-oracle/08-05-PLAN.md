  flat (allow_increase=False), LONG_ONLY.

  (window_slow), WIN=3 (window_sign). Source identifiers in SMA_MACD_strategy:
  `ta.trend.SMAIndicator(close, window, True).sma_indicator()` and
  `ta.trend.MACD(close, window_fast=FAST, window_slow=SLOW, window_sign=WIN, fillna=False).macd_diff()`.
  Reuse those EXACT calls in the precompute helper (do not re-derive the math).
- THE QUIRK (replicate verbatim — SMA_MACD_strategy lines 88-94): the
  `short_sma[-1] >= long_sma[-1]` SMA filter gates BOTH the entry AND the exit. The exit
  branch is `elif` NESTED INSIDE the filter `if`. When the filter is False, a held long is
  NOT exited on a MACD down-cross — it stays open until a later bar where the filter is True
  AND a down-cross occurs.
    - Entry: filter True AND MACDhist[-1] >= 0 AND MACDhist[-2] < 0 -> buy.
    - Exit:  filter True AND MACDhist[-1] <= 0 AND MACDhist[-2] > 0 -> close.
    - Warm-up: while fewer than max(long_window, 100) bars are available -> no signal.
- Fill timing: next-bar-open. A signal decided on completed bar T fills at bar T+1's open.
  This is backtesting.py's default (trade_on_close=False) and backtrader's default for a
  market order issued in next() — do NOT enable cheat-on-open/close.
- FRACTIONAL-UNITS LANDMINE (RESEARCH §1, highest-ranked risk): plain backtesting.py
  `Backtest` and backtrader's `PercentSizer` both floor position size to whole units -> 0 BTC
  -> 0 trades -> meaningless comparison. backtesting.py MUST use `FractionalBacktest`;
  backtrader MUST use a CUSTOM sizer that returns a FLOAT `0.95 * cash / price`.
- Engine modules are NEW code under scripts/ -> 4-space indentation (spaces, per CLAUDE.md).
  They MUST NOT be imported under tests/ or at pytest-collection time (D-10): importing
  backtesting.py (bokeh) / backtrader (numpy-2 deprecations) would trip
