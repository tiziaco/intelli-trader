<objective>
For EVERY divergence recorded in 08-07's `tests/golden/CROSS-VALIDATION.md` reconciliation
table, trace it to a root cause and disposition it per **D-05 (root-cause decides; iTrader is
correct unless proven otherwise)**:

- **(a) Cause = a genuine iTrader bug** → fix the result-bearing engine code → author a named
  `tests/golden/REFREEZE-M5C-<bug>.md` expected-diff note → get owner sign-off (blocking
  checkpoint) → re-freeze the golden artifacts. This is a **sanctioned result-changing event**
  (Phase 8 is the program's last change point).
- **(b) Cause = a legitimate reference-engine semantic difference** that iTrader handles
  correctly → document it and **KEEP iTrader's numbers** (default disposition: iTrader is
  correct unless the trace proves a defect).

Record all dispositions back into `CROSS-VALIDATION.md` under a "Root-Cause Dispositions"
section.

**This plan is CONDITIONAL.** If 08-07 reports near-zero divergence with no genuine iTrader
bug, this resolves to "all divergences documented as legitimate; no re-freeze" — which is
still a required, verifiable disposition step. The bug-fix path is a no-op in that case.

Purpose: Close out the cross-validation evidence with a defensible per-divergence verdict so
the final oracle (frozen in 08-09) is provably trustworthy. (M5-10, D-05)
Output: Updated `tests/golden/CROSS-VALIDATION.md` with dispositions; conditionally a named
`REFREEZE-M5C-<bug>.md` note + re-frozen golden artifacts + a code fix.
</objective>
