<objective>
Replace the per-bar re-summation in `PositionManager.get_total_realized_pnl` (the ~13% W1 hotspot
#3, PERF-02) with a running `Decimal` accumulator owned by `PositionManager` (D-01), fed from the
existing close funnel in `Portfolio` (D-02), proven byte-identical by a three-layer correctness lock:
a written single-funnel invariant audit (D-02), the byte-exact SMA_MACD oracle + determinism
double-run, and a dedicated equivalence regression test (D-03).

This is a **behavior-preserving** change — *less re-summation*, never a float swap. The accumulator
seeds `Decimal('0.00')` and never mid-sum quantizes, so the per-bar value is byte-identical to
today's full-precision Decimal sum (D-05). Closed-position retention is unchanged (D-07);
unrealized / market-value loops are explicitly out of scope (D-06).

Purpose: eliminate the O(positions)-growing-to-O(n²) per-bar realised-PnL re-sum on the equity /
metrics hot path while changing no numbers.
Output: a loop-free `get_total_realized_pnl`, the accumulator + apply method, both settle arms wired,
the written invariant audit, and the equivalence test — with gate (a) green.
</objective>
