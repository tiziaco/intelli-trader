### Three defects found during execution, beyond the audit

1. **`calculate_signals_for` — a THIRD colliding symbol.** A test-harness helper at
   `test_strategy_add_warmup.py:186,221` and `test_strategy_external_add_lifecycle.py:270,336`.
   A bare `s/calculate_signals/on_bar/` would have silently renamed it to `on_bar_for`, breaking
   two integration tests. The audit warned about the singular collision but not this one.
   True occurrence count is **90 word-boundary matches**, not the 94 a substring grep reports —
   the 4-occurrence delta is exactly this helper.

2. **The plan's own acceptance criterion `grep -c "def on_bar" … returns 1` is wrong** — it
   returns **2**, because `def on_bars_loaded` (line 605) contains `def on_bar` as a substring.
   The same substring-collision family as the above.

3. **The blanket doc rename degenerated five lines in the design spec** into statements like
   "Rename `on_bar` → `on_bar`". This is an inherent consequence of OWNER_DECISION_B (rename the
   spec) colliding with the completion gate (zero old-name occurrences in `docs/`). Repaired by
   rephrasing each to name the entry point descriptively rather than by its old literal — e.g.
   "Rename the per-bar entry point — then the sole imperative-verb name on the BAR route — to
   `on_bar`". The document remains true and the gate stays clean.

## Singular-symbol safety proof

Two distinct protections, and one deliberate removal that must not be mistaken for a failure.

**1. The rename never touched the singular.** Immediately after the word-boundary perl pass and
before any hand edits, all four singular sites were intact:

```

ARCHITECTURE.md:253: `calculate_signal` → `SignalEvent`, invoked by `StrategiesHandler.on_bar` per BAR.
CLAUDE.md:434:       Subclass implements `calculate_signal(...)`

```

Both shared-line hazards behaved correctly: `ARCHITECTURE.md:253` (both symbols on ONE line) and
`CLAUDE.md:460/461` (adjacent lines) each kept the singular and changed only the plural.

**2. `calculate_signals_for` survived byte-identical.** `git diff -- tests/` contains **zero**
`+`/`-` lines matching `calculate_signals_for`; all 4 occurrences remain.

**3. The singular was then deliberately removed — correctly.** A tracked-file grep now reports
**0** occurrences of `calculate_signal`. This is not rename leakage: `calculate_signal` **does not
exist anywhere in the source** (`git ls-files 'itrader/*' | xargs grep 'calculate_signal'` → none;
`base.py` defines `update` / `evaluate` / `generate_signal`). All four sites were false claims
sitting on lines this rename touched, corrected per the brief's bonus-defect instruction.

   `itrader/trading_system/trading_interface.py` and mis-described the ingress as constructing
   `OrderEvent`s. Replaced with `LiveTradingSystem.add_event` and its D-10 fail-closed semantics,
   resolving the contradiction with `CLAUDE.md:78`.

   plane; `ManagedStrategies` and `StrategyLifecycleManager` rows added.

   reproduces June-2026 dispatch code. Renamed for navigability **plus** a note that it is not a

5. **Stale line citation dropped, not guessed.** `stateful-indicator-design.md:241` cited
