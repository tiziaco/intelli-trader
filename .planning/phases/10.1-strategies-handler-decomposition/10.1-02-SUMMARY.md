# Phase 10.1 Plan 02: Extract the Roster into ManagedStrategies — Summary

DECOMP-01's roster half is done: `ManagedStrategies` is the single owner of `strategies`,
`min_timeframe`, `_pending_removals`, and the two SHORT-01/D-07 registration flags. The handler
holds none of that state and preserves its public surface by delegation, with **zero** edits to any
pre-existing test file. `calculate_signals` and `is_warm` are byte-identical to their pre-change
form and the oracle is byte-exact.

## What was built

**Task 1 — `ManagedStrategies`** (`itrader/strategy_handler/managed_strategies.py`, TABS, commits
`20f75adb` RED / `cc9414d0` GREEN). A pure state owner in the `BracketBook` (D-05) mould: no queue,
no handler back-reference, none of the three live deps. Four method bodies moved verbatim with
their decision-tagged docstrings, de-underscored only where a handler-private became public
surface. Three new shared members (`by_name`, `remove`, and the pending-removal accessor trio) have
no single donor body. 19 isolation tests construct the collaborator directly — no handler, no
queue, no store — which is the genuine test-surface win of the extraction.

The logger is injected, deviating from `BracketBook`'s logger-free shape, so `add_strategy`'s
closing `info` log stays *inside* the moved body. Left on a handler delegator it would be silently
dropped in 10.1-03 when `_add_strategy_verb`'s call target is repointed — a behaviour change
disguised as code motion. `update_config` was deliberately **not** moved (no roster mutation;
`ConfigurationError` contract, not `ValueError`).

**Task 2 — handler wiring** (`strategies_handler.py`, TABS, commit `1d02e937`). Five state
assignments replaced by one unconditional `self._managed = ManagedStrategies(...)`; the four
methods became 1-line delegations with their docstrings preserved on the handler side; the three
identical `{strategy.name: strategy ...}` comprehensions became `self._managed.by_name()`; the
roster and pending-set mutations became `remove` / `mark_pending` / `discard_pending` / `is_pending`.

## Plan deviations (audit-driven)

All five pre-dispatch audit corrections were confirmed against the live code and applied.

1. **[CRITICAL_correction_1 — confirmed, applied]** Every line number in the plan is stale by
   roughly +20 after Wave 1. Measured actuals matched the audit's table exactly
   (`_pending_removals` 182, flags 184/185, `min_timeframe` 190, `strategies` 192). Every construct
   was located by content. No construct was missing, so no halt condition fired.

2. **[CRITICAL_correction_2 — confirmed, applied as instructed]** The plan is silent on
   `_allow_short_selling` / `_enable_margin`. Confirmed 11 tests across 9 files write them on the
   handler after construction and then register a non-`LONG_ONLY` strategy; there is no public
   `allow_short_selling` attribute, so the dependency on the private is genuine.

   **Resolution: read/WRITE delegating properties (getter + setter) forwarding to the single
   `ManagedStrategies` instance.** One source of truth, zero test edits, no shadow attribute. I did
   *not* take the shadow-attribute route the audit flagged as dangerous: these two flags are a
   capability gate, and `direction_admissible` is the shared predicate behind both `add` and
   `reconfigure(direction=...)` precisely so the two cannot drift (T-10-55). A handler-side copy
   reading `True` while the gate consulted a stale `False` would be a correctness defect, not a
   plumbing detail. Verified by running the 9 files explicitly: 11 passed.

3. **[correction 3 — prescription followed; stated reason confirmed false]** The `strategies`
   property returns the identical list object, never a copy — as the plan instructs. I verified the
   plan's *reason* is wrong: `calculate_signals` does a fresh `self.strategies` attribute lookup per
   call, so a rebind would be picked up next tick and the oracle is indifferent. The load-bearing
   constraint is the **21 in-place mutation sites** in `test_strategies_live_membership.py` (12) and
   `test_strategies_handler_remediation.py` (9), which a copy would silently neuter. The code
   comment I wrote cites the real reason (the 21 test sites), not the false one.

4. **[correction 4 — confirmed, resolved; see the Wave-3 handoff below]** There are 9
   `_pending_removals` code sites, not the 4 the plan lists. All 9 are now resolved.

5. **[correction 5 — not treated as a gate]** The must_haves' "185 test field-references (113
   handler-scoped)" figure is unreconcilable (no stated methodology). I used the green full suite as
   the real check, as instructed. The plan's stated `2533 passed` baseline is also stale — Wave 1
   moved it to 2544.

### Other places the code disagreed with the plan

- **Task 1's live-dep acceptance grep is docstring-blind — reporting this rather than a hollow
  green.** `grep -vn '^\s*#' managed_strategies.py | grep -cE 'registry_store|strategy_catalog|...'`
  is specified to return `0`; it returns **`3`**. The grep strips `#` comments but not docstrings,
  so the module docstring's own *statement of the negative invariant* ("none of the live deps
  (`registry_store`, `strategy_catalog`, `portfolio_read_model`) ever reach it") trips its own gate.
  The three hits are lines 39–41, all prose. I substituted an AST scan (walk for `Name` / `Attribute`
  / `Import` nodes matching the five forbidden identifiers) which returns **NONE** — the invariant
  genuinely holds. The gate as written cannot pass for any module that documents the invariant it
  enforces.

- **The audit's claim that "every file in `tests/unit/strategy/` is 4-space" is an overstatement.**
  The directory is mixed: 21 of 27 files are 4-space, but `test_causal_guard.py`,
  `test_indicators.py`, `test_mark_unwarm.py`, `test_indicator_reset.py`, `test_primitives.py` are
  TABS and `test_indicator_convergence.py` is mixed. The 4-space choice is still correct — it is the
  majority and it matches the plan's named analog `test_strategies_handler_registration.py` — so no
  behaviour changed, but the stated justification was wrong.

- **The logger bind did have to move**, as the plan anticipated. It is now immediately after
  `self.feed`, at the top of `__init__`, with a comment recording why. `self.logger.info('Strategies
  Handler initialized')` stays at the end of `__init__`, so log ordering relative to construction is
  unchanged.

- **`min_timeframe` read-only is safe — verified, not assumed.** I grepped `itrader/`, `tests/`, and
  `scripts/` for assignments to all three moved containers. The only hit outside the two files in
  scope is `screeners_handler.py:113`, which is `ScreenersHandler`'s own unrelated `min_timeframe`.
  All other apparent hits were `==` comparisons. No surviving write on `StrategiesHandler`, so no
  setter is needed for the three read-only accessors.

- **`get_strategies_universe`'s pending-membership read** did not need a call-site rewrite: the
  whole loop moved into `get_universe`, so the read is now internal to the collaborator.
