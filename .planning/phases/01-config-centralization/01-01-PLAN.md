<objective>
Aggregate the cardinality-1 system-wide singletons into `SystemConfig` with an eager-vs-lazy
import-safety split, normalize the `extra` policy, and run the conservative dead-config audit
(CFG-01, CFG-02, CFG-04).

Per D-02 (cardinality rule) `SystemConfig` aggregates only `performance`, `monitoring`, an eager
`runtime` (Settings), and a lazy `sql` accessor. Per D-03 `order` is kept OUT (reclassified
cardinality-N, lives with `OrderHandler` via `OrderConfig.default()`) and per D-04 no per-instance
template fields are added. Per D-05/D-06 `sql` resolves `SqlSettings` on first access only via a
`functools.cached_property` (never at import), and per D-07 `runtime` stays eager. Per D-09 flip
`SystemConfig` from `extra="ignore"` to `extra="forbid"` (the domain YAML is orphaned/dead — nothing
loads it). Per D-12/D-13 the audit removes only provably-unreferenced config and verifies
`__pycache__` hygiene. D-01 keeps the model mutable (not frozen) in P1.

Purpose: give the engine ONE import-safe config singleton whose Postgres arm is never built at
import — the inertness lever the whole milestone leans on.
Output: modified `itrader/config/system.py`, a new unit test, an extended inertness probe, and the
orphaned domain-override cleanup.
</objective>
