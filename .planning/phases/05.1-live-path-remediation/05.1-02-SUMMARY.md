## Decisions Made

- **A3 injection point (`_initialize_live_session` wrap):** the OKX `VenueReconciler.reconcile()` halt fires during `start()` before the processing thread spawns; on the offline `'paper'` venue that branch is skipped, so wrapping `_initialize_live_session` (the last `start()` step before the reconcile branch) reproduces the exact ordering — the halt lands, then the loop's RUNNING stamp clobbers it. No OKX network/credentials.
- **A2 observable-behavior assertion + convention drain:** per the register rule ("encode observable behavior, not storage shape"), the test asserts the persisted `venue_order_id` reached via a shared `global_queue` + shared `InMemoryOrderStorage` and an `on_<event>` naming-convention drain. The recommended D-06 ORDER-ACK-event fix (exchange enqueues an ack → `OrderHandler.on_order_ack` stamps + persists) turns it GREEN without a rewrite. No fixture hand-stamping — that is exactly the V17-02 anti-pattern.
- **A5 real-Order accumulation:** a real `Order` (qty 1.0) driven through the real `on_fill` makes the double-count the production `_apply_executed` path. A strict-partial 0.2 increment never terminalizes, so the reservation-release / bracket-consume / portfolio-`get_position` branches are skipped and the collaborators stay unused Mocks.
