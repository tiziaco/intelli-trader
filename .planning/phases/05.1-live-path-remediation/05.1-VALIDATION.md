# Phase 05.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
> Spine: the CONF-A A1..A8 RED tests (TDD mode ON) pin each bug offline; CONF-B is the single human-gated online proof.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | pytest ^8.4.2 (`filterwarnings=["error"]`, `--strict-markers`, `--strict-config`) |
| **Config file** | `pyproject.toml` `[tool.pytest.ini_options]` |
| **Quick run command** | `poetry run pytest tests/unit -q` |
| **Full suite command** | `poetry run pytest tests` (NOT `make test` — see memory: env quirks disable logs / abort on missing .env) |
| **Static gate** | `poetry run mypy --strict itrader` |
| **Oracle gate (byte-exact, must stay dark)** | `poetry run pytest tests/integration/test_backtest_oracle.py` |
| **Estimated runtime** | ~unit fast; full suite includes slow/integration |

---

## Sampling Rate

- **After every task commit:** Run the task's targeted CONF-A RED test (e.g. `poetry run pytest tests/unit/<...>/test_<bug>.py -q`) — expect RED→GREEN across the fix.
- **After every plan wave:** Run `poetry run pytest tests` + `poetry run mypy --strict itrader` + the oracle gate.
- **Before `/gsd:verify-work`:** Full suite green, mypy strict clean, oracle byte-exact, all CONF-A A1..A8 GREEN.
- **Max feedback latency:** unit-test seconds; CONF-B is out-of-band (human-gated).
