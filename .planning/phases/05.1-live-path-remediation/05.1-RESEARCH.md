## Summary

Every `:NN` anchor in the ADR register and CONTEXT was captured against branch
`v1.7/phase-5-sandbox-path` @ `cfaed3f1`. I re-read the current branch
(`v1.7/phase-5.1-live-path-remediation`, HEAD `a7648598`, one docs commit past `a4c440ee`) and
verified every anchor D-01..D-20 names. **Result: the overwhelming majority of anchors are stable —
Wave-1 files (`account/base.py`, `account/simulated.py`, `account/venue.py`, `portfolio.py`, the five
`"backtest"` storage sites, `core/enums/system.py`) match the register line-for-line.** No decision
references a symbol that no longer exists (zero GONE anchors).

**Two structural landmines** the planner must not trust blind:

1. **`venue_reconciler.py` is RENAMED-BY-PATH.** The register (canonical_refs + D-09) places it at
   `itrader/order_handler/reconcile/venue_reconciler.py`. It actually lives at
   **`itrader/portfolio_handler/reconcile/venue_reconciler.py`** (alongside `drift.py`).
   `order_handler/reconcile/` contains only `reconcile_manager.py` (the D-08 target, a *different*
   reconciler). These are two distinct classes — do not conflate them.
2. **`live_trading_system.py` grew (1344→1391 lines); two anchors shifted EARLIER.** `halt()` moved
   `:598-601 → :567` (status flip now `:601`); `_link_venue_account_to_portfolios` moved `:554 → :532`
   (though its `>1` fail-loud guard stayed at `:554`). Everything else in that file the register cited
   still matches: `_update_status:745`, RUNNING stamp `:993`, `add_event:1323`, bar-sink `:411`, order
   store `:274`, signal store `:285`, S9 loop backstop `:1052-1057`.

On the API side, the pinned **ccxt 4.5.56** source settles most of ARCH-3's four CONF-B questions at
the documentation level: OKX `fetch_my_trades` hits `/trade/fills-history` (**last-3-months** window,
**limit default 100 / max 100 per page**, ccxt auto-pagination via `params={'paginate': True}`);
`tradeId` (unified `id`) and `ordId` (unified `order`) are **always present**, but `clOrdId` can be an
**empty string** for venue-external/manual fills; and `fetch_positions`/`watch_positions` structurally
**cannot return spot holdings** (their `instType` enum is MARGIN/SWAP/FUTURES/OPTION only). The
`quote_currency="USDT"` default bug (D-03) is **confirmed live** — `VenueAccount(self._okx_connector)`
at `live_trading_system.py:400` passes no quote while the traded pair is `_OKX_STREAM_SYMBOL="BTC/USDC"`.

**Primary recommendation:** Plan against the re-anchor tables below, not the register's line numbers.
Treat the API contracts as doc-settled EXCEPT the four items flagged "CONF-B only" (actual `[]` payload,
snapshot/trade timing coherence, real pagination depth on the demo sub-account, and post-fill venue/order
id behavior) — those bound how far Wave-2 reconcile adoption can be trusted and must be confirmed by the
human-gated online run before Wave-2 detail is frozen.

### D-03 spot base-balance position truth — `fetch_balance` / `watch_balance`

- **`fetch_balance(params={})`** (`ccxt/okx.py:2842`): hits `privateGetAccountBalance` (trading account,
  default) or `privateGetAssetBalances` (`params.type='funding'`). Returns the ccxt-unified **balance

  `{currency: {free, used, total}}`. [CITED: docs.ccxt.com/?id=balance-structure] [VERIFIED: ccxt/okx.py:2842 source]
- **Spot base-position truth = `total[BASE]`** (e.g. `total["BTC"]`). `VenueAccount._extract_balance`
  (`venue.py:111`) today reads only `total[quote]`/`free[quote]`; **D-03 must add a base-currency read**

## Common Pitfalls

### Pitfall 1: Trusting register line numbers blind
**What goes wrong:** editing `halt()` at `:598` (empty there now) or grepping `order_handler/reconcile/venue_reconciler.py` (wrong package).
**How to avoid:** use the Re-Anchor Map above; re-grep the symbol before every edit (`halt()`→`:567`, `venue_reconciler`→`portfolio_handler/reconcile/`).
**Warning signs:** an edit lands on a docstring or an unrelated method.

### Pitfall 2: Indentation hazard (tabs vs spaces)
**What goes wrong:** handler modules (`portfolio_handler.py`, `okx.py`, `reconcile_manager.py`, `order_handler/`) use **tabs**; `account/` (`base.py`/`simulated.py`/`venue.py`), `config/`, `core/`, `price_handler/feed/`, `events_handler/events/` use **4 spaces**. A mixed-indent diff breaks a tab file's import.
**How to avoid:** match the file you edit; never normalize. (Confirmed by direct read — `venue.py` is spaces, `okx.py` is tabs.)

### Pitfall 3: `since` silently disables `limit` on `fetch_my_trades`
**What goes wrong:** D-09 passes `since=oldest_created_at` + `limit=100` expecting a 100-row cap after `since`; ccxt drops `limit`, returns the 100 MOST RECENT, silently missing the window's older rows.
**How to avoid:** paginate (`params={'paginate': True}`) when `since` is set; never rely on `limit` alongside `since`.

### Pitfall 4: `clOrdId==""` / no unified `clientOrderId` (external fills)
**What goes wrong:** correlating a manual/external fill on `clientOrderId` fails (empty → buffered forever); the uncorrelated-fill buffer grows unbounded (D-16).
**How to avoid:** dedup on `{symbol}:{tradeId}` (always present); treat an uncorrelated fill as an external-trade SIGNAL (bound + alarm the buffer per D-16), not noise.

### Pitfall 5: Backtest oracle must stay byte-exact
**What goes wrong:** any change touching `transact_shares`/settle math (the hottest money path) or the `available` alias shifts the equity curve → oracle fails.
**How to avoid:** all fixes live-path-only / oracle-dark; the `SimulatedCashAccount.available` alias deletion is oracle-dark (no backtest caller) but re-run `test_backtest_oracle.py` (`134 / 46189.87730727451`, `check_exact=True`) per task.

### Pitfall 6: `connector.call()` from the connector loop thread self-deadlocks
**What goes wrong:** D-17's gap backfill on the loop thread blocks `.result()` → 30s full-loop stall → livelock.
**How to avoid:** loop-thread code awaits the client coroutine DIRECTLY; never bridge through `call()` (AUD-4c contract). The engine-thread warmup path is safe and stays unchanged.
