- **D-02:** [ARCH-1 / V17-14 + enforcement] Re-type `Portfolio.account` (`portfolio.py:104`) from the concretion to the ABC. Replace the two bare `cast(SimulatedMarginAccount, self.account)` narrowings (`portfolio.py:438`, `:834`) with an `isinstance` check raising a typed `ConfigurationError`/`StateError` BEFORE any mutation. `to_dict` reads (`portfolio.py:888-889`) become contract-covered by the ABC surface. - **D-03:** [ARCH-2 / V17-04 truth arm] Per-market-type venue-truth adapter inside `VenueAccount`: derivatives keep the positions channel (unchanged); **spot** derives per-symbol position truth from BASE-currency balance totals (`fetch_balance`/`watch_balance`), NOT `fetch_positions` (`_extract_positions` `venue.py:127-148` returns `[]` on spot). Quote currency ALWAYS from wiring (the traded pair's quote — USDC), never the `quote_currency="USDT"` default (`live_trading_system.py:400`, `venue.py:73`).
- **D-04:** [ARCH-2 / V17-04 guard arm] Session-start baseline guard sequenced AFTER reconciliation: compare base-asset balance against the engine's post-reconcile believed position within a per-instrument dust epsilon; any residual (unexplainable) mismatch → HALT before trading. Quote-side cash NOT asserted (deposits are legitimate funding). Unexplained divergence is NEVER auto-adopted. Includes the "just-applied engine fill vs not-yet-refreshed venue snapshot" spurious-halt band policy so a first spot fill does not halt on drift.
- **D-05:** [ARCH-4 Layer 1 / V17-03] Add `VALID_STATUS_TRANSITIONS` (`dict[SystemStatus, set[SystemStatus]]`) next to `SystemStatus` (`core/enums/system.py:14`), mirroring `VALID_ORDER_TRANSITIONS`. Enforce at ONE mutation point — `_update_status` (`live_trading_system.py:745`) with `halt()`'s direct write (`:598-601`) routed through the same enforcement. `HALTED` has NO legal exits in the table; only an explicit operator `reset_halt()` (deliberately outside the table) clears it. `start()` checks post-reconcile status and refuses to enter RUNNING from HALTED (fixes the unconditional RUNNING stamp at `:993`).

### Wave 2 — Restart real (V17-02 FINAL; ARCH-3 posture (i) + ARCH-4 Layer 2 PROVISIONAL — pending CONF-B)

- **D-06:** [V17-02] Persist the venue ack: `OkxExchange` emits a new frozen ORDER-ACK event (new `EventType` member + `_routes` entry) carrying `order_id → venue_order_id`; `OrderHandler` consumes it, stamps the mirror, `update_order()` persists. Delete fixture hand-stamping from restart tests and drive the real path. (Persisted `venue_order_id` is what makes ARCH-3 reconcile adoption able to match downtime fills.)
- **D-07:** [ARCH-3 posture (i) / V17-05 — PROVISIONAL] Wire the EXISTING cached portfolio SQL storage in live (same `SqlBackend` spine as order/signal stores) by threading the environment from the live composition root into the five hardcoded-`"backtest"` sites (`portfolio.py:96`, `position_manager.py:65`, `transaction_manager.py:47`, `metrics_manager.py:112`, `account/simulated.py:111`). Transactions persist with `venue_trade_id`; on restart, positions/cash AND the settled-trade dedup ledger rehydrate from Postgres BEFORE `reconcile()` runs. Role: **engine ledger = SOURCE of portfolio truth; venue = CHECK** — postures never mixed. RED tests encode observable behavior (restart remembers state), not storage shape.
- **D-08:** [V17-06 + V17-12] Per-order `venue_trade_id` dedup inside `ReconcileManager._apply_executed` (`reconcile_manager.py:154-216`) reading the rehydrated durable ledger before accumulating `filled_quantity`; key BOTH dedup layers (`portfolio_handler.py:838` + `venue_correlation.py:168`) as `f"{symbol}:{trade_id}"` to also close the instrument-scoped-tradeId collision (V17-12).
- **D-09:** [V17-10] Reconciler fill fetch (`venue_reconciler.py:131,449-452`) becomes per-symbol, paginated, `since = oldest active order's created_at`; log loudly when the venue window cannot cover the oldest active order (rehydrate-window bound, F/U-13).
- **D-10:** [ARCH-4 Layer 2 — PROVISIONAL] Durable halt record on the same `SqlBackend` spine (reason, timestamp): `halt()` persists it; `start()` refuses RUNNING while an unresolved halt record exists; `reset_halt()` resolves it. Purpose: a supervised auto-restart must never silently clear a breaker-class halt whose cause is not re-detectable at start. Storage home rides ARCH-3 posture (i).

- **D-11:** [V17-07] Add `except Exception → _escalate_connector_halt` catch-all to BOTH supervisors (fail-safe: unknown ⇒ halt, never silent task death); move `json.loads` inside the per-message guard (`okx_provider.py:274`); wrap `_stream_account`/`_stream_positions` (`venue.py:160-179`, bare `while True`) in the same supervisor; extract the duplicated ~100-line state machine into one shared helper; `connector.spawn` done-callback logs `task.exception()` and escalates.

- **D-14:** [V17-11] During pause/halt (`live_trading_system.py:736-743`), DEFER system-generated protective orders (bracket children, OCO/orphan cancels) into a replay queue drained on resume; ALWAYS let CANCEL commands through (they reduce risk); keep suppressing only fresh entry orders.
- **D-15:** [V17-13] Drop the local pending-overlay entry (`venue.py:252,307`) once the venue ack for that order arrives (venue `free` already excludes the hold), instead of waiting for terminal release — removes the buying-power double-count.
- **D-16:** [05-13 carry-overs] Move mark-seen AFTER successful `_emit_fill` (or validate payload inside `resolve` before consuming the dedup slot) — WR-02; route `release()` drains through `resolve` not raw `_emit_fill` — WR-03; `release_pending(clordid)` on submit failure — WR-01; reject `capacity < 1` — IN-01; bound + alarm the uncorrelated-fill buffer (`venue_correlation.py:178`); seed `adopt()`'s cumulative counter from rehydrated `filled_quantity`.
- **D-17:** [V17-15] Make the mid-session gap backfill loop-native — an async `fetch_ohlcv_backfill` awaited directly inside the candle coroutine (no blocking `connector.call()` bridge from loop-thread code), or hand the gap range to a dedicated backfill coroutine spawned on the loop. Kills the 30s full-connector stall → livelock (`live_bar_feed.py:171-173,298`, `connectors/okx.py:162-166`). The warmup path (engine thread) stays unchanged.
- **D-18:** [V17-16] Route external order creation through the admission pipeline (signal-form entry or a direct `AdmissionManager` call) so validation + sizing + reservation + mirror engage; wire `OkxExchange.on_order` preflight (`validate_order` `okx.py:734`, `validate_symbol` `okx.py:743`) as defense-in-depth mirroring `simulated.py:202/473`; optionally restrict `add_event` (`live_trading_system.py:1323`) to non-ORDER event types.

## Specific Ideas

- **TDD is ON (`--tdd`).** CONF-A A1..A8 are the RED gates; each fix plan follows RED → GREEN → REFACTOR against its pinned test. The bug register explicitly says "Write these as failing tests first — they become the regression suite."
- **Dependency order is load-bearing:** Wave 1 is the go/no-go — nothing downstream is observable until settlement works. V17-01/03/04 + ARCH-4 Layer 1 must land together (every halt is only real once HALTED latches). Wave 2 (V17-02/05/06/10) is mutually dependent (persisted `venue_order_id` powers reconcile adoption; durable ledger powers dedup). Wave 3 is additive resilience.
- **Provisional Wave 2:** ARCH-3 posture (i) and ARCH-4 Layer 2 are PROVISIONAL pending the CONF-B empirics. Plans for D-07/D-08/D-10 must encode observable behavior (restart remembers state; duplicate delivery idempotent), not storage shape, so a posture flip after CONF-B does not invalidate the RED tests. Wave 2's *detailed* implementation should not be treated as frozen until CONF-B runs — but the RED tests can be written now.
- **Sub-account discipline (ARCH-2):** one dedicated OKX demo sub-account per live system (1:1:1 sub-account↔VenueAccount↔portfolio); balance ≡ strategy position by construction. Demo creds are a no-real-money demo sub-account (authorized for the gated CONF-B run; still verify `sandbox=True` before any order).
- **Known accepted behavior:** until Wave 2's reconcile sync arm lands, a restart while holding a position halts at the baseline guard — correct fail-safe (state was lost; a human decides).
</specifics>

## Success Criteria (consequences_positive)

- A real OKX demo fill settles into the portfolio (position + cash + transaction) and is observable post-fill (CONF-B GREEN after Wave 1).
- Settlement path is contract-covered: mypy sees the ABC-vs-concretion conformance; the parametrized conformance test is a permanent gate — ABC drift can never ship again.
- A reconcile-time halt latches: `start()` after a halt stays HALTED; the engine never trades on state the reconciler declared untrustworthy.
- Spot venue truth is real: `VenueAccount.positions` reflects base-balance holdings; drift/orphan checks are no longer structurally blind; first spot fill does not spuriously halt.
- Restart remembers state: positions/cash + settled-trade dedup ledger rehydrate durably; duplicate delivery is idempotent; downtime fills are recoverable within the paginated window.
- Streams fail safe (unknown error ⇒ halt, never silent death); external order injection cannot bypass validation/mirror; the gap-backfill livelock is eliminated.
- SMA_MACD backtest oracle stays byte-exact; `mypy --strict` clean; full suite green via `poetry run pytest tests`.

## Risk Summary (consequences_negative)

- **Wave 1 mid-milestone risk:** touching `transact_shares`/settle paths and the account ABC is the hottest money path — mitigated by keeping fixes live-path-only and gating on the byte-exact oracle + conformance test.
- **ARCH-3 provisional-flip risk:** CONF-B evidence could flip the durable-ledger posture — mitigated by encoding observable behavior (not storage shape) in D-07/D-08/D-10 RED tests.
- **Scope/context risk:** 20 decisions may exceed a single planner's budget — accept a phase split over fidelity degradation.
- **Operational consequence (accepted):** until Wave 2 lands, a restart while holding a position halts at the baseline guard (correct fail-safe).
