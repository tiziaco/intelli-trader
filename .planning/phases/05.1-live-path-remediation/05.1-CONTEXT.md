<specifics>
## Specific Ideas

- **TDD is ON (`--tdd`).** CONF-A A1..A8 are the RED gates; each fix plan follows RED → GREEN → REFACTOR against its pinned test. The bug register explicitly says "Write these as failing tests first — they become the regression suite."
- **Dependency order is load-bearing:** Wave 1 is the go/no-go — nothing downstream is observable until settlement works. V17-01/03/04 + ARCH-4 Layer 1 must land together (every halt is only real once HALTED latches). Wave 2 (V17-02/05/06/10) is mutually dependent (persisted `venue_order_id` powers reconcile adoption; durable ledger powers dedup). Wave 3 is additive resilience.
- **Provisional Wave 2:** ARCH-3 posture (i) and ARCH-4 Layer 2 are PROVISIONAL pending the CONF-B empirics. Plans for D-07/D-08/D-10 must encode observable behavior (restart remembers state; duplicate delivery idempotent), not storage shape, so a posture flip after CONF-B does not invalidate the RED tests. Wave 2's *detailed* implementation should not be treated as frozen until CONF-B runs — but the RED tests can be written now.
- **Sub-account discipline (ARCH-2):** one dedicated OKX demo sub-account per live system (1:1:1 sub-account↔VenueAccount↔portfolio); balance ≡ strategy position by construction. Demo creds are a no-real-money demo sub-account (authorized for the gated CONF-B run; still verify `sandbox=True` before any order).
- **Known accepted behavior:** until Wave 2's reconcile sync arm lands, a restart while holding a position halts at the baseline guard — correct fail-safe (state was lost; a human decides).
</specifics>

## Success Criteria (consequences_positive)

- A real OKX demo fill settles into the portfolio (position + cash + transaction) and is observable post-fill (CONF-B GREEN after Wave 1).
- Settlement path is contract-covered: mypy sees the ABC-vs-concretion conformance; the parametrized conformance test is a permanent gate — ABC drift can never ship again.
- A reconcile-time halt latches: `start()` after a halt stays HALTED; the engine never trades on state the reconciler declared untrustworthy.
- Spot venue truth is real: `VenueAccount.positions` reflects base-balance holdings; drift/orphan checks are no longer structurally blind; first spot fill does not spuriously halt.
- Restart remembers state: positions/cash + settled-trade dedup ledger rehydrate durably; duplicate delivery is idempotent; downtime fills are recoverable within the paginated window.
- Streams fail safe (unknown error ⇒ halt, never silent death); external order injection cannot bypass validation/mirror; the gap-backfill livelock is eliminated.
- SMA_MACD backtest oracle stays byte-exact; `mypy --strict` clean; full suite green via `poetry run pytest tests`.

## Risk Summary (consequences_negative)

- **Wave 1 mid-milestone risk:** touching `transact_shares`/settle paths and the account ABC is the hottest money path — mitigated by keeping fixes live-path-only and gating on the byte-exact oracle + conformance test.
- **ARCH-3 provisional-flip risk:** CONF-B evidence could flip the durable-ledger posture — mitigated by encoding observable behavior (not storage shape) in D-07/D-08/D-10 RED tests.
- **Scope/context risk:** 20 decisions may exceed a single planner's budget — accept a phase split over fidelity degradation.
- **Operational consequence (accepted):** until Wave 2 lands, a restart while holding a position halts at the baseline guard (correct fail-safe).
