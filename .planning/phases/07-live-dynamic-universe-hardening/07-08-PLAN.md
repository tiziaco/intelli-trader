<objective>
Implement the PRIMARY WR-02 readiness gate (D-01) — the missing half of the readiness enforcement.
`AdmissionManager` already holds the injected `self._universe` seam (Phase 6, used today only for
`leaving_symbols()`) and is already wired live-only via
`OrderHandler.set_universe → OrderManager.set_universe → AdmissionManager.set_universe`
(live_trading_system.py:1247). This plan adds `_enforce_readiness_admission` to `process_signal` so that a
still-`PENDING` (or `FAILED`) symbol **cannot be traded via a bypassing external `OrderEvent`/`SignalEvent`**:
an unsized signal admitted by the D-10 allowlist for a non-READY symbol flows
`OrderHandler.on_signal → OrderManager → AdmissionManager.process_signal` and, with this gate, is REJECTED at
admission instead of being sized into a live order for an unwarmed symbol.

D-01 verbatim: "Admission is the primary gate consumer (a `pending` symbol cannot be traded via a bypassing
external `OrderEvent`); the strategy loop carries a cheap defensive check." Plan 07-04 Task 1 delivers the
SECONDARY strategy-loop defensive check; THIS plan delivers the PRIMARY admission gate.

Purpose: close the confirmed threat that an externally-injected SIGNAL for a still-PENDING symbol gets sized
into a live order for an unwarmed symbol — the strategy-loop check alone does not cover a direct injection.
Output: the primary admission-side readiness gate, oracle-inert by construction, with a direct-injection
rejection test.
</objective>
