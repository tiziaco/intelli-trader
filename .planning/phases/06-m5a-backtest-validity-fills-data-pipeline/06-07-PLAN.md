<objective>
Close UAT Gap 2 (CR-01, Critical): add a parent-filled gate to `MatchingEngine.on_bar`
so bracket children (SL/TP) cannot fill while their parent entry order still rests in
the book. Today, a BUY-LIMIT entry at 95 with a TP SELL-LIMIT at 110 sees the TP fill on
a rally bar while the entry never triggered — silently opening a reverse position from
flat and leaving the orphaned parent unprotected (its OCO children were consumed).

Purpose: bracket correctness for non-MARKET primaries, owner-routed as a Phase 6 blocker
before Phase 7 (M5b risk layer) builds on the matching engine.
Output: two-pass `on_bar` in matching_engine.py + regression tests locking the gate, with
the golden oracle proven byte-exact (the golden SMA_MACD path uses only market orders
with sl=0/tp=0, so this fix is behavior-inert on the oracle — D-21 inert workstream).
</objective>
