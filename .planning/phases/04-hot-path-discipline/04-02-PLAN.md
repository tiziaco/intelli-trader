<objective>
Realize the PERF-04 win: memoize `get_type_hints` per concrete strategy class so `Strategy.to_dict`
stops re-walking the MRO and re-resolving annotations on every signal snapshot (D-05). Add a
module-level `@functools.cache def _declared_hints(cls)` and route BOTH `to_dict` (base.py:356, hot —
per signal) and `_apply_params` (base.py:146, cold) through it for one consistent path. Lock
behavior-preservation with a dedicated equivalence test + a `to_dict` snapshot regression (D-07).

Purpose: hotspot #6 (~2% W1 / ~14% W2) is `to_dict` calling `get_type_hints(type(self))` on every
snapshot though the annotations are constant per class. Memoization collapses the per-signal cost to a
dict lookup. Byte-identical by construction (same function, cached -> same keys + order), so the
byte-exact SMA_MACD oracle is unaffected. Resolution is memoized, NOT removed — neither site uses the
resolved types, but removal would risk snapshot key-ordering in a byte-exact phase (deferred).

Output: the `_declared_hints` helper, both call sites routed through it, equivalence + snapshot tests.
</objective>
