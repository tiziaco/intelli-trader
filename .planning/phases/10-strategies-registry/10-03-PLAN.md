<objective>
Two independent live-path correctness edits that everything downstream depends on, both gated by the
backtest oracle:

1. **D-07** — wire the inert `is_active` flag into `calculate_signals` so `enable`/`disable` (Plan 06)
   has something to gate. One `continue` guard on the shared hot path.
2. **F-1** — make the feed ring-depth derivation timeframe-aware, and export the shared
   `required_base_depth` helper the `add` (Plan 07) and `reconfigure` (Plan 08) arms need.

Purpose (F-1): `cache_registration.py:225::derive_warmup_depth` is
`max((s.warmup for s in strategies), default=1)` with **no timeframe scaling**, while `base.py:391`
derives `warmup` from indicator `min_period` in **strategy-timeframe** bars. The `LiveBarFeed` ring
holds **base** bars and `window()` resamples them on read. So a strategy coarser than the base cadence
needs `warmup × multiple` base bars but gets `warmup`. A 4h strategy with `warmup=100` on a 1h base
feed resamples a 100-bar ring to 25 bars, never reaches 100, and **silently never trades, forever**.
This is invisible today only because SMA_MACD's timeframe equals the live base cadence — and P10's
rehydrate (D-01), runtime `add` (D-10), and timeframe reconfigure (D-15) each create a path straight
into it. `derive_warmup_depth`'s own docstring names itself "the NAMED, replaceable D-17 seam ... CF-10
later generalizes ONLY this function body" — this is exactly the change the seam was built for.

**F-1 remediation decision (RESEARCH OQ-2):** take option **1** (timeframe-aware depth) here, and
option **3** (loud-reject when the required depth exceeds an existing ring's capacity) in Plans 07/08
via the shared helper. Option **2** (resizing an already-created `deque`, whose `maxlen` is fixed at
creation) is a feed-lifecycle operation and is deferred to the existing
`strategy-timeframe-finer-than-base-resubscribe` todo — its natural home. Rationale: option 1 is small,
is the correct fix, and alone fixes the boot/rehydrate path (rings are created after
`register_strategy_warmup` runs); option 3 covers the runtime paths where the ring already exists and
cannot resize, and fails **loud** rather than silently dark.

Output: a correctly-sized live ring, a shared warmability boundary, and a wired `is_active` gate.

Decisions implemented: **D-07** (enable/disable = the `is_active` gate, keep warm). Also unblocks
**D-15** (F-1 gates its timeframe arm).
</objective>
