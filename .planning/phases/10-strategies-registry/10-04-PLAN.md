<objective>
Build the `strategy_handler/registry/` collaborator's two upstream halves: the **D-01 injected type
catalog** (the allowlist that resolves `strategy_type` -> class without `itrader` ever importing a
concrete strategy) and the **D-04/D-20 authoring-param codec** (instance -> `config_json` -> instance,
delegating policy fields to the Plan 01 core codec).

Purpose: this is the load-bearing half of D-01's type-vs-instance split. Strategy **types** are code
in the owner's private submodule repo; strategy **instances** are DATA and the store is their source
of truth. Rehydrate (Plan 05) and runtime `add` (Plan 07) both reduce to `catalog × row × codec ->
Strategy` — this plan builds `catalog × codec`.

Output: `itrader/strategy_handler/registry/{__init__,catalog,config_codec}.py` (TABS) + a shared test
catalog in `tests/support/` + round-trip coverage for every shipped strategy.

Decisions implemented: **D-01** (injected catalog, the store is the source of truth for instances),
**D-04** (flat authoring blob, `_DERIVED_FIELDS`, the three aliasing traps), **D-05** (placement in a
`strategy_handler/` collaborator, symmetric serialize/deserialize sharing the codec), **D-16** (pairs
serialize with no special case), **D-20** (stamp `config_version` now, build no migration framework).
</objective>
