<objective>
Build the D-03 tagged-union policy codec in `itrader/core/policy_codec.py` — a structured
serializer/deserializer for the six frozen sizing/SLTP policy value objects, separate from the
one-way `to_dict()` observability snapshot.

Purpose: `to_dict()` serializes policies as `repr()` (`"FractionOfCash(Decimal('0.95'))"`), which is
not reconstruction-safe. Every downstream P10 capability (D-01 rehydrate, D-04 `config_json`, D-09
runtime `add`) needs a policy blob that reconstructs **without string evaluation**. Per D-05 this
codec lives in `core/` — it serializes core value objects, depends on nothing else in `itrader`, and
obeys the money policy.

Output: `itrader/core/policy_codec.py` (4-SPACE) + `tests/unit/core/test_policy_codec.py`.

Decisions implemented: **D-03** (build the structured codec now, generic dataclass introspection +
injectable kind->class registry, Decimals as strings), **D-05** (codec placement in `core/`).
</objective>

    Module docstring cites **D-03** and **D-05** and states the two contracts: (a) this is the
    reconstruction-safe counterpart to `Strategy.to_dict()`'s one-way repr snapshot, and (b) it is a
    money boundary — Decimals cross JSON as strings.

    Public surface:
    - `PolicyRegistry` — type alias `dict[str, type]`, the kind->class map.
    - `default_policy_registry() -> PolicyRegistry` — build the map by iterating
      `typing.get_args(SizingPolicy)` and `typing.get_args(SLTPPolicy)` (imported from
      `itrader.core.sizing`), keyed on each class's `__name__`. Deriving from the unions is the
      point: hand-listing is how `PercentFromDecision` got omitted from CONTEXT's D-03 list. Accept
      an optional `overlay: PolicyRegistry | None = None` that merges OVER the derived default so the
      app can register private-repo IP policies without `itrader` importing them.
    - `encode_policy(policy: Any) -> dict[str, Any]` — emit `{"kind": type(policy).__name__, ...one
      key per dataclass field...}`.
    - `decode_policy(blob: Mapping[str, Any], registry: PolicyRegistry) -> Any` — look the class up
      by the `kind` tag in the registry (a plain dict lookup — the registry IS the allowlist), then
      construct it with coerced kwargs.
    - `PolicyCodecError(ValueError)` and `UnknownPolicyKindError(PolicyCodecError)`.

    Type resolution — the load-bearing detail:
    - Resolve field types with `typing.get_type_hints(cls, localns=...)`, NOT
      `dataclasses.fields()[i].type`. `PercentFromFill.trail_type` is the quoted forward reference
      `"TrailType | None"`, so `field.type` returns that raw string and is unusable.
    - `get_type_hints` on `PercentFromFill` raises `NameError` because `TrailType` is deliberately not
      importable at `core/sizing.py` module level. Resolve it by passing an explicit `localns`
      mapping `{"TrailType": <the enum>}`, obtained by a **function-local** import of
      `itrader.config` inside the hint-resolution helper — mirroring exactly what
      `sizing.py::__post_init__` already does at :264 and preserving the core->config direction (a
      module-level import here would invert it and break the inertness gate).
    - Memoize the per-class resolved hints with `functools.cache` keyed on the class object, copying
      the `base.py:130-133` `_declared_hints` idiom. The memo is keyed on the class object (not its
      name), so two distinct classes sharing a `__name__` stay correct.

    Coercion, per declared type, both directions:
    - `Decimal` -> encode as `str(value)`; decode via `Decimal(str(value))`. Refuse a non-finite
      Decimal on encode with `PolicyCodecError` (a NaN/Infinity token has no safe JSON round-trip).
      Never construct a Decimal from a binary float — that is the locked money defect.
    - `X | None` -> unwrap the union with `typing.get_args`, drop `NoneType`, dispatch on the single
      remaining arm; `None` passes through both ways.
    - An `Enum` arm -> encode `value.value`; decode `EnumCls(raw)`.
    - `int` / `str` / `bool` -> pass through.
    - An unhandled declared type -> raise `PolicyCodecError` naming the class and field. Fail loud;
      never fall through to a silent pass-through that would smuggle an uncoerced value into a
      frozen policy.

    Class resolution is a **dict lookup in the injected registry and nothing else**. Do not resolve a
    kind tag by dynamic module import and do not evaluate any part of the blob as source text — the
    `kind` tag arrives from an external `STRATEGY_COMMAND` payload, so the registry lookup IS the
    access-control allowlist. Reconstructing `to_dict`'s repr form would require string evaluation;
    that is the entire reason this codec exists. State this in the module docstring as prose, without
    naming the forbidden builtins, so the commit-time source gates below stay meaningful.

    `decode_policy` constructs the class normally, so each `__post_init__` re-validates on the way
    back for free — do not duplicate those validators in the codec (they would drift).

    `mypy --strict` applies to `core/` (no `ignore_errors` override) — annotate fully.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external `STRATEGY_COMMAND` payload -> `config_json` -> `decode_policy` | An operator/FastAPI-supplied policy blob crosses into value-object construction here. Untrusted by construction. |
| `config_json` row in SQL -> `decode_policy` on rehydrate | A stored blob (written by an earlier, possibly-different code version) is reconstructed at boot. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-10-01 | Elevation of Privilege | `decode_policy` kind resolution | critical | mitigate | Class resolution is a dict lookup in the injected `PolicyRegistry` and nothing else. No dynamic module import, no source-text evaluation of any blob field. Gated by the `grep -c 'eval('` == 0 and `grep -c 'import importlib'` == 0 acceptance criteria. |
| T-10-02 | Elevation of Privilege | repr-based reconstruction | critical | mitigate | The tagged-union codec IS the mitigation — `to_dict()`'s repr form would require string evaluation to reconstruct. `to_dict()` stays a one-way observability snapshot and is never a decode source. Gated by the same greps. |
| T-10-03 | Tampering | uncoerced field smuggling | high | mitigate | Coercion is driven by the resolved declared type; an unhandled declared type raises `PolicyCodecError` rather than passing the raw value through. `__post_init__` re-validates every decoded policy. |
| T-10-04 | Tampering | money precision corruption | high | mitigate | Decimal fields cross JSON as strings and re-enter via `Decimal(str)`; non-finite Decimals are refused on encode. Gated by `grep -c 'Decimal(float'` == 0 and the `-k decimal` test. |
| T-10-05 | Denial of Service | pathologically large blob | low | accept | `config_json` size is bounded by the SQL column and the operator surface; rate limiting belongs to the deferred FastAPI layer (LR-01), which owns the ingress. |
| T-10-06 | Information Disclosure | secrets in the encoded blob | low | accept | Policies are pure numeric value objects with no credential fields; `config_json` holds authoring params only. |

**Canon-referral breadcrumb:** SQL injection via `strategy_name`/`config_json` is canon — covered by
SEC-01 (parameterized SQLAlchemy Core, already honored throughout the store) and `/gsd-secure-phase`;
not minted as a bespoke prohibition here.
</threat_model>
