Implement STRAT-03: **atomic reconfiguration**. Trial-validate the full merged config on a throwaway
instance, persist, apply, re-warm â€” with the D-15 allowlist, the D-14 dark/warm boundary, the F-1
warmability gate, and D-17's pair refusal.

Purpose: this is the requirement's own wording â€” "quiesce -> apply -> re-warmup". RESEARCH sharpened
what is actually missing: `_apply_params` is **already** atomic (the WR-02 resolve-into-locals trial
phase commits at `base.py:295-299`), and the single-writer engine thread **already** provides the
quiesce (no in-flight signal exists between event cycles). The genuine tear is that `reconfigure`
calls `validate()` and `_run_init()` **after** that commit, so a cross-field validation failure leaves
a live, trading strategy mutated into a state its own validator rejects. The fix is a trial
construction: `cls(**merged)` runs the whole validate chain on a throwaway before the live instance is
touched.

Output: a reconfigure that either fully happens or fully does not, with the DB never diverging from the
live instance in the applied-but-unpersisted direction.

Decisions implemented: **D-12** (apply live, KEEP open positions), **D-13** (validate -> persist ->
apply -> re-warm; the persist-fail and apply-fail asymmetry; between-cycle application), **D-14**
(re-warm semantics; the grew/shrank/unchanged boundary), **D-15** (the mutability allowlist; timeframe
constrained-mutable, finer-than-base rejected), **D-17** (all pair reconfiguration refused). Consumes
**F-1**'s `required_base_depth` (Plan 03) and **P-4**'s merge semantics.
