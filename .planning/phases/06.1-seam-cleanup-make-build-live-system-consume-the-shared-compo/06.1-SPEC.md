# Phase 6.1: Seam Cleanup — Specification

**Created:** 2026-07-14
**Ambiguity score:** 0.09 (gate: ≤ 0.20)
**Requirements:** 4 locked

## Goal

`build_live_system` wires its handler graph by calling the shared `compose_engine` (made store/feed-agnostic) instead of hand-rolling a parallel copy — and the interim scaffolding around it (`LiveSystemComponents`, the duplicated `for_exchange` spec-builder, the backtest-graph-eager live barrel) is removed — with **zero** change to live behavior and the backtest oracle byte-exact at `134 / 46189.87730727451`.

## Background

Phase 6 delivered the live composition root (`build_live_system` → `LiveRunner` → pure-injection facade) as a **deliberately shallow, behavior-preserving interim step** whose code is explicitly marked "interim" and defers deeper work to Phase 7. Four seams did not converge on the intended design and are **not** on any later phase's agenda:

- **`build_live_system` (`live_trading_system.py:1438-1734`) re-implements `compose_engine` by hand.** Lines 1537-1572 rebuild the execution/strategies/order/event handlers almost verbatim — including re-inlining an `_estimate_commission` closure (1541-1545) that `compose.py` already promoted to the typed `FeeModelCommissionEstimator` (compose.py:54-78). `compose.py`'s own module docstring names live as the intended second consumer of `compose_engine` ("a future `build_live_system` (fast-follow) consume"); it was never wired. The blocker: `compose_engine` currently **hardcodes** `CsvPriceStore` + `BacktestBarFeed` (compose.py:161-165), so it is not yet mode-agnostic — live needs a `LiveBarFeed`.
- **`LiveSystemComponents` (`live_trading_system.py:103-134`)** is a 20-field `Any` bag built in exactly one place and unpacked field-by-field in exactly one place (`__init__:178-197`) — single-producer/single-consumer ceremony with no behavior, validation, or reuse. `_initialize_live_session` (926-937) additionally reconstructs an **interim `Engine`** from facade attributes with inert placeholder `clock`/`time_generator`.
- **`for_exchange` (`live_trading_system.py:255-283`)** duplicates `build_live_system`'s spec-building: the `SimpleNamespace` "spec" and the `{'okx':'okx','paper':'okx'}.get(...)` default-provider map appear in both (274-283 and 1630-1632).
- **The barrel (`trading_system/__init__.py:5`)** eagerly imports the live module, putting it on the backtest import graph. That is the root cause of the pervasive lazy-imports-inside-methods across `live_trading_system.py` (needed to keep `test_okx_inertness.py` green) — including imports that are pure (non-ccxt/non-SQL) and need not be lazy at all.

This phase lands **before Phase 7** so P7's `SafetyController`/`ReconciliationCoordinator`/`StreamRecoveryHandler` extraction builds on the clean seam rather than the hand-rolled copy.

## Requirements

1. **SEAM-01 — Live consumes `compose_engine` (oracle-gated centerpiece)**: `build_live_system` obtains its handler graph from `compose_engine` via a store/feed injection seam; the hand-rolled handler construction and the re-inlined `_estimate_commission` closure are deleted.
   - Current: `build_live_system:1537-1572` hand-rolls execution/strategies/order/event handlers + inlines `_estimate_commission`; `compose_engine` hardcodes `CsvPriceStore`+`BacktestBarFeed` (compose.py:161-165) so it cannot be reused as-is.
   - Target: `compose_engine` accepts an injected store + feed (backtest passes today's `CsvPriceStore`+`BacktestBarFeed`; live passes `CsvPriceStore`+`LiveBarFeed`); `build_live_system` calls it and reuses `FeeModelCommissionEstimator`; no parallel handler-wiring copy remains in `live_trading_system.py`.
   - Acceptance: `grep` shows no second construction of `ExecutionHandler`/`OrderHandler`/`StrategiesHandler`/`EventHandler` inside `live_trading_system.py` (they are built only in `compose.py`); the SMA_MACD oracle is byte-exact `134 / 46189.87730727451` (`check_exact=True`) via `tests/integration/test_backtest_oracle.py`, and the determinism double-run is identical.

2. **SEAM-02 — Collapse `LiveSystemComponents` + interim `Engine`**: the 20-field `Any` bag is replaced by the compose `Engine` + a small venue-extras holder, and the interim `Engine` reconstruction in `_initialize_live_session` is removed.
   - Current: `LiveSystemComponents` (103-134) is a single-producer/single-consumer 20-field bag; `_initialize_live_session:926-937` rebuilds an interim `Engine` with placeholder `clock`/`time_generator`.
   - Target: the facade is injected with the real compose `Engine` plus a focused venue-extras object (connector/exchange/data-provider/account/bundle/lifecycle + the SQL/halt-store handles); `_initialize_live_session` reads the engine off the injected state, constructing no throwaway `Engine`.
   - Acceptance: `LiveSystemComponents` no longer exists (grep-clean) and no `Engine(...)` is constructed inside `live_trading_system.py`; live construction/lifecycle tests pass unchanged.

3. **SEAM-03 — One shared spec-builder**: `for_exchange` and `build_live_system` derive the live spec (venue + default data-provider) through a single helper.
   - Current: the `SimpleNamespace` spec + `{'okx':'okx','paper':'okx'}.get(...)` default map are written twice (274-283 and 1630-1632).
   - Target: exactly one function/typed builder produces the live spec; both entry points call it.
   - Acceptance: the `{'okx':'okx','paper':'okx'}` literal and the spec-construction appear in exactly ONE location (grep count == 1); `for_exchange('okx'|'paper'|other)` and direct `build_live_system(spec)` produce identical specs (unit test asserts equality across the venue set).

4. **SEAM-04 — De-lazy the barrel + hoist pure imports**: `trading_system/__init__.py` no longer eagerly pulls the live module onto the backtest import graph, and the pure (non-ccxt/non-SQL) imports in `live_trading_system.py` move to module top.
   - Current: `trading_system/__init__.py:5` eagerly imports `LiveTradingSystem`/`build_live_system`; `live_trading_system.py` lazy-imports even pure modules (`Engine`, `BacktestClock`, `TimeGenerator`, `SessionInitializer`, `UniverseHandlerConfig`) to protect inertness.
   - Target: the barrel exposes the live surface without importing it on the backtest path (lazy `__getattr__` re-export or equivalent); genuinely-heavy imports (ccxt.pro/SQL/venue) stay lazy inside `build_live_system`; the pure imports are hoisted to module top.
   - Acceptance: `tests/integration/test_okx_inertness.py` stays green (backtest import path pulls no `ccxt.pro`/SQL); importing `build_backtest_system` from the barrel does not import `itrader.trading_system.live_trading_system` (asserted); the ~45 existing `for_exchange`/`build_live_system`/`LiveTradingSystem` call sites still resolve (`mypy --strict` clean, suite green).

## Boundaries

**In scope:**
- Store/feed injection seam on `compose_engine` + backtest call-site pass-through (SEAM-01)
- `build_live_system` rewired to consume `compose_engine` + `FeeModelCommissionEstimator` (SEAM-01)
- Removal of `LiveSystemComponents` and the interim `Engine` reconstruction; venue-extras holder (SEAM-02)
- Single shared live-spec builder for `for_exchange` + `build_live_system` (SEAM-03)
- Barrel de-lazying + pure-import hoist in `live_trading_system.py` (SEAM-04)

**Out of scope:**
- Loop-lifecycle callback web / `_on_loop_start` / `_on_loop_error` / the 9 LiveRunner callbacks — owned by **P7** (SafetyController status latch + dispatch gate + CONTROL-event conversion of resume/halt flags), **P8** (injected ErrorPolicy), **P9** (SystemStore stats read-model). Touching it now = code those phases rewrite.
- Moving `_initialize_live_session` to construction time — **blocked** by the test contract (add-strategy-after-construction + monkeypatch `_initialize_live_session`); enabled later by **P10** strategy rehydrate. This phase keeps session init deferred to `start()`.
- Folding `UniverseHandler`'s 5 non-freeze setters into its constructor — **P7** repoints `set_freeze_gate`; low-value churn now.
- Relocating `wire_universe` / membership into `UniverseHandler` — **working as intended**: `wire_universe` is shared by BOTH runners and `UniverseHandler` is live-only (RUN-06); moving it in would force backtest to duplicate the oracle-critical ordering.
- Any change to live runtime behavior (order flow, safety, reconcile, stream) — this is a **behavior-preserving** refactor.

## Constraints

- **Oracle byte-exact (per-PLAN gate):** SEAM-01 modifies the shared `compose_engine`, which the backtest path uses — the SMA_MACD run must stay `134 / 46189.87730727451` (`check_exact=True`) with the determinism double-run identical. This is the milestone-wide gate applied at the plan level.
- **OKX import-inertness (hard gate):** `tests/integration/test_okx_inertness.py` stays green after the barrel change (SEAM-04) — the backtest import path pulls no `ccxt.pro`/SQL.
- **Held throughout:** Decimal money end-to-end; single UUIDv7; `mypy --strict` clean on new/changed code; `filterwarnings=["error"]` green.
- **Indentation is split per file — measure bytes, never normalize:** `compose.py` and `universe_wiring.py` are TABS; `live_trading_system.py` and `live_runner.py`/`session_initializer.py` are 4-SPACE; `trading_system/__init__.py` is spaces. A mixed-indentation diff breaks a tab file.
- No new third-party dependency; no `poetry` change.
