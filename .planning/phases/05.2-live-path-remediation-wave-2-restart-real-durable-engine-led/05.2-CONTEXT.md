## Implementation Decisions

Carved from the master register (`v17_arch_decisions.md` ARCH-3/ARCH-4 Layer 2, `v17_bugs.md` Wave 2). Plans MUST cite each `D-NN` in `must_haves`/`truths`/`objective`.

- **D-06:** [V17-02 — FINAL] Persist the venue ack: `OkxExchange` emits a new frozen ORDER-ACK event (new `EventType` member + `_routes` entry) carrying `order_id → venue_order_id`; `OrderHandler` consumes it, stamps the mirror, `update_order()` persists. Delete fixture hand-stamping from restart tests and drive the real path. (Persisted `venue_order_id` is what makes ARCH-3 reconcile adoption able to match downtime fills.) Analog: an existing `events_handler/events/*.py` frozen event + `core/enums/event.py::EventType` + `full_event_handler.py::_routes` (3-part registration or `_dispatch` raises `NotImplementedError`).
- **D-07:** [ARCH-3 posture (i) / V17-05 — PROVISIONAL pending CONF-B] Wire the EXISTING cached portfolio SQL storage in live (same `SqlBackend` spine as order/signal stores, analog `live_trading_system.py` order/signal store wiring) by threading the environment from the live composition root into the five hardcoded-`"backtest"` sites (`portfolio.py`, `position_manager.py`, `transaction_manager.py`, `metrics_manager.py`, `account/simulated.py`). Transactions persist with `venue_trade_id`; on restart, positions/cash AND the settled-trade dedup ledger rehydrate from Postgres BEFORE `reconcile()` runs. Role: **engine ledger = SOURCE of portfolio truth; venue = CHECK** — postures never mixed. RED tests encode observable behavior (restart remembers state), not storage shape.
- **D-08:** [V17-06 + V17-12] Per-order `venue_trade_id` dedup inside `ReconcileManager._apply_executed` (`order_handler/reconcile/reconcile_manager.py`) reading the rehydrated durable ledger before accumulating `filled_quantity`; key BOTH dedup layers (`portfolio_handler.py` + `venue_correlation.py`) as `f"{symbol}:{trade_id}"` to also close the instrument-scoped-tradeId collision (V17-12).
- **D-09:** [V17-10] Reconciler fill fetch (`portfolio_handler/reconcile/venue_reconciler.py` — NOTE `portfolio_handler/`, not `order_handler/`) becomes per-symbol, paginated (`params={'paginate':True}`), `since = oldest active order's created_at`; log loudly when the venue window cannot cover the oldest active order (rehydrate-window bound, F/U-13). CCXT landmine (per 05.1-RESEARCH.md): `since` disables `limit`; `fetch_my_trades` → `/trade/fills-history` (last-3-months window, 100/page).
- **D-10:** [ARCH-4 Layer 2 — PROVISIONAL] Durable halt record on the same `SqlBackend` spine (reason, timestamp), with a new chained Alembic head: `halt()` persists it; `start()` refuses RUNNING while an unresolved halt record exists; `reset_halt()` resolves it. Purpose: a supervised auto-restart must never silently clear a breaker-class halt whose cause is not re-detectable at start. Storage home rides ARCH-3 posture (i). Builds on the 05.1 D-05 in-process latch.
