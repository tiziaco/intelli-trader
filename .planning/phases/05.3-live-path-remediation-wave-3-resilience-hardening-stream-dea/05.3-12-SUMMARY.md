# Phase 05.3 Plan 12: Resume-Gate (D-28/WR-03) + Backfill Fail-Loud (D-29/WR-05) Summary

**Resume of NEW order submission is now gated on EVERY wired venue stream arm being healthy (no more trading blind to fills after a single arm's reconnect), and the loop-native gap backfill fails loud + refuses a nested backfill on an under-returning venue page instead of recursing.**

## Performance

- **Duration:** ~12 min
- **Started:** 2026-07-06T07:35Z
- **Completed:** 2026-07-06T07:40Z
- **Tasks:** 2 (both TDD: RED → GREEN)
- **Files modified:** 6 (4 source, 2 test)

## Accomplishments

- **D-28 (WR-03):** `OkxExchange` and `OkxDataProvider` each expose a public `is_streaming_healthy()` over their own already-tracked `_streams_down` set; `LiveTradingSystem._all_venue_streams_healthy()` reads a compound predicate and `_maybe_resume_after_reconnect` resumes submission only when every wired arm reports up. A candle-stream reconnect while the fill stream is still down now keeps submission paused (closing the trade-blind-to-fills window). No engine-side aggregation/namespacing added; pause-on-any-down semantics unchanged.
- **D-29 (WR-05):** `_replay_and_deliver` now clamps the replayed interior at BOTH ends (new symmetric `< since_ms` low clamp mirroring the existing `> last_ms` break) and raises a typed `MalformedDataError` when the first in-range replayed bar is not exactly `first_missing` — an under-returning page escalates to a connector halt instead of re-entering the gap branch. A `_replaying_backfill` re-entrancy guard (try/finally) makes `update()`'s gap branch fail loud rather than spawn a nested backfill — the shape-independent structural stop.
- **D-27:** both fixes ship production-premise tests — a resume-gate integration test driving `_maybe_resume_after_reconnect` with one arm down (not the predicate in isolation), and a backfill-contiguity unit feeding an under-returning page (not the happy-path replay).

## Decisions Made

- Followed the plan's interface prescription exactly: per-arm predicate + engine compound read, no engine-side aggregate stream set, no namespaced stream names (`LiveTradingSystem` footprint kept to one helper + one guard for the slated post-Phase-6 refactor).
- Reused `MalformedDataError` (`itrader/core/exceptions/data.py`) for the fail-loud case; message is a fixed literal plus symbol/tf/expected-vs-actual-ts coordinates only — no `str(payload)`, preserving the secret-scrub discipline (T-05.3-29b accepted).
