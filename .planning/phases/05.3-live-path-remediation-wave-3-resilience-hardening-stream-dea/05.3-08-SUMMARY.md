# Phase 05.3 Plan 08: Halt-Lifecycle Fixes (D-20 early refusal + D-21 off-loop write) Summary

Two halt-lifecycle fixes on the live path: a durably-HALTED engine now refuses at the TOP of
`start()` with zero venue I/O (D-20/WR-01), and a connector-fatal escalation no longer runs the
blocking durable `record_halt` SQL write on the connector asyncio loop thread — it hands off a
thread-safe flag and the engine thread does the blocking write (D-21/WR-02).

## What was built

### Task 1 — D-20 / WR-01: early durable-halt refusal
The durable-halt refusal gate (`_halt_record_store.has_unresolved()` → re-latch HALTED via
`_update_status` → `return False`) was moved from its late position (after the full OKX handshake +
`VenueAccount.snapshot()` + `VenueReconciler.reconcile()`) to the TOP of `start()`, right after
`_update_status(STARTING)` and before `_initialize_live_session()` / any venue I/O. A durably-HALTED
engine now stays inert: no `connect`, no `snapshot`, no state-mutating reconcile, no second durable
record. The `not self._is_halted()` conjunct is kept, and the separate D-05 in-process `_is_halted()`
check stays at its original position (after reconcile) to latch a halt raised DURING this run's
reconcile/baseline guard — no double-refuse.

### Task 2 — D-21 / WR-02: off-loop durable halt write (flag pattern)
The OKX stream arms (`_okx_exchange`, `_okx_data_provider`) now wire `_request_connector_halt` as
their halt signal instead of `halt()` directly. `_request_connector_halt` (fired from the connector
asyncio loop thread on a fatal error / exhausted ceiling / catch-all) only flips a thread-safe
`_pending_connector_halt` flag. The engine thread drains it via `_maybe_halt_after_connector_fatal`
(called in both the event-received and queue-idle paths of `_event_processing_loop`, mirroring
`_maybe_resume_after_reconnect`) and runs the blocking `halt()` there — durable `record_halt` write +
status flip + CRITICAL alert all off the loop. `halt()`'s winner-only/idempotent guard keeps the
single-write + D-10 latch ordering; the V7 secret scrub (`type(exc).__name__` + fixed
`'connector-fatal'`, never `str(exc)`) is preserved. The engine-thread halt signals (portfolio drift
compare at composition, VenueReconciler in `start()`) keep calling `self.halt` directly — they never
run on the connector loop.

**D-21 mechanism chosen: flag pattern** (over `run_in_executor` / queued-event) — smallest change,
no new EventType, and it reuses the established pause/resume flag-handoff pattern.
