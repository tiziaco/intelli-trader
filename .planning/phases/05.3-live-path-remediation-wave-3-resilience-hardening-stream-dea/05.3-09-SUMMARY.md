# Phase 05.3 Plan 09: Ungate rehydrate() → activate SimulatedAccount restart restore (D-23) Summary

Ungated `PortfolioHandler.rehydrate()` from the `if self.exchange == 'okx'` arm in
`LiveTradingSystem.start()` so a durable paper/simulated engine restarts with its persisted
cash + realized-PnL instead of construction-time initial cash — reaching the previously-dead
`SimulatedCashAccount.restore_cash` + `PositionManager.restore_realised_pnl` purely via
resequencing, with no logic change to the restore leaves.

## What was built

### Task 1 — D-23: lift rehydrate() out of the okx arm
Previously `rehydrate()` sat INSIDE the `if self.exchange == 'okx' and self._venue_account is not None:`
block, AFTER `_link_venue_account_to_portfolios()` had already swapped every portfolio onto the
`VenueAccount` (whose `restore_cash` is a documented no-op). On a non-okx (paper/simulated) engine that
whole arm is skipped, so `rehydrate()` never fired: `SimulatedCashAccount.restore_cash` +
`PositionManager.restore_realised_pnl` (built and unit-tested in 05.2) were dead, and a durable paper
account restarted with its construction-time initial cash.

The fix hoists the store-gated rehydrate call:

```python
# D-23: ungated — runs for ANY durable store, sequenced BEFORE the okx block
if hasattr(self._order_storage, 'rehydrate'):
    self.portfolio_handler.rehydrate(
        self.order_handler.order_manager.seed_applied_trades)

if self.exchange == 'okx' and self._venue_account is not None:
    self._venue_account.snapshot()
    self._venue_account.start_streaming()
    self._link_venue_account_to_portfolios()
    if hasattr(self._order_storage, 'rehydrate'):
        # VenueReconciler.reconcile() — stays okx-only
        ...
    self._run_session_baseline_guard()
```

Rehydrate now runs FIRST (before any `VenueAccount` swap), so a paper/simulated account restores while
its portfolio still holds the `SimulatedAccount`. `VenueReconciler.reconcile()` / `snapshot()` /
`_link_venue_account_to_portfolios()` stay strictly inside the okx arm — RESTORE semantics only on the
paper path (no venue reconcile). The `seed_applied_trades` sink (D-22 order-mirror dedup ring) is still
threaded through. The in-memory backtest backend is a per-portfolio getattr clean-skip inside
`rehydrate()` (oracle-dark), so the backtest hot path is untouched.
