### Gaps Summary

Three of the phase's 13 D-NN decisions have a confirmed gap between "code exists and is unit-tested" and "the feature actually functions/is reachable in production":

1. **D-15 (blocker, CR-01):** The buying-power double-count fix (`drop_pending`) rests on a false premise for the wired spot venue — `_venue_balance` is never refreshed by the balance stream for `market_type='spot'` (positions-only), so dropping the local overlay on ack removes the *only* tracker of a resting order's cash commitment. This does not merely fail to fix V17-13 — it introduces a live buying-power over-statement where none existed as starkly before (previously the overlay understated buying power until terminal release; now it can be wrong in the *other* direction, over-stating it, allowing a second order to be admitted against already-committed cash). This is a decision-level conflict between D-15 and the D-01 single-channel cash design that the plan did not resolve.

2. **D-12 (WR-01):** The missed-fill catch-up machinery (`catch_up_missed_fills`, the disconnect-ts floor, the bounded `fetch_my_trades` page) is fully built and unit-tested but has zero production call sites. It is dead code — fills settling during a stream outage are not recovered on reconnect, only at the next full startup reconcile.

3. **D-11 (WR-02, partial):** The connector-level dead-task fail-safe (`_on_task_done` → halt) correctly *observes* `task.exception()` but cannot *escalate* because `OkxConnector.set_halt_signal` is never wired into the composition root. The exchange/provider/portfolio arms are all armed; the connector arm is the one omission.

All three gaps share a pattern: the individual code unit is correct-in-isolation and passes its own unit test, but the composition-root wiring or a false cross-module assumption breaks the end-to-end behavior the phase goal requires. This is exactly the class of defect goal-backward verification is meant to catch — SUMMARY.md claims for 05.3-01, 05.3-02, and 05.3-04 all describe the mechanism as complete without surfacing that it is unreachable/mis-premised in production.
