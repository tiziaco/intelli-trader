<objective>
Two engine order-flow gating fixes, one of them the phase's HIGH security threat:
- **D-18** [V17-16 — HIGH]: close the zero-validation external-order injection. Route external order creation through the `AdmissionManager` pipeline (validation + sizing + reservation + mirror), wire the DEAD `OkxExchange.validate_order`/`validate_symbol` preflight in `on_order` as defense-in-depth (mirroring `simulated.py`), and restrict `add_event` so raw `OrderEvent`s cannot be injected onto the queue.
- **D-14** [V17-11]: during pause/halt, DEFER system-generated protective orders (bracket children, OCO/orphan cancels) into a replay queue drained on resume; ALWAYS let CANCEL through (reduces risk); keep suppressing only fresh ENTRY orders.

Purpose (D-18): today `add_event(OrderEvent)` reaches the live queue with only a `_running` check — no validation, sizing, reservation, or mirror. An external/web caller can inject an arbitrary order that bypasses every control (ASVS V4/V5 access-control + input-validation defect). Purpose (D-14): the current pause gate blanket-suppresses BOTH SIGNAL and ORDER, so protective children/cancels are silently dropped during pause — a risk-management hole.
Output: injected orders traverse admission + preflight; protective orders replay on resume; new RED→GREEN; oracle byte-exact.
</objective>
