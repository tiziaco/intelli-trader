## Accomplishments

- **D-18 (HIGH, V17-16):** `LiveTradingSystem.add_event` now refuses raw `EventType.ORDER` injection — an external/web caller can no longer inject an unvalidated, unsized, unreserved order onto the live queue. The sanctioned path is signal-form entry (a `SignalEvent` routes through `OrderHandler.on_signal` → `AdmissionManager` for validation + sizing + reservation + mirror). Non-ORDER events enqueue unchanged; the internal order flow (handlers `global_queue.put` directly) is unaffected.
- **D-18 defense-in-depth:** `OkxExchange.on_order` now runs the previously-dead `validate_order`/`validate_symbol` preflight before any venue call (mirroring `simulated.py`). A preflight failure (non-positive quantity, unknown symbol) is a *definitive* rejection → `FillEvent(REFUSED)` via a new `_refuse_preflight` helper, and the venue is never called.
- **D-14 (V17-11):** The `_dispatch_live` pause/halt gate no longer blanket-suppresses SIGNAL+ORDER. It branches by order kind — CANCEL always passes (risk-reducing), a PROTECTIVE order (bracket child, `parent_order_id` set) is deferred onto a bounded replay queue, and a fresh ENTRY / SIGNAL stays suppressed. `resume_submission` drains the queue via `_replay_deferred_protective`, replaying each protective order through `_dispatch_live` so a just-filled position is never left naked.
- **A8 spine GREEN (3/3):** `test_pause_defer_replay.py` now passes (protective-replayed + cancel-always were RED; entry-stays-suppressed control stayed GREEN).

## Decisions Made

- **Sanctioned external-order path = SIGNAL-form entry.** `add_event` keeps enqueuing non-ORDER events, so submitting a `SignalEvent` remains the admission-backed route (queue → `EventHandler` → `OrderHandler.on_signal` → `AdmissionManager`). No parallel validator was built (per plan and D-18).
- **Replay-drain is snapshot-then-clear.** `_replay_deferred_protective` copies the deque into a local batch and clears it before re-dispatching. If a re-dispatch finds the engine HALTED and re-defers, it appends to the now-empty deque instead of spinning the drain loop forever.
- **Preflight rejection routes to the definitive path**, not the D-13 ambiguous/in-flight path — a preflight-refused order never reached the venue, so `FillEvent(REFUSED)` (mirror → REJECTED) is correct, mirroring the `ccxt.InvalidOrder` branch.
