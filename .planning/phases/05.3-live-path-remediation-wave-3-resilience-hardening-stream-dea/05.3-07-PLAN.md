<objective>
Durable restart-integrity, two folded-warning fixes on the shared 05.2 SQL substrate:
- **D-19** [WR-04, owner-locked single atomic transaction]: wrap the `on_fill` position upsert (`set_position`, store-first COMMIT #1) and the cash-scalar `save_account_state` (independent `engine.begin()`, COMMIT #2) in ONE transaction so a crash between them can never tear positions ahead of cash.

Purpose: today the two persists are independent commits on one engine → a crash leaves an inconsistent baseline that `VenueReconciler` restarts from; and the order-mirror ring starts empty on restart so a re-delivered pre-restart trade re-books. Both are cross-restart integrity defects.
Output: single-transaction persist; symmetric restart-seeded dedup; both RED tests GREEN; oracle byte-exact.
</objective>
