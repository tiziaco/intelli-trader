### Simulated-account restart restore (05.2 review follow-up — owner-confirmed 2026-07-05)

- **D-23:** [SimulatedExchange cash-restore gap — **owner design choice locked: option (a), restore the persisted scalar**] **Ungate `portfolio_handler.rehydrate()` from the `if self.exchange == 'okx'` arm** (`live_trading_system.py:1337/1351/1360`): run it whenever the store exposes `rehydrate()` (`hasattr(self._order_storage, 'rehydrate')`, i.e. Postgres spine present), REGARDLESS of exchange. Keep `VenueReconciler.reconcile()` / `_venue_account.snapshot()` / `_link_venue_account_to_portfolios()` INSIDE the okx arm only. This ACTIVATES the already-written-but-currently-dead `SimulatedCashAccount.restore_cash` (`simulated.py:167`, sets `self._balance` from `load_account_state()["cash_balance"]`) + `PositionManager.restore_realised_pnl` (WR-03) for a durable paper/simulated account — today they are unreachable because rehydrate fires only on the okx path, where `_link_venue_account_to_portfolios` (1340) has already swapped every portfolio onto the `VenueAccount` (whose `restore_cash` is a no-op) BEFORE rehydrate (1360).
  - **Semantics — RESTORE, not reconcile:** the DB `portfolio_account_state` row is the engine's OWN persisted memory, not an independent authority; the `SimulatedAccount` ledger is the sole source of truth (no external venue to self-correct against). Consistency of the restored cash+positions pair is guaranteed by **D-19's atomic write**, NOT by a venue snapshot — a torn restore is PERMANENT on the simulated path (nothing heals it), which is exactly why D-19 matters more here than on live.
  - **NOT option (b)** (recompute cash+positions by replaying the durable `transactions` ledger, event-sourcing style) — deferred; and NOT "recompute from positions" (cash is not derivable from positions — both are derived from the transaction stream).
  - **Safety:** `rehydrate()` is internally getattr-guarded per portfolio (in-memory backtest backend is a clean skip — 1357-1359), so ungating is inert when no durable store is wired; SMA_MACD stays oracle-dark. Files: `live_trading_system.py:1337-1360` (ungate + resequence), `portfolio_handler.py` (`rehydrate`), `simulated.py:167` (`restore_cash`), `position/position_manager.py` (`restore_realised_pnl`).

### Gap Remediation (post-verification, owner-confirmed 2026-07-05)

Closes the three confirmed gaps in `05.3-VERIFICATION.md` (`gaps_found`, 20/23). Each shares one pattern: the code unit is correct-in-isolation and green on its unit test, but composition-root wiring (or a cross-module premise) breaks the end-to-end behavior. Plans MUST cite each `D-NN`.

- **D-24:** [fixes D-15 / CR-01 — **owner design choice locked: option (a), gate the drop for the single-channel spot leaf**] Make `VenueAccount.drop_pending` (`account/venue.py:702`) a **no-op when `market_type == 'spot'`** — the D-01 single-channel cash leaf. **Rationale:** the `_pending` overlay bridges "cash committed to a resting order" vs. "cash reflected in `_venue_balance`." That gap closes at ack ONLY when a cash channel refreshes `_venue_balance` (margin/swap). On the wired spot leaf, `_write_balance_stream` (`venue.py:281-314`) is **positions-only** and `_venue_balance` is re-baselined solely by `snapshot()` — so the overlay is the **SOLE** tracker of the hold for the order's entire resting life; dropping it on ack snaps `available_balance` back to full settled balance → buying-power **over-statement**. On spot there was never a double-count for D-15 to remove (V17-13's double-count exists only on a stream-refreshed account), so D-15's drop-on-ack stays **dormant-valid** for a future margin account. The existing terminal fill/cancel path already pops the overlay, so "hold until terminal release" needs no new code — just the guard. Prefer an intent-revealing predicate (e.g. `self._cash_stream_nets_holds`) over the raw string if cheap; `market_type == 'spot'` is the acceptable proxy for what's wired today. **NOT** option (b) (net venue `used`/`free` into `_venue_balance`) — rejected: it reintroduces the dual-channel cash authority D-01 deliberately killed (cash flicker on stream lag, re-opens a settled decision). File: `account/venue.py:702` (`drop_pending`), context `venue.py:281-314` (`_write_balance_stream`), `live_trading_system.py:471` (spot construction).

- **D-25:** [fixes D-12 / WR-01] Wire the orphaned `OkxExchange.catch_up_missed_fills()` (`okx.py:626`, zero production call sites today) into the resume drain `_maybe_resume_after_reconnect` (`live_trading_system.py:905-939`), **before** `self._venue_account.snapshot()`. **Resume path only** — at plan time grep for a distinct `attempt==1` reconnect entry point (D-12's original wording named both); wire it too ONLY if it genuinely exists on the live path, do not invent a call site. The `_disconnect_ts_ms` floor + bounded `fetch_my_trades` page already exist; this is pure wiring. File: `live_trading_system.py:905-939`, `okx.py:626-673`.

- **D-26:** [fixes D-11 / WR-02] Arm the connector-level halt escalation at the composition root: add `self._okx_connector.set_halt_signal(self._request_connector_halt)` alongside the exchange/provider/portfolio arms (`live_trading_system.py:502-505,566`). `OkxConnector._on_task_done` (`connectors/okx.py:209`) already *observes* `task.exception()` correctly; today its `if self._halt_signal is not None:` guard is always False because `set_halt_signal` (`connectors/okx.py:94`) has no caller. One line closes the observe-but-can't-escalate gap. File: `live_trading_system.py` (OKX wiring, ~502-505/566).

- **D-27:** [cross-cutting DoD for D-24..D-26] Every gap fix MUST ship a test that exercises the **production wiring/premise**, not the mechanism in isolation — the failure class the verifier flagged (all three passed their unit tests; the unit was never the problem). Specifically: **D-24** — construct `VenueAccount(market_type='spot')`, snapshot a settled balance, reserve + ack an order, assert `available_balance` does NOT snap back to full settled while the order is still open. **D-25** — integration-level: the resume path recovers a fill that settled during a simulated disconnect window (not `catch_up_missed_fills` in isolation). **D-26** — wiring-level: assert the composition root arms `_okx_connector`'s halt signal at `start()` (not the existing `_on_task_done` unit test, which injects a halt-signal double directly).
