## Implementation Decisions

Carved from the master register (`v17_bugs.md` Wave 3 + 05-13 carry-overs). Plans MUST cite each `D-NN` in `must_haves`/`truths`/`objective`.

- **D-11:** [V17-07] Add `except Exception → _escalate_connector_halt` catch-all to BOTH supervisors (fail-safe: unknown ⇒ halt, never silent task death); move `json.loads` inside the per-message guard (`okx_provider.py`); wrap `_stream_account`/`_stream_positions` (`account/venue.py`, bare `while True`) in the same supervisor; extract the duplicated ~100-line state machine into one shared helper (`_run_stream_supervisor` already exists at `okx.py`, already scrubs secrets — preserve the scrub); `connector.spawn` done-callback logs `task.exception()` and escalates.
- **D-12:** [V17-08] On resume AND the attempt==1 reconnect path, run a bounded `fetch_my_trades(symbol, since=disconnect_ts)` routed through `_handle_trade` (safe once D-08 dedup lands); translate `watch_orders` CANCELLED/EXPIRED (`okx.py`) into `FillEvent(CANCELLED/EXPIRED)` so venue-side cancels reconcile the mirror. (Steady-state arm may fold into the Phase-7 mid-session spec; pull the missed-fill catch-up forward here.)
- **D-13:** [V17-09] On `TimeoutError`/any ambiguous transport error at submit (`okx.py`, `connectors/okx.py`), do NOT emit `FillEvent(REFUSED)` — mark the mirror in-flight/unknown and resolve via `fetch_order(clOrdId)` retry or the next reconcile; only a definitive venue rejection produces REFUSED.
- **D-14:** [V17-11] During pause/halt (`live_trading_system.py`), DEFER system-generated protective orders (bracket children, OCO/orphan cancels) into a replay queue drained on resume; ALWAYS let CANCEL commands through (they reduce risk); keep suppressing only fresh entry orders.
- **D-15:** [V17-13] Drop the local pending-overlay entry (`account/venue.py`) once the venue ack for that order arrives (venue `free` already excludes the hold), instead of waiting for terminal release — removes the buying-power double-count. Trigger = the D-06 ORDER-ACK event (05.2).
- **D-16:** [05-13 carry-overs] Move mark-seen AFTER successful `_emit_fill` (or validate payload inside `resolve` before consuming the dedup slot) — WR-02; route `release()` drains through `resolve` not raw `_emit_fill` — WR-03; `release_pending(clordid)` on submit failure — WR-01; reject `capacity < 1` — IN-01; bound + alarm the uncorrelated-fill buffer (`venue_correlation.py`); seed `adopt()`'s cumulative counter from rehydrated `filled_quantity` (needs D-07).
- **D-17:** [V17-15] Make the mid-session gap backfill loop-native — an async `fetch_ohlcv_backfill` awaited directly inside the candle coroutine (no blocking `connector.call()` bridge from loop-thread code), or hand the gap range to a dedicated backfill coroutine spawned on the loop. Kills the 30s full-connector stall → livelock (`live_bar_feed.py`, `okx_provider.py`, `connectors/okx.py`). The warmup path (engine thread) stays unchanged.
- **D-18:** [V17-16] Route external order creation through the admission pipeline (signal-form entry or a direct `AdmissionManager` call) so validation + sizing + reservation + mirror engage; wire `OkxExchange.on_order` preflight (`validate_order`, `validate_symbol`) as defense-in-depth mirroring `simulated.py`; optionally restrict `add_event` (`live_trading_system.py`) to non-ORDER event types.
