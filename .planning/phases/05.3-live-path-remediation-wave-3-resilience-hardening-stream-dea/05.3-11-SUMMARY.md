# Phase 05.3 Plan 11: Close WR-01 / WR-02 Wiring Gaps (D-25 / D-26) Summary

Closed the two confirmed production-wiring gaps from 05.3-VERIFICATION: the resume drain now
re-fetches fills that settled during a stream outage (D-25 / WR-01), and the composition root
arms the connector's halt signal so a task dying outside the stream supervisors escalates to a
fail-safe halt (D-26 / WR-02). Both are pure composition-root wiring on `live_trading_system.py`
with production-wiring tests per D-27; the oracle stays byte-exact and the OKX stack stays
hot-path-inert.

## What Was Built

### Task 1 — D-25 / WR-01: catch_up_missed_fills wired into the resume drain
- `_maybe_resume_after_reconnect` (`live_trading_system.py`) now calls
  `self._okx_exchange.catch_up_missed_fills()` **before** `self._venue_account.snapshot()`,
  inside the existing `try/except` (a catch-up failure logs and re-sets `_pending_stream_resume`
  — stays paused, never resumes blind), guard-claused on `_okx_exchange is not None` (mirrors the
  `_venue_account` guard). The previously-orphaned method (okx.py:626, zero call sites) now has
  exactly **one** production call site (line 949).
- Integration test `tests/integration/test_resume_missed_fill_catchup.py` drives the RESUME PATH
  (not the exchange method in isolation): with a stubbed connector returning an outage-window
  trade and `_disconnect_ts_ms` set, the resume drain routes the trade through `_handle_trade`
  exactly once, **before** the snapshot, consumes the floor, and clears the pause. A second case
  proves the `_okx_exchange is None` guard branch resumes cleanly.

### Task 2 — D-26 / WR-02: arm the connector's halt signal at the composition root
- Added the 4th `set_halt_signal` arm — `self._okx_connector.set_halt_signal(self._request_connector_halt)`
  — alongside the exchange/provider arms (`live_trading_system.py`). `OkxConnector._on_task_done`'s
  `if self._halt_signal is not None:` guard is no longer always-False: a task dying outside the
  exchange/provider stream supervisors now escalates to a fail-safe engine halt instead of
  logging-and-vanishing.
- Wiring test added to `tests/integration/test_live_system_okx_wiring.py`
  (`test_start_arms_okx_connector_halt_signal`): constructs an OKX system, runs `start()` fully
  offline, and asserts the **real** connector's `_halt_signal` equals `_request_connector_halt`
  (not the `_on_task_done` unit's injected double).

## attempt==1 grep result (per D-25 grounding)
Grep of `attempt` across `live_trading_system.py`, `connectors/okx.py`, `execution_handler/exchanges/okx.py`
shows the only `attempt` logic is inside the okx.py reconnect supervisor (`_run_stream_supervisor`).
There is **NO distinct `attempt==1` live-path entry point** that would need its own
`catch_up_missed_fills` call site: every reconnect (attempt>1 marks the stream down, sets
`_disconnect_ts_ms`, fires `on_stream_down`; a successful re-subscribe fires `on_stream_up`) funnels
through the `_pending_stream_resume` flag → the engine-thread `_maybe_resume_after_reconnect` drain.
Wiring the resume drain is therefore the single, correct, complete call site. **No call site was
invented.**
