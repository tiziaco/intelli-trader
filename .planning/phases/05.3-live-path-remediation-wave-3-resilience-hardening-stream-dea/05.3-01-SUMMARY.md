# Phase 05.3 Plan 01: Fail-safe OKX stream supervisors (D-11 / V17-07) Summary

**Every supervised OKX coroutine (exchange order/fill arm, provider candle arm, VenueAccount balance/position streams) now escalates an unknown error to a fail-safe HALT instead of dying silently; garbage candle frames escalate via a per-message guard; the connector spawn done-callback observes task.exception(); the secret scrub is preserved and the SMA_MACD oracle stays byte-exact.**

## Performance

- **Duration:** ~40 min
- **Completed:** 2026-07-05
- **Tasks:** 2
- **Files modified:** 7 (4 source, 3 test)

## Accomplishments
- Closed V17-07 (silent stream death): a `except Exception -> _escalate_connector_halt(..., "unexpected error")` fail-safe arm on BOTH the `OkxExchange` order-arm and `OkxDataProvider` candle-arm supervisors — an unclassified error HALTS the engine rather than letting the `asyncio` task die unobserved.
- Moved the provider `json.loads` inside a per-message guard (logs `type(exc).__name__` only, re-raises) so a garbage/non-JSON candle frame escalates to a halt via the supervisor catch-all (T-05.3-03) instead of crashing the raw read loop.
- Wrapped the two bare `VenueAccount` streams (`_stream_account`/`_stream_positions`) in the same bounded-retry supervisor shape (CancelledError re-raise -> fatal -> transient backoff -> catch-all), with a scrubbed `_escalate_connector_halt` + `set_halt_signal` seam.
- Replaced the connector `spawn` discard-only done-callback with `_on_task_done`: it untracks the task AND observes `task.exception()` (skipping `CancelledError`), logs the type only, and escalates `connector-fatal`.
- A6 spine (`test_supervisor_catchall.py`) turned 3/3 GREEN; secret scrub preserved (no `str(exc)` on any halt egress); `mypy --strict` clean; oracle byte-exact (134 / 46189.87730727451).

## Decisions Made
- **Supervisor-fold Discretion NOT taken (follow-up).** The plan left folding the three `_run_stream_supervisor` copies (exchange / provider / venue) into one shared helper optional. They carry different transient tuples (the provider adds `aiohttp.ClientError`/`ConnectionError`/`asyncio.TimeoutError` and treats a clean return as socket-close-reconnect) and different `_on_stream_down`/budget wiring, so folding now would need careful parametrisation. Left as three inline copies; a shared helper is a candidate follow-up.
- **Venue survival assertion re-expressed** via `watch_balance.call_count == 3` (blip -> recovery -> stop) rather than a `_venue_balance` value — the balance stream is deliberately positions-only since the cash-double-count fix (30cfb73), so cash is `snapshot()`-only.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Migrated two harness loop-breaker sentinels to BaseException**
- **Found during:** Task 1 (full-suite regression check)
- **Issue:** The new fail-safe `except Exception` catch-all correctly intercepts ANY unclassified exception. Two sibling tests (`test_okx_data_provider.py::test_supervisor_post_snapshot_payload_resets_budget`, `test_reconnect_resilience.py::test_okx_provider_transient_error_reconnects_and_survives`) used unclassified `Exception` sentinels (`_StopLoop`, `_StopSupervisor`) purely as harness loop-breakers, expecting them to PROPAGATE (they verify NON-halt behaviour). Post-catch-all those sentinels were mis-escalated to a halt, failing both tests.
- **Fix:** Changed both harness sentinels from `Exception` to `BaseException` (a control-flow signal like `asyncio.CancelledError` — not a real venue error), so the production catch-all does not weaken and the tests' non-halt intent is preserved. Updated their docstrings to explain the D-11 rationale.
- **Files modified:** `tests/unit/connectors/test_okx_data_provider.py`, `tests/unit/execution/test_reconnect_resilience.py`
- **Verification:** Both tests GREEN; A6 spine 3/3 GREEN; no production behaviour weakened.
- **Committed in:** `867e4dbf` (Task 2 commit)
