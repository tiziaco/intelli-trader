<objective>
Close the two BLOCKER findings from the Phase-2 code review (02-REVIEW.md):

- **CR-01 (real, completes LEV-03)** — `_build_primary_order` threads the admission-clamped
  EFFECTIVE leverage onto the MARKET order (`Order.new_order(..., leverage=effective_leverage)`)
  but the LIMIT and STOP arms call `Order.new_limit_order` / `Order.new_stop_order`, factories
  that neither accept nor set `leverage`, so the entity defaults to `Decimal("1")`. The admission
  cash reservation reserves `notional / effective_leverage` REGARDLESS of order type, so a levered
  LIMIT/STOP entry reserves `notional/L` but locks `aggregate_notional/1` (full notional) at the
  position — locked margin ≠ reservation. This is the exact LEV-03 invariant violation, just for
  the non-MARKET path 02-07 missed (masked today only because SMA_MACD is MARKET-only).

- **CR-02 (mitigate now, full fix → Phase 3)** — the margin partial-close arm in
  `_process_transaction_margin` clamps `closed_qty = min(transaction.quantity, prior_qty)` but
  reads `realised_increment` after the FULL unclamped `transaction.quantity` mutates the position.
  An over-close (SELL qty > open long) leaves a residual flipped SHORT, the position stays open,
  and the code re-locks margin on a position whose leverage was set for the original LONG and
  settles a wrong cash delta. This is UNREACHABLE in Phase 2 (shorts/flips are gated off at
  `strategies_handler.add_strategy`), so the full flip-settlement economics belong to Phase 3.
  This plan adds the cheap FAIL-LOUD guard (reject the over-close) so it can never silently
  mis-settle if a flip fill ever reaches the arm.

Everything is owner-gated and oracle-dark: with `enable_margin` off the leverage threading and the
guard never fire and the SMA_MACD spot oracle (134 / 46189.87730727451) holds byte-exact.

INDENTATION (CRITICAL — TABS in all three production files: order.py, admission_manager.py,
portfolio.py; match the file, never normalize).
MONEY: Decimal end-to-end; never Decimal(float); enter via to_money(); leverage default Decimal("1").
