<objective>
Build the lock-and-settle margin cash model gated by `enable_margin` (D-09/D-10/D-11),
which is byte-exact site #2 (the settlement path). Add a position-keyed `locked_margin`
container to `CashManager` (distinct from the order-keyed pending reservation, Pitfall 2),
make `available_balance` subtract it (`balance − reserved − locked_margin`, Pitfall 6),
give `Position` one effective leverage set at open (D-06), and branch
`Portfolio.process_transaction` on `enable_margin`: spot debits the full notional
unchanged; margin debits ONLY commission on open + locks `aggregate_notional / L`, and on
close settles realized PnL + releases the lock pro-rata (D-11).

Purpose: This is the only model that supports Phase-3 shorts (no notional to spend) and
Phase-4 liquidation (a locked margin to floor loss against). It is byte-exact site #2:
the spot arm MUST remain `apply_fill_cash_flow(net_delta)` exactly as today, and
`locked_margin == Decimal("0")` in spot mode so `available_balance` is byte-identical.
Output: position-lifetime locked margin + lock-and-settle lifecycle; SMA_MACD byte-exact.

Runs in Wave 2 PARALLEL with Plan 03 (zero file overlap — Plan 03 owns the order domain +
compose root, Plan 04 owns the portfolio cash/position internals).
</objective>
