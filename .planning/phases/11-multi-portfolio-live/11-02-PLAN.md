<objective>
Land the W5 attribution boundary (D-28 — the one wave marked *independent, parallelizable*): rename
the engine-side identifiers off the venue's field spelling (D-16 / LR-19 / MPORT-04), consolidate the
wire-vocabulary extraction behind the single helper that already does most of it, and convert the
strippable `assert` guarding a venue-bound identifier into a real raise (D-18).

D-17 is the load-bearing constraint on what this plan must NOT do: MPORT-04's "every submitted order
is tagged with its portfolio" is **already satisfied** — `SignalEvent`, `OrderEvent`, `OrderAckEvent`,
`FillEvent` and `Order` all carry a typed `portfolio_id` today. Attribution is derivable three
independent ways without touching the identifier string. So this is a rename plus a guard fix, not a
format change.

Purpose: the venue's spelling leaking into engine identifiers is what makes a second venue expensive,
and a guard that vanishes under `python -O` is no guard at all on an identifier bound for a real
exchange.

Output: engine identifiers renamed, one documented venue-vocabulary seam, a real raise, and unit tests
proving both the rename and the preserved wire contract.
</objective>
