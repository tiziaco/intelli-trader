<objective>
Land MPORT-07 (D-27): make the execution exchange per-`(venue, account_id)`.

**This is the requirement that makes the rest of the phase actually work.** The discussion found that
MPORT-01..06 was necessary but not sufficient: connectors are keyed on the pair, but the EXCHANGE is
keyed by bare venue name while the OKX exchange holds exactly ONE connector. Two portfolios on one
venue with different accounts therefore both resolve to the same exchange object, and **account B's
orders would be submitted through account A's authenticated session** — with per-account credentials,
per-account venue accounts, and the distinct-account invariant all correct. The decisive constraint is
the private per-account fill stream: one shared exchange cannot correctly subscribe to two accounts'
fill streams at all.

Investigation showed every piece of mutable state on the OKX exchange is ALREADY account-scoped
(connector, correlation index, stream handles, connection status) and the markets/precision map lives
on the connector rather than the exchange. So the exchange's true cardinality already IS the pair; it
has only ever been instantiated once because there has only ever been one account. **D-27 makes an
existing dimension explicit rather than adding one.**

Purpose: without this, per-account credentials and accounts are all correct and orders still route to
the wrong venue account — silently.

Output: a pair-keyed exchange registry, an account-resolving order route, every one of the ten source
sites and twenty-five test sites converted, and a routing gate that proves an order for one account
never reaches the other account's exchange.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| order event -> authenticated venue session | The exchange lookup decides which real, authenticated, money-holding account an order is submitted through |
| `ExecutionHandler` -> portfolio layer | A read-only cross-domain seam; the account id it returns is what selects the session |
| backtest-shared code path -> live code path | The same handler class serves both; a keying change on the shared path can break the byte-exact oracle |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
