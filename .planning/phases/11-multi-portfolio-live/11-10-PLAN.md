<objective>
Complete the W6 reconcile boundary (D-28): replace the global latched halt on unexplained drift with a
per-portfolio quarantine, gate new entries for a quarantined portfolio, give an operator a CONTROL
command to release it, and surface the list so an operator can see what to release.

**The blast radius is the whole argument.** Today an unexplained residual on one account latches a
global halt and stops everything. After this phase, accounts are fully isolated — own connector, own
credentials, own venue account, own exchange — and the distinct-account invariant guarantees no
sharing. So drift on account A carries **zero information** about account B, and halting B is simply
wrong. The owner asked what the quarantine would cost before committing, got a per-piece breakdown
showing most of it already exists, and took the more correct blast radius rather than the smaller diff.

**And it genuinely is assembly rather than new machinery.** The safety controller already holds exactly
this shape as global scalars behind a status lock; this is a per-portfolio set beside them. The
admission manager is already fully keyed by portfolio, so the gate is one guard clause. The critical
alert channel and the status read-model both already exist.

Note the deliberate contrast with the distinct-account collision: quarantine is right HERE because the
accounts are isolated; a collision refuses to start because there is no safe way to choose which
colliding portfolio is legitimate.

Output: a per-portfolio quarantine set, a one-clause admission gate, an operator-only CONTROL release,
and a pull surface listing what is quarantined and why.
</objective>
