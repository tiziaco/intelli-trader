<objective>
Implement `SqlOrderStorage` — the order-mirror operational backend (OPS-01) on the shared `SqlBackend` spine, money as native `Numeric` (OPS-04), validated by a testcontainers-Postgres round-trip. Purpose: the order seam is the first of three operational backends; it sets the bracket-FK + child-table pattern the phase reuses.
Output: `models.py`, `sql_storage.py`, an edited `storage_factory.py`, a deleted `postgresql_storage.py` (+ its pyproject mypy override removed), and a Postgres round-trip test.
</objective>
