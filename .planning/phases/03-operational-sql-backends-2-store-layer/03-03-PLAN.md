<objective>
Implement `SqlPortfolioStateStorage` — the portfolio-state operational backend (OPS-02) on the shared spine, across six normalized tables (D-03), money native `Numeric` (OPS-04), validated on testcontainers Postgres. The defining nuance: the `PortfolioStateStorage` ABC has NO `portfolio_id` parameter on any of its ~21 methods (the in-memory backend is one-instance-per-Portfolio), so the SQL backend binds a `portfolio_id` at construction and scopes every query to it (Pitfall 1). Mirrors `itrader/results/{models,sql_storage}.py`.

Purpose: portfolio state is the richest of the three seams (six collections, bound-scope isolation, Position identity-equality). It proves the cross-portfolio-isolation invariant Phase 4 rehydration depends on.
Output: `models.py` (six tables), `sql_storage.py` (bound portfolio_id, ~21 methods), an edited `storage_factory.py`, and a Postgres round-trip test.
</objective>
