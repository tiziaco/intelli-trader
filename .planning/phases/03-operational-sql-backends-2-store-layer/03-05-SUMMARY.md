## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Added missing `import itrader.storage.types` to the autogenerated migration**
- **Found during:** Task 2 (hand-review of the autogenerated revision)
- **Issue:** Alembic autogenerate renders the custom `UtcIsoText` `TypeDecorator` by its fully-qualified name (`itrader.storage.types.UtcIsoText()`) in every time column but does NOT emit the import — the standard custom-type autogenerate gotcha. `alembic upgrade head` would raise `NameError`.
- **Fix:** Hand-added `import itrader.storage.types` (with an explanatory comment) to the migration's import block.
- **Files modified:** `itrader/storage/migrations/versions/2cbf0bf6b0b6_operational_baseline.py`
- **Verification:** `alembic upgrade head` applies cleanly on fresh Postgres and SQLite; `test_migrations.py` green.
- **Committed in:** `67e9a82` (Task 2 commit)

**2. [Rule 1 - Bug] Updated existing chain tests broken by the now-non-empty chain + fixed the Postgres teardown's cleanliness contract**
- **Found during:** Task 2 (the baseline turned `upgrade head` from creating 1 table into creating 10)
- **Issue:** The SQLite test asserted `applied == []` (empty chain) and the Postgres test asserted the same while its teardown dropped only `alembic_version` — its docstring promises a pristine shared session container, but it would now leak 9 operational tables.
- **Fix:** Both tests now assert the single baseline revision stamps `alembic_version` and builds the operational tables; the Postgres test reverses the whole migration via `command.downgrade(cfg, "base")` before dropping `alembic_version`.
- **Files modified:** `tests/integration/storage/test_migrations.py`
- **Verification:** `test_migrations.py` 3 passed; full storage suite 36 passed (no leftover-table collisions with sibling round-trip tests, which use `create_all(checkfirst=True)`).
- **Committed in:** `67e9a82` (Task 2 commit)
