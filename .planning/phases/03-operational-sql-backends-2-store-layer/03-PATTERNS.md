### `itrader/order_handler/storage/models.py` (model, transform)

**Analog:** `itrader/results/models.py` (the `build_results_tables` idempotent registrar).

**Imports pattern** ‚Äî `itrader/results/models.py:15-26`. Import Core constructs from `sqlalchemy`, spine types from `itrader.storage`:
```python
from sqlalchemy import (
    Boolean, Column, ForeignKey, Index, Integer, Numeric, String, Table, MetaData,
)
from itrader.storage import Uuid, UtcIsoText, json_variant
```
> `Uuid`, `UtcIsoText`, `json_variant` are the ONLY id/time/json types allowed (RESEARCH ¬ßDon't Hand-Roll). `Numeric` (unbounded, `asdecimal=True` default) is the money type ‚Äî imported straight from `sqlalchemy`, NOT from the spine (there is deliberately no money TypeDecorator ‚Äî `itrader/storage/types.py:18-21`).

**Idempotent registrar pattern** ‚Äî copy `itrader/results/models.py:37-112` shape exactly (the `if "<name>" in metadata.tables: reuse else: build` guard per table). This phase's function:
```python
def build_order_tables(metadata: MetaData) -> dict[str, Table]:
    tables: dict[str, Table] = {}
    if "orders" in metadata.tables:
        tables["orders"] = metadata.tables["orders"]
    else:
        tables["orders"] = Table("orders", metadata, ...)   # column map below
    # order_state_changes child table (same guard)
    return tables
```

**`orders` column map** ‚Äî RESEARCH.md ¬ßTable Maps (lines 292-320) + the verbatim `Order` field list at `itrader/order_handler/order.py:46-108`. The full `Table(...)` excerpt is pre-written in RESEARCH.md ¬ßCode Examples (lines 662-693), including the **D-02 self-referential FK** `Column("parent_order_id", Uuid(as_uuid=True), ForeignKey("orders.id"), nullable=True, index=True)` and the **D-08 index** `Index("ix_orders_portfolio_status", "portfolio_id", "status")`.
- `child_order_ids` (`order.py:86`) is **NOT a column** (D-02) ‚Äî derived on read via `SELECT id FROM orders WHERE parent_order_id = :id`.
- `state_changes` (`order.py:82`, a `List[OrderStateChange]`) ‚Üí **`order_state_changes` child table**, PK `(order_id, seq)`, columns per RESEARCH lines 328-331 (`order_id` Uuid FK, `seq` Integer, `from_status`/`to_status` Str, `timestamp` UtcIsoText, `reason` Str, `triggered_by` Str, `additional_data` json_variant NULL). `OrderStateChange` fields are at `order.py:19-30`. **Load-bearing for D-10** ‚Äî `Order.__eq__` (dataclass, `order.py:35`) compares `state_changes` field-wise.

### `itrader/order_handler/storage/sql_storage.py` (storage backend, CRUD)

**Analog:** `itrader/results/sql_storage.py` (the `SqlResultsStore` class).

**Composition + `create_all(checkfirst=True)` constructor** ‚Äî copy `itrader/results/sql_storage.py:83-110` exactly:
```python
class SqlOrderStorage(OrderStorage):
    def __init__(self, backend: SqlBackend) -> None:
        self.backend = backend
        self.engine = backend.engine
        tables = build_order_tables(backend.metadata)
        self.orders = tables["orders"]
        self.state_changes = tables["order_state_changes"]
        backend.metadata.create_all(self.engine, checkfirst=True)   # tests/dev; deploy uses Alembic
        self.logger = get_itrader_logger().bind(component="SqlOrderStorage")

    def dispose(self) -> None:
        self.backend.dispose()   # WR-03: delegate, never self.engine.dispose()
```
> `SqlResultsStore.dispose` (`sql_storage.py:108-110`) is the WR-03 delegation precedent ‚Äî every `Sql<Concern>Storage` must delegate to `backend.dispose()`.

**Parameterized Core read** ‚Äî copy `itrader/results/sql_storage.py:218-222` (`select(table).where(col == bindparam("id"))` + `.mappings()`). NEVER f-string SQL (SEC-01 / V5.3). The `get_order_by_id` second-query child rebuild (RESEARCH Pitfall 6) reuses this idiom against `parent_order_id`.

**Parameterized Core write (atomic multi-row)** ‚Äî copy `itrader/results/sql_storage.py:146-176` (`save_run`): build row dicts, `with self.engine.begin() as conn: conn.execute(insert(table), rows)`. For `add_order` with a bracket, insert the parent row before child rows (RESEARCH Pitfall 6 ‚Äî FK ordering).

**Enum text round-trip (D-07)** ‚Äî RESEARCH ¬ßCode Examples (lines 696-702): write `order.status.value`; read back via `OrderStatus(row["status"])` or `order_status_map[...]` (the maps live in `itrader/core/enums/`). Applies to `type`/`status`/`action`/`trail_type`.

**ABC surface** ‚Äî `itrader/order_handler/base.py:14-273` defines the ~15 abstract methods (`add_order`, `remove_order`, `get_pending_orders`, `get_order_by_id`, `update_order`, `get_orders_by_status`, `get_active_orders`, `get_orders_by_time_range`, `get_order_history`, `search_orders`, `count_orders_by_status`, ‚Ä¶). ### `itrader/portfolio_handler/storage/sql_storage.py` (storage backend, CRUD ‚Äî bound `portfolio_id`)

**Analog:** `itrader/results/sql_storage.py` constructor pattern, with the **Pitfall-1 divergence**.

**Bind `portfolio_id` at construction (CRITICAL ‚Äî Pitfall 1):** The `PortfolioStateStorage` ABC (`itrader/portfolio_handler/base.py:14-389`) has **NO `portfolio_id` parameter on any of its ~21 methods** ‚Äî the in-memory backend is one-instance-per-`Portfolio`. The SQL backend MUST replicate that scoping:
```python
class SqlPortfolioStateStorage(PortfolioStateStorage):
    def __init__(self, backend: SqlBackend, portfolio_id: uuid.UUID) -> None:
        self.backend = backend
        self.engine = backend.engine
        self._portfolio_id = portfolio_id
        tables = build_portfolio_tables(backend.metadata)
        # ... assign self.positions, self.transactions, etc.
        backend.metadata.create_all(self.engine, checkfirst=True)
        self.logger = get_itrader_logger().bind(component="SqlPortfolioStateStorage")
```
Every SELECT carries `.where(table.c.portfolio_id == self._portfolio_id)`; every INSERT injects `self._portfolio_id` (including `CashOperation` / `PortfolioSnapshot` rows, whose objects carry no `portfolio_id` field). A method without a `portfolio_id` filter is the warning sign (Pitfall 1).

**Append-only ordering (Pitfall 7):** `get_snapshots()` / `get_transaction_history()` / `get_cash_operations()` MUST `ORDER BY` a stable key (the `seq` for snapshots, or `(portfolio_id, time)` + tiebreak) ‚Äî Postgres has no implicit insertion order. Mirror the `order_by(...).limit(n)` shape at `itrader/results/sql_storage.py:265-269`.

**All other patterns** (composition, `create_all`, `dispose` delegation, `bindparam` reads, `engine.begin()` writes, enum text, Decimal‚ÜîNumeric) are identical to `SqlOrderStorage` above.

## Shared Patterns

### Composition over inheritance (Phase-1 D-01)
**Source:** `itrader/storage/backend.py:18-44` (`SqlBackend`) + `itrader/results/sql_storage.py:83-86`.
**Apply to:** all three `Sql<Concern>Storage` classes. Each takes a `SqlBackend` constructor arg (has-a), registers its tables on `backend.metadata`, sources `self.engine = backend.engine`. NEVER a shared `SqlStorageBase` (deliberately absent ‚Äî `backend.py:4-9`).

### Spine type vocabulary (SPINE-03)
**Source:** `itrader/storage/types.py:34` (`Uuid`/`UuidType`), `:37-64` (`UtcIsoText`), `:67-69` (`json_variant`); re-exported from `itrader/storage/__init__.py:10-19`.
**Apply to:** every `models.py`. `Uuid(as_uuid=True)` for EVERY id column; `UtcIsoText` for EVERY datetime column (rejects naive datetimes ‚Äî `types.py:52-58`); `json_variant()` for the one `signals.config` dict column. Money is `sqlalchemy.Numeric` (unbounded) imported direct ‚Äî there is NO money TypeDecorator on the spine (`types.py:18-21`).

### Parameterized Core, never f-string SQL (SEC-01 / V5.3)
**Source:** `itrader/results/sql_storage.py:218-222` (read), `:171-175` (write).
**Apply to:** every query in all three backends. Constant `Table`/`Column` objects + `bindparam`; reads via `.mappings()`; writes inside `with self.engine.begin() as conn:`.

### Idempotent table registrar
**Source:** `itrader/results/models.py:37-112` (`build_results_tables`, the `if name in metadata.tables` guard).
**Apply to:** every `models.py` ‚Äî `build_order_tables` / `build_portfolio_tables` / `build_signal_tables`. Importable by `env.py` for autogenerate (same function reused both code paths ‚Äî D-09 single source of truth).

### Quarantine ‚Äî keep the backtest import path SQL-free (GATE-01)
**Source:** `itrader/results/sql_storage.py:22-25` + `itrader/storage/__init__.py:1-8` (the `SqlResultsStore` is deliberately NOT re-exported from `itrader/results/__init__.py`).
**Apply to:** all three new modules ‚Äî do NOT re-export `Sql<Concern>Storage` from any package `__init__.py`. The factory imports them lazily inside the `'live'` arm (the pattern already at `order_handler/storage/storage_factory.py:50-51`). The `'backtest'`/`'test'` arm and the in-memory backends import zero SQLAlchemy symbols ‚Äî keep it so (oracle inertness: 134 / `46189.87730727451`).

### Engine lifecycle delegation (WR-03)
**Source:** `itrader/storage/backend.py:36-44` + `itrader/results/sql_storage.py:108-110`.
**Apply to:** every `Sql<Concern>Storage.dispose()` ‚Äî call `self.backend.dispose()`, NEVER `self.engine.dispose()` directly (one concern's shutdown must not flush a shared pool). Critical under `filterwarnings=["error"]` ‚Äî undisposed engines trip a GC `ResourceWarning` ‚Üí test failure (Pitfall 4).

### Bound logger
**Source:** `itrader/results/sql_storage.py:106`.
**Apply to:** each backend ‚Äî `self.logger = get_itrader_logger().bind(component="Sql<Concern>Storage")`.
