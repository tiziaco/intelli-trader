## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Test-expectation correction] Repr-exact literals pinned to the live quantize scale**

- **Found during:** Task 1 (the repr-exact margin/cash assertions).
- **Issue:** The plan mandates repr-exact `str(x) == str(expected)` assertions (never float `==`). The naive literals (e.g. `Decimal("1000")`) did not repr-match the live readings: the un-quantized lock / `aggregate_notional` / `avg_price` carry the CSV price repr (`100.0` → `1000.0` / `2000.0`), cash-scale fields (`balance`/`available`/`equity`) repr at 2dp (`.00`), and after the close-path settlement the position internals re-quantize (`qty` → `10.0`, `aggregate_notional`/`locked` → `.00`).
- **Fix:** Pinned each expected literal to the observed live repr (probed via an instrumented dry-run), keeping the hand-computed VALUE identical and documenting the scale inline. This is a correctness correction to the test's expectation, not a code change — the existing `short_roundtrip`/`partial_cover` leaves sidestepped it by using value-equality (`==`); this plan's repr-exact requirement surfaces the scale explicitly.
- **Files modified:** `tests/e2e/short_scale_in/test_short_scale_in_scenario.py`, `tests/e2e/short_scale_in_partial_cover/test_short_scale_in_partial_cover_scenario.py`
- **Verification:** both scenarios PASS; values match the inline hand computation.
- **Committed in:** `1b0219f` (Task 1 commit)
