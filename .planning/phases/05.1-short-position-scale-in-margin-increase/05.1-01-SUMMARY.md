# Phase 05.1 Plan 01: Short Position Scale-In (Admission Gate Lift) Summary

Lifted the unconditional short-increase admission rejection behind the existing
`allow_increase` flag — a byte-symmetric mirror of the long INCREASE gate — so a same-side
SELL add against an open SHORT is now ADMITTED (when `allow_increase=True`) and falls
through to the existing direction-agnostic `resolve_entry` sizing + check-and-reserve gate,
with zero new sizing function and zero new settlement branch. Proven both ways (admit/reject)
plus the D-06 reserve-side correctness, and the CR-02 over-cover guard is regression-locked
for the SHORT side.

## What Was Built

- **Task 1 (`feat`, c790165):** The short-add site (`admission_manager.py:537-556`) is now
  conditional on `not signal_event.allow_increase`, mirroring the long gate at `:577-591`.
  `allow_increase=False` → audited `ADMISSION_INCREASE` rejection with the mirrored "position
  increase not allowed by strategy (allow_increase=False)" reason text; `allow_increase=True`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Plan-assumption correction] D-06 reserve-side assertion rewritten to match live code**

- **Found during:** Task 2 (the admit-arm D-06 reserve assertion).
- **Issue:** The plan (PATTERNS Analog 2 / truth 3) assumed the SELL-add reserves
  `notional/leverage` at the admission check-and-reserve gate, mirroring the long reserve
  formula `available_cash == remaining - price * expected`. Against the live code this is
  false: `admission_manager.py:259-264` (D-03/T-05-15) reserves ONLY for the cash-DEBITING
  primary (`primary.action is Side.BUY`) — "SELLs and bracket SL/TP children are exempt (no
  OCO double-reservation)". A short SELL CREDITS cash; it books no admission reservation. Its
  margin LOCK is recomputed at SETTLEMENT (`portfolio.py:423-441`), which the plan itself
  assigns to Plan 05.1-02 ("Plan 02 owns the settlement-path portion of SCALE-02").
- **Fix:** Rewrote the admit-arm D-06 assertion to the verified, non-vacuous fact:
  `available_cash` is UNCHANGED across the admit (`reserved_available == remaining`). This
  pins that the SELL-add neither over-reserves the SELL-side notional nor debits the credited
  short proceeds at admission — the honest SELL-side mirror of the long BUY's `price * qty`
  reservation (a BUY reserves because it debits; a SELL reserves nothing because it credits).
  Documented the finding inline with the `admission_manager.py:264` citation. Also corrected
  the pre-condition assertion from the long `remaining < 10000` to the short `remaining >
  10000` (SELL proceeds raise cash in the margin-off harness).
- **Files modified:** `tests/unit/order/test_admission_rules.py`
- **Commit:** 780dfb1

This is a correctness correction to the test's expectation, not a code change — the admission
gate (Task 1) and the guard (Task 3) match their plan acceptance criteria exactly. The
divergence is consistent with the plan's own SCALE-02 split (admission-gate portion here,
settlement-path portion in Plan 05.1-02); Plan 05.1-02 should assert the settlement-side
margin lock for the short add.
