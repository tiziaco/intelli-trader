<objective>
Add the `is_deleted` flag and swap the `portfolios` plain `UNIQUE(venue_name, account_id)` constraint
for a **partial** unique index `WHERE NOT is_deleted`, in the SQLAlchemy registrar **and** in a new
Alembic revision.

`enabled` becomes *paused* (keeps its account, always re-enablable — ACCT-10's subject) and
`is_deleted` becomes *retired* (releases the pair). This revises
`portfolio_definition_store.py:23`'s documented *"PLAIN (never partial, conditional or deferrable)"*
note — whose rationale is about **simultaneous** sharing conflating buying power, which the partial
index still refuses; only **sequential** reuse becomes possible.

Output: one migration revision, an amended registrar, and a registrar test that proves the partial
index behaves on SQLite.
