<objective>
Make `fee_model_provider` `(venue, account_id)`-aware across its four files, and replace the false
comment in the live root with the truth.

Purpose: today the provider is a **zero-arg closure** in `compose.py` that captures
`execution_handler.exchanges.get((COMPUTE_VENUE, DEFAULT_ACCOUNT_ID))` at wiring time — the **paper**
exchange, always. The live root's comment explains the resulting `None` by claiming *"the live OKX
exchange exposes no `fee_model` attribute"*, which is false: the provider never resolves an OKX
exchange. The comment made a venue-blind provider look like a venue-aware one that happened to find
nothing, which is why the gap survived review.

code does.
