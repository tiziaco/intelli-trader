<objective>
Split `add_portfolio`'s durable write from its runtime build, stop rehydrate from re-entering
creation, delete the two guards that existed only to make that re-entry safe, and correct the
flagship multi-portfolio fixture that becomes illegal the moment ACCT-03 lands.

Purpose: the source defect is that `add_portfolio` does two unrelated jobs, and
`rehydrate_portfolios` calls it — so restoring and creating already share one method. The visible
scar is `_persist_definition`'s **"GATED ON ABSENCE"** guard, which exists ONLY so rehydrate
re-entering creation does not wipe the row it just read. Splitting the two verbs deletes the gate,
closes **ACCT-07 structurally** rather than by patching the early return, and makes **ACCT-05
trivially true** — with exactly one creation path there is no path that can forget to attach an
account.

Output: `create` / `materialize` as separate verbs, a rehydrate that only materializes, a
`_persist_definition` with no null-account escape hatch, and a fixture that names two real accounts.
