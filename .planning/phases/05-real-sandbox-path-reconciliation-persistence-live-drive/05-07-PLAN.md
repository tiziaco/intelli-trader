<objective>
Implement two-sided restart rehydration (RECON-05/D-03/D-05): the store side already exists
(`CachedSql*.rehydrate()` — INTENT truth); add the venue side. On startup, before RUNNING, take the
REST venue snapshot, reconcile it against the rehydrated working set, adopt in-band deltas as
reconciling FillEvents driven through the idempotent fill path, halt-and-alert on venue positions with
no stored intent, and re-adopt brackets by re-linking parent/child (per-bracket halt on an unconfident
leg). Persist the venue order id on the order mirror so legs re-link confidently.

Purpose: close the broker-side gap the v1.6 store-only tests never covered — after downtime, the store
and the venue can disagree, and the engine must resolve that safely before it trades again.
Output: `venue_reconciler.py` + venue-order-id persistence + root invocation + two integration suites.
</objective>
