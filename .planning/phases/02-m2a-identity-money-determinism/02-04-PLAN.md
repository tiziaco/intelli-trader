<objective>
Retype the five domain entities to UUID ids + Decimal money and remove the float↔Decimal cash
round-trip (M2-01 entity fields + M2-02). Each entity owns its money state; this plan types
`order.py`, `transaction.py`, `position.py`, `portfolio.py` money fields as `Decimal` and id fields
as `uuid.UUID`/NewType aliases, and removes the `portfolio.cash += float(transaction_cost)` round-trip
at `transaction_manager.py:229` (the defect #17 targets) while keeping the existing `Decimal(str(...))`
entry pattern. This plan does NOT route cash through `CashManager` (that is M4 #22) and does NOT
touch order/transaction audit timestamps (M2b SC2) — it only types fields Decimal and kills the
round-trip.

Note on ordering: depends on Plan 03 because Plan 03's idgen swap makes `order.id` a runtime UUID
(so the storage UUID-keying in Plan 03 and the entity `id: UUID` annotation here stay consistent),
and on Plan 02 for `core/ids.py` + `core/money.py`. No file overlap with Plan 03.

Purpose: Money is Decimal end-to-end on the cash path with no float round-trips (M2-02).
Output: Decimal/UUID-typed entities; the cash round-trip removed; a Decimal-money unit test.
</objective>
