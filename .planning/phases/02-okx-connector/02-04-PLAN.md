<objective>
Build the data arm: `OkxDataProvider`, an independent data client that streams OKX candles over a
native `/ws/v5/business` socket (the only place the `confirm` closed-bar flag survives), gates on
`confirm=="1"`, and provides REST `fetch_ohlcv` backfill — all through the injected connector session.

Purpose: This is CONN-01, the phase's confirm-flag escape hatch and the single most likely source of
paper-parity failure if done wrong (ccxt's unified `watch_ohlcv` drops `confirm`). The native socket's
host MUST be driven from the connector's `sandbox` bool (wspap vs live) — the `x-simulated-trading`
header does nothing on WS. The provider hands only completed bars to the Phase-3 `LiveBarFeed`.

Output: `itrader/price_handler/providers/okx_provider.py` (4-SPACE) + unit tests proving the
confirm gate, the sandbox host selection, and the Decimal-edge backfill.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OKX venue -> OkxDataProvider | untrusted candle payloads (variable field counts, floats-as-strings) cross here |
| sandbox bool -> native WS host | wrong host = streaming from the LIVE venue while believing it is demo |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-MISROUTE | Tampering / Elevation | native /business socket host | mitigate | host driven from connector.sandbox (wspap vs ws); test asserts wspap when sandbox=True — the x-simulated-trading header is NOT relied on for WS (Pitfall 2, HIGH severity) |
| T-02-04-CONFIRM | Tampering (data integrity) | confirm gate | mitigate | gate on row[8]=="1"; forming bars dropped — a missed gate silently corrupts paper-parity (Pitfall 1) |
| T-02-04-FLOAT | Tampering (data integrity) | candle + backfill Decimal edge | mitigate | to_money(str(x)) on every field; no astype(float)/Decimal(float) (CONN-05, Pitfall 5) |
| T-02-04-MALFORMED | Input Validation | raw candle row | mitigate | validate row length >= 9 before indexing confirm (V5); malformed rows skipped-and-logged |
| T-02-04-SESSION | Denial of Service (self-inflicted) | aiohttp session lifecycle | mitigate | close session on stop; tests never open real sockets; respect connection budget (no resubscribe storm) |
</threat_model>
