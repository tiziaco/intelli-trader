- **WR-02** — enable SQLite foreign-key enforcement at the backend (SqlEngine), dialect-guarded,
  so declared FKs are honored everywhere the engine runs (test-SQLite, the runtime results-store
  SQLite, and the Turso slot), matching Postgres. Prove it with an orphan-insert-raises test.

  the durable operational schema is Alembic-owned end-to-end (no un-stamped-table drift on live
  Postgres). Schema for tests moves to a shared `provision_schema` fixture. The EPHEMERAL
  `results/sql_storage.py` store is EXCLUDED — it stays create_all-owned per D-14.

Purpose: close the review's robustness/portability gaps (schema-drift hazard, unenforced
cross-backend FK integrity, nondeterministic rehydrate order) without disturbing the byte-exact
oracle, the OKX import-inertness gate, or the create_all-vs-migration parity gate.
Output: a schema-pure durable-store spine, backend-level SQLite FK enforcement, a deterministic
rehydrate, a shared test provisioning helper, and two new proving tests.
