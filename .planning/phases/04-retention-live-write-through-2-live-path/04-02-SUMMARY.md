# Phase 4 Plan 02: Live Portfolio-State Write-Through Wrapper Summary

Built `CachedSqlPortfolioStateStorage` — the live-only store-first decorator that composes the untouched Phase-3 `SqlPortfolioStateStorage` with an in-memory working set, persists the two purge-derived accumulators to a dedicated `portfolio_account_state` table (A2), and rehydrates open-only on restart (D-03) — wired into the factory `'live'` arm but with the composition root left at `"backtest"` (D-01).

## What was built

- **`portfolio_account_state` table (A2)** — a single-row-per-portfolio carrier (`portfolio_id` PK, `cash_balance`, `realized_pnl`, `total_equity`, `peak_equity`, `open_positions_count`, `updated_time`) registered on the shared operational MetaData via `build_portfolio_tables`. The untouched `SqlPortfolioStateStorage` (D-04) simply ignores it.
- **Framework-chain Alembic migration `47f2b41f3ffe`** — autogenerated against a testcontainers Postgres at baseline head; `down_revision = "2cbf0bf6b0b6"`, `create_table('portfolio_account_state')` only (no churn on existing tables; `alembic check` reports no drift). Hand-added `import itrader.storage.types` so the `updated_time UtcIsoText` FQN resolves at `upgrade head` (Pitfall 5 / D-09 — identical gotcha to the baseline).
- **`CachedSqlPortfolioStateStorage`** — implements all 21 `PortfolioStateStorage` methods store-first (Pitfall 8) with the read-through split (D-02): open/current reads cache-only, history reads (closed positions / transactions / cash-ops / snapshots) read through to the store. Adds `save_account_state` (synchronous parameterized delete-then-insert upsert), `load_account_state` (scoped SELECT → dict | None), and `rehydrate()` (open positions + reservations + locked margin + account scalars, never closed/transactions/cash-ops). Bound `portfolio_id` scopes every account-state + rehydration read (Pitfall 1 / V4).
- **Factory `'live'` arm** — now returns `CachedSqlPortfolioStateStorage(SqlPortfolioStateStorage(sql_backend, portfolio_id), max_snapshots=max_snapshots)`; the wrapper import stays inside the `'live'` branch (GATE-01 quarantine — the backtest path stays SQL-free).
