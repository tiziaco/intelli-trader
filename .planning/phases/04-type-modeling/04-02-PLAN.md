<objective>
Harden the config/events/execution boundary (TYPE-02, TYPE-03) without touching the order-domain core:
D-05 introduce a class-based `ErrorSeverity` enum; D-08 convert fee/slippage dispatch to enum-member
comparison with `assert_never`; D-09 validate `rebalance_frequency` at the Pydantic boundary; D-10

D-11); D-12 retype the portfolio/events/validators entity-id annotations to `core/ids` NewTypes (with the
D-13 `user_id: int` carve-out).

Purpose: close stringly-typed / `int` / `Any` boundary defects with the canonical enum/NewType forms.
These sites are oracle-dark (the fee/slippage oracle runs Zero* models; rebalance is
`auto_rebalance=False`; id retypes are annotation-only) so behavior is byte-identical.
Output: one new enum module + 7 edited modules; oracle byte-exact, mypy --strict clean, e2e 58/58.
</objective>
