## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| connector asyncio loop ↔ engine thread | Reconnect resume splits across two threads; the ring writer is loop-native only |
| venue REST (snapshot / catch-up fills) → engine | Engine-thread I/O during resume reads venue truth |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-03 | Tampering | LiveBarFeed ring writer | high | mitigate | CF-2: backfill loop-native + assertion that no engine-thread path reaches the ring writer (concurrent writers corrupt the bar stream) |
| T-07-12 | Availability | resume gate | medium | mitigate | D-12: on snapshot/catch-up failure stay paused, retry on next stream-up — never resume blind |
| T-07-13 | Business Logic | all-streams-healthy gate | medium | mitigate | Resume only when EVERY wired arm reports healthy (WR-03/D-28); a single arm's reconnect cannot resume while another is down |
| T-07-01 | Information Disclosure | resume-failure log | medium | mitigate | Log message is fixed-shape; no venue payload/secret leaked (V7) |
| T-07-SC | Tampering | package installs | low | accept | Zero installs; backtest-dark |
