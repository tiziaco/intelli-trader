<objective>
Drop the defensive per-call `.copy()` from all five `InMemoryPortfolioStateStorage`
getters under the D-19 single-writer contract, rewrite the `PortfolioStateStorage`
ABC docstrings to a read-only-view contract, and replace the never-firing per-tick
snapshot-trim whole-list copy with count-only / last-only accessors. Lock each change
with a deterministic behavioral assert (D-01) — written together with the production
edit so the assert PASSES against the landed change (TDD_MODE is OFF; these are
regression-LOCKS, not expected-red test-first cycles).

Purpose: PERF-01 — kill the per-tick container copies on the portfolio read seam. The
copy never protected correctness (single writer, D-19); it only added per-tick cost.
Output: copy-free getters, two new storage accessors, the consuming metrics-manager
rewrite, and behavioral-assert regression locks that are GREEN at task completion.

Explicitly NOT in scope (D-04): do NOT add any `*_snapshot()` copy-returning twin. A
query-based live/Postgres backend is copy-safe for free; the hedge is speculative API
for deferred work and violates the "no pre-building for deferred work" discipline. This
is a bounded, owner-flagged gap-discovery delta — record it in the SUMMARY, do not
silently fold it.
