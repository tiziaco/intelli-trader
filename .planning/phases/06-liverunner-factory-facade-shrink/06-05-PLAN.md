<objective>
Extract the live session-wiring out of the God object into two focused collaborators (RUN-05 + RUN-04 live-reuse + D-12):

- `LiveRouteRegistrar` (`route_registrar.py`) — ONE central declarative route table (mirrors the single `EventHandler._routes` literal where list order IS execution order), referencing the built handlers'/`UniverseHandler`'s methods, installed into the single `EventHandler` ONCE, no subclass, no runtime mutation (LR-16 / D-10). It registers the BUSINESS/live route set ONLY: `UNIVERSE_POLL`, `UNIVERSE_UPDATE`, `STRATEGY_COMMAND`, `BARS_LOADED`, `BARS_LOAD_FAILED`, `FILL` (appended). The CONTROL routes stay OUT of P6 (their P7/P9 consumers don't exist — D-23).
- `SessionInitializer` (`session_initializer.py`) — the distinct class (D-12) owning the live session wiring: `wire_universe(engine)` (the RUN-04 shared helper — live now GAINS the WR-03 desync assert) → `register_strategy_warmup(feed, strategies)` (the RUN-07 seam) → build + wire the now-first-class `UniverseHandler` (the D-11 shape: `set_venue_metadata` unconditionally over the uniformly-resolved venue exchange, the 4 read-model setters, the interim `set_freeze_gate` callable → facade `_is_halted`/`_is_submission_paused`) → compose routes via `LiveRouteRegistrar`.

This plan is behavior-preserving-interim: `SessionInitializer` is invoked through the existing `_initialize_live_session` delegation (still called at `start()`/`run_paper_replay`), so the suite stays green. The FLIP to construction-time invocation (D-12) + the LiveRunner wiring land in 06-06 with `build_live_system`.

Per D-04 this plan does NOT touch the facade's safety/reconcile/stream method bodies; the freeze-gate + the WR-03/membership guards are wired, not reimplemented.

Purpose: collapse the ~175-line `_initialize_live_session` into a testable-in-isolation `SessionInitializer` over the shared `wire_universe`, and make the correctness-load-bearing cross-handler route ordering greppable in ONE place.
Output: `LiveRouteRegistrar` + `SessionInitializer`; `_initialize_live_session` delegates to them; the old `_LiveWarmupConsumer` is removed from LTS; inertness + paper-parity + mypy green.
</objective>
