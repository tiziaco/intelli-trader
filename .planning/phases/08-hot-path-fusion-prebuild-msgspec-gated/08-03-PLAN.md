<objective>
Two independent deterministic wins on independent files (no overlap → parallel-safe):

- **Req 3 (mechanical):** Replace `bar_feed.py:255-258`'s `for ts, row in frame.iterrows()` prebuild
  (materializes ~69k throwaway pandas Series) with `itertuples(index=True)` / column-array zips,
  preserving the `Bar.from_row` D-14 `Decimal(str(...))` string path byte-for-byte. CRITICAL pin:
  `itertuples` yields numpy/native scalars, so `str(np.float64(x))` MUST produce the SAME string as
  `str(series_value)` did under `iterrows` — verify `str()` parity; if it diverges, feed `Bar.from_row`
  a Series-shaped access path. There is NO existing `itertuples` usage in the repo (one net-new
  mechanical pattern; the `from_row` contract is the constraint).

- **Req 4 (D-06):** Cache the serialized STATIC portion of `Strategy.to_dict` (:639-705) PER INSTANCE
  (per-class would leak one instance's declared windows into another — a correctness bug). Build
  lazily on first `to_dict`, stash on `self`; per call refresh ONLY the two genuinely runtime-mutable
  fields — `is_active` (:856/:859) and `subscribed_portfolios` (:846/:853) — overwriting them IN PLACE
  so key ordering is unchanged (byte-identical). Ship a documented `_invalidate_to_dict_cache()` seam
  (never called in backtest — no declared-param setter exists in Phase 8).

Purpose: Kill ~69k throwaway Series + the per-signal `_declared_hints`/`_json_safe` re-walk, zero
numeric change. Output: itertuples prebuild + field-for-field equivalence test (Req 3); per-instance
static cache + invalidation seam + snapshot-drift test (Req 4); oracle byte-exact.
</objective>
