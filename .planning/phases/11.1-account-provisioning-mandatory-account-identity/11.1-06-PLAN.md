and the inversion is deliberate. The owner settled two constraints that pull against each other:
the registry key is retired in full with no transitional alias, AND D-06/D-08 must land in one wave
because landing D-06 alone makes the commission reach-in return nothing and degrade to a value that
happens to equal the golden one. Renaming FIRST satisfies both: the key `('paper', default)` already
exists and resolves when plan 11.1-07 changes *who builds* the object behind it, so the
green-and-wrong intermediate state never exists in the tree.

Purpose: one venue name, one registry key, one object — with every reader converted in the same
commit.

Output: a single-entry exchange registry; a default-deny allowlist that admits the new name; a
fail-loud universe lookup; both construction arms and the oracle generator renamed; ~30 test/script
files migrated; the alias-identity test inverted.
