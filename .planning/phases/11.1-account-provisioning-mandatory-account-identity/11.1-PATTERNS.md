**Analog:** `itrader/connectors/provider.py` — copy the memo shape *exactly* (RESEARCH § Don't Hand-Roll:

**Module-header + import pattern** (`connectors/provider.py:31-38`) — `from __future__ import annotations`
plus a `TYPE_CHECKING`-only block is what keeps the module import-inert (GATE-01):

```python

if TYPE_CHECKING:
    from itrader.venues.bundle import VenueBundle
    from itrader.venues.registry import ExecutionVenueRegistry
```

*Note:* `provider.py` imports `get_itrader_logger` at runtime — that is fine (already on the inert path,
see `venues/assemble.py`), but keep the annotation imports guarded.

**The memo — copy verbatim in shape** (`connectors/provider.py:67-80`):

```python
    def __init__(self, plugins: dict[str, ConnectorPlugin]) -> None:
        self._plugins = plugins
        self._memo: dict[tuple[str, str], LiveConnector] = {}
        self.logger = get_itrader_logger().bind(component="ConnectorProvider")

    def get(self, venue: str, account_id: str, spec: Any) -> LiveConnector:
        """Return the shared connector for ``(venue, account_id)``; build it once on first call.

        Fails loud with ``KeyError`` when ``venue`` has no registered plugin.
        """
        key = (venue, account_id)
        if key not in self._memo:
            self._memo[key] = self._plugins[venue].build(spec)
        return self._memo[key]
```

`VenueBundles` is the same shape one layer up — `self._memo[key] = self._registry.get(venue).build_bundle(ctx, spec, connectors)`.
Fail-loud on an unknown venue falls out of `ExecutionVenueRegistry.get`'s bare `self._plugins[name]`
(`venues/registry.py:43-45`) — **do not** wrap it in `.get(...) or default`.

**The `"default"` account normalization** — reuse the existing spelling, do not invent a fourth
(`venues/assemble.py:56` declares it locally, `okx_plugin.py:295` / `:351` apply it):

```python

# ``build_bundle`` (and ``venue_uid_guard`` mirrors). Declared locally rather than
# imported from ``execution_handler`` so this module keeps its zero-dependency

_DEFAULT_ACCOUNT_ID = "default"
```

**Barrel exclusion (load-bearing):** `venues/__init__.py:1-24` re-exports only `bundle` + `registry`
and its docstring is the GATE-01 acceptance evidence — *"imports NO concretion … so importing
`itrader.venues` pulls nothing heavy."* `bundles.py` must be imported **directly**
(`from itrader.venues.bundles import VenueBundles`) and **must not** be added to `__all__`. The exact
precedent for "lives in the package, deliberately not in the barrel" is `connectors/provider.py:22-26`:

## Shared Patterns

### 1. Lazy concretion inside `build*` bodies — the GATE-01 mechanism
**Source:** `venues/paper_plugin.py:88-92`, `venues/okx_plugin.py:279-281`, `:344-345`
**Apply to:** every `build_bundle` / `build_provider` / plugin body that touches a concretion.
**Rule:** the *only* sanctioned exception to the project's "no lazy imports, no late init" preference.
Registration is inert (stores an object); the heavy import happens when `build` runs. Enforced by
`tests/integration/test_okx_inertness.py`.

### 2. `TYPE_CHECKING` forward-refs for heavy annotations
**Source:** `trading_system/engine_context.py:45-57`, `connectors/provider.py:37-38`,
`venues/registry.py:23-26`
**Apply to:** `venues/bundles.py`, `core/commission_estimator.py` (F-7).
**Rule:** `from __future__ import annotations` *or* string forward-refs, plus a `TYPE_CHECKING` block.
Keeps runtime purity AND `mypy --strict` narrowing.

### 3. Fail-loud lookup, never `.get()` + guard
**Source:** `venues/registry.py:43-45` (`return self._plugins[name]`)
**Counter-example to fix:** `trading_system/universe_wiring.py:97-100`
**Apply to:** every new lookup, and every converted one on the oracle path.

### 4. Memoized provider over a `(venue, account_id)` key
**Source:** `connectors/provider.py:67-80`
**Apply to:** `VenueBundles`. Copy the shape *exactly* — divergence re-opens the double-`OkxExchange`
fill-stream bug D-08 exists to prevent.
