<objective>
Land **D-18 (VENUE-08)**, alone and last, with the byte-exact oracle as its primary acceptance
criterion: the commission estimator is DECOMPOSED, not merely re-sourced.

`FeeModelCommissionEstimator` (`compose.py:57-81`) does two unrelated jobs. It late-resolves the
exchange's current fee model — a wiring concern — and it applies the admission convention
`side="buy", order_type="market"` — an admission-policy concern. D-18 splits them: the `core/` seam
narrows to a fee-model provider, and the convention moves into `AdmissionManager`, which owns it.

Two facts make this plan delicate, and both are settled here rather than left to the executor.
**RESEARCH F-7:** a `() -> FeeModel` Protocol cannot import `FeeModel` at runtime from `core/`,
whose zero-dependency rule is pinned in its own docstring and in CLAUDE.md — the shape is the
`TYPE_CHECKING`-guarded forward-ref already used at `engine_context.py:45-57`. **RESEARCH F-8:** a
five-test conformance file imports the class being deleted, including `test_late_binding`, which
guards the exact property this decision calls load-bearing.

And one trap governs the verification: on this seam `Decimal("0")` is BOTH the correct golden value
and the value a broken seam returns. Plan 11.1-02 installed a NON-ZERO-fee reservation test ahead of
this change precisely so value-identity can be proven rather than inferred.

Purpose: one owner per concern, with late binding intact and the reservation value provably
unchanged.

Output: a narrowed `core/` Protocol; a deleted adapter; an `AdmissionManager` that owns the
convention; three retyped call sites; a rewritten conformance file that keeps its late-binding test.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| execution config -> admission reservation | `update_config` can change the venue's fee model at runtime. The reservation computed at admission must reflect the rate the exchange will actually charge. |
| order domain -> execution domain | The order domain reads an execution-side fee estimate through an injected Protocol rather than importing across the boundary. `core/` must not gain a runtime dependency on the execution package. |
| estimator return -> money ledger | The estimate becomes part of the locked cash amount. A float here imports binary-float-repr error into a correctness-critical value. |
