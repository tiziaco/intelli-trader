# Phase 11.1 Plan 10: Decompose the Commission Estimator (D-18) Summary

Split the commission-estimate adapter into its two real concerns — a `core/` fee-model
PROVIDER for wiring and the buy/market admission convention owned by `AdmissionManager` —
with the reservation value proven byte-identical across the move by exact `Decimal`-repr
comparison, and with the late-binding property regained at the WIRING level after the
decomposition was empirically shown to have dropped it.

## What Was Built

### Task 1 — `core/` narrows to a fee-model provider (commit `a7ac28c2`)

`itrader/core/commission_estimator.py` now exposes `FeeModelProvider`, a
`@runtime_checkable` Protocol whose single member is `__call__(self) -> Optional["FeeModel"]`.
The `FeeModel` type rides a `TYPE_CHECKING`-guarded import plus a string forward-ref — the
RESEARCH F-7 shape, matching the sanctioned `engine_context.py:53-65` idiom — so the module
imports nothing from `itrader` at runtime while `mypy --strict` still narrows the return.

The module docstring's unqualified "imports nothing from ``itrader``" claim was corrected to
"AT RUNTIME" **in the same commit that made it conditional**, per the plan's explicit
requirement that the file must not be left self-contradicting.

`tests/unit/core/test_commission_estimator.py` was rewritten against the new shape (8 tests,
up from 7), keeping the standalone-function / `runtime_checkable`-isinstance style.

### Task 2 — the convention moves to `AdmissionManager`, the adapter dies (commit `b9a216ae`)

- The adapter class and the reach-in that built it are **deleted** from `compose.py`. What
  remains is a named `fee_model_provider()` closure that captures the **exchange** and reads
  `fee_model` **inside** the call, with a comment citing `simulated.py:775` as the reason.
- `AdmissionManager._estimate_commission` is now guard-clause shaped with **two separate**
  zero paths — `provider is None` and `fee_model is None` — and applies the
  `side="buy", order_type="market"` convention verbatim, still wrapped in the untouched
  WR-04 `to_money(...)` normalization (`to_money(` count in the file: **4 before, 4 after**).
- `OrderHandler`, `OrderManager` and `AdmissionManager` now name **one** type
  (`Optional[FeeModelProvider]`), resolving RESEARCH G7's three-way inconsistency.
- The now-unused `SimulatedExchange` and `Decimal` imports were removed from `compose.py`
  (verified nothing else in the file used them).

### Task 3 — value identity proven, and a real coverage hole closed (commit `91b4dd49`)

The guard file gained the two-way zero split and `test_provider_is_dereferenced_once_per_estimate`
(a call-counting provider proving nothing is memoized on `self`). A new
`tests/integration/test_compose_wiring.py::test_the_wired_fee_model_provider_is_late_bound`
drives the provider `compose_engine` **actually injects** — see Deviation 2 for why that test
had to exist.

## Deviations from Plan

### 1. [Rule 3 — blocking] `CommissionEstimator` deletion moved from Task 1 to Task 2

**Found during:** Task 1.
**Issue:** the plan has Task 1 delete `CommissionEstimator` and *also* requires Task 1 to end
with `mypy` green. Those are contradictory: `order_handler.py`, `order_manager.py` and
`admission_manager.py` all still import that symbol until Task 2 retypes them, so deleting it
in Task 1 leaves the tree red at that commit.
**Fix:** Task 1 added `FeeModelProvider` and marked `CommissionEstimator` RETIRED with an
explicit "deleted in the next commit of this plan" note; Task 2 deleted it together with the
three call sites. Every commit in this plan is independently green.
**Net effect:** none — `CommissionEstimator` does not exist at HEAD.

### 2. [Rule 2 — missing critical guard] The plan's fail-first prediction is FALSE, and the gap it hid is real

**Found during:** Task 3 fail-first proof.
**Plan claim:** "changing the `compose.py` provider to CAPTURE `exchange.fee_model` … makes
`test_late_binding` and `test_fee_model_swap_is_observed_on_the_next_call` FAIL while the
oracle stays GREEN. Record BOTH observations — that divergence is the entire justification for
these tests."
**Actual:** the oracle stays green (correct), but **neither test fails** — and nor does any
other test in the suite. The reason is structural, and it is a consequence of the decomposition
itself: pre-D-18 the late-binding test constructed `FeeModelCommissionEstimator`, i.e. the
production WIRING OBJECT, so a capturing change in `compose.py` reddened it (this is exactly
plan 11.1-02's probe C1). Post-D-18 the provider is a closure built inside `compose_engine`,
so both late-binding tests necessarily build their **own** provider and assert the **Protocol
contract**. The contract is guarded; the wiring is not. This is the same shape as 11.1-02's
recorded deviation ("probe B does not red test 5") and it means D-18, implemented exactly as
written, would have **silently removed its own load-bearing regression guard** — which the
plan's own prohibition forbids ("MUST NOT … replace it with a weaker assertion").
**Fix:** added `tests/integration/test_compose_wiring.py::test_the_wired_fee_model_provider_is_late_bound`,
which resolves the provider off
`engine.order_handler.order_manager.admission_manager.fee_model_provider`, swaps the exchange's
fee model through the real `update_config` mechanism, and asserts by **object identity** that
the next deref sees the swap. Probe 2b confirms it reds on the exact regression.
**Files modified:** `tests/integration/test_compose_wiring.py` (+42 lines).
**Commit:** `91b4dd49`.

records T-11.1-48 in the live path where it actually bites. The two remaining in-scope prose
mentions (in `compose.py` and `admission_manager.py`) were reworded to describe the deleted
class without naming it, so the criterion holds literally.
**Commit:** `b9a216ae`.

  refactor. Both code sites now say so.
- **The `grep -c "FeeModelCommissionEstimator" itrader/ -r == 0` criterion is now permanently
  satisfied**, which means future readers cannot find the deleted class by name in `itrader/`.
  It is findable via `git log -S FeeModelCommissionEstimator` and is named in the test-file
