<objective>
Relax the `LONG_ONLY` registration guard in `StrategiesHandler.add_strategy` so
`SHORT_ONLY` / `LONG_SHORT` strategies are admissible ã gated on `allow_short_selling`
AND `enable_margin` BOTH being on (SHORT-01 / D-07). With either flag off (the default),
registering a non-`LONG_ONLY` strategy still raises loudly. SMA_MACD is `LONG_ONLY` with
both flags off ã byte-exact (134 / `46189.87730727451`).

Purpose: `enable_margin` turns on the lock-and-settle model (Phase-2 D-09) ã the only model
that can represent a short (a short has no notional to "spend"; spot debit-notional cannot
express it). Coupling both flags gives fully-collateralized shorts (no leverage) by default
(`locked_margin = notional / 1`); levered shorts are a separate opt-in dial (D-07). The two
flags live on `TradingRules` (`config/portfolio.py`); `compose.py` already reads
`trading_rules` for the order handler ã thread the same two flags into `StrategiesHandler`.
Output: a two-flag-gated `add_strategy` + the construction-time wiring.
</objective>
