<objective>
Wire CARRY-01: each BAR, accrue `days × close × |size| × rate/365` on every open short and
debit realized cash incrementally via a `BORROW_INTEREST` `CashOperation` (D-02 / D-03). The
days basis derives from the bar's **business time** (`this_bar.time − last_accrual.time`),
NEVER `datetime.now(UTC)` (D-04 — determinism). `borrow_rate` is read per open short from the
injected `Universe` read-model (D-01); carry NEVER folds into `Position.realised_pnl`
(D-08 — one carry site, clean attribution). With `borrow_rate=Decimal("0")` / no open shorts,
nothing accrues and SMA_MACD stays byte-exact (134 / `46189.87730727451`).

Purpose: This is the PRIMARY new wiring and the phase's highest-risk task (RESEARCH Pitfall 1).
The current mark path (`update_portfolios_market_value` → `update_market_value_of_portfolio`)
discards everything except `bar.close` and stamps `datetime.now(UTC)` — it threads neither the
bar's business time (needed for the days basis) nor the `Universe` (needed for `borrow_rate`).
Both are available at the handler level (`self._universe`, set via `set_universe`); thread them
down. Carry books through the existing `CashManager._create_operation` ledger primitive — no new
mechanism. Carry must be visible in the equity curve as it accrues so the P4 liquidation trigger
sees carry-eroded equity (NOT lump-sum-at-close).
Output: a per-bar carry accrual debiting realized cash via `BORROW_INTEREST`, fully deterministic.
</objective>
