<objective>
Re-point the consumers onto the Account abstraction with the backtest oracle held byte-exact. This is
the byte-exact-critical consumer wiring: Portfolio constructs its account leaf by `enable_margin`
(D-03), delegates accounting to it (`Portfolio.cash` -> `account.balance`, ~12 settlement call sites
re-pointed receiver-only), and drops `user_id` (ACCT-04). PortfolioHandler re-points the
`PortfolioReadModel.reserve/release` seam to `.account.reserve/release` with its Protocol signature
FROZEN (D-06/D-07 — zero order-domain ripple), removes the margin/liq math now living on the account,
and keeps the liquidation emission shell (ACCT-02). The golden-master wiring ripple from the user_id
strip is resolved deliberately, and the now-dead CashManager is deleted — including re-pointing the one
PRODUCTION importer that deletion orphans (sql_storage.py's `CashOperation` import) to the new account/
barrel home.

This plan owns the PRODUCTION change + the single oracle integration test it directly edits
(test_backtest_oracle.py). The much larger TEST-consumer ripple — 26 files reaching
`portfolio.cash_manager.*`, the two files that instantiate `CashManager` directly, and the
`add_portfolio`/`PortfolioSpec` `user_id` call-sites — is deliberately split into two sibling Wave-4
plans that run AFTER this plan re-points the production home: **01-03b** (unit + integration tests) and
**01-03c** (e2e scenarios + harness). They are peers of each other (no file overlap) and both depend on
this plan; the terminal byte-exact + full-suite gate (01-05) runs only after all three complete.

Purpose: complete the extraction so all balance/margin truth flows through Account, behind the
unchanged PortfolioReadModel seam, with no math altered — only receivers re-pointed — and leave the
production tree internally consistent (no dangling cash_manager importer) the moment CashManager is
deleted.
Output: re-pointed portfolio.py + portfolio_handler.py, user_id stripped across production wiring +
the oracle test, sql_storage.py CashOperation import re-pointed, cash_manager deleted.
</objective>
