## 1. Executive Summary

**46 unique findings** after de-duplication (W1: 14 · W2: 13 · W3: 11 · W4: 10 + seed verdict
− 2 cross-workstream duplicates + 1 synthesis doc finding).

**Top themes:**

1. **The architecture is fundamentally sound.** All core invariants were explicitly verified
   clean: `core/` purity (zero intra-itrader imports), events-package purity, no cross-domain
   handler writes, D-18 (`OrderManager` has no queue access / no handler back-reference),
   reporting-layer purity, single `_routes` dispatch literal. The seed example
   (`TradingSystem._signal_store` / D-12) was adjudicated **justified-by-decision** — not drift.
2. **Defensive `.copy()` in the in-memory portfolio storage is the dominant perf cost.** The
   single-writer backtest (D-19) pays a full snapshot-list copy *per tick* for a trim check that
   never fires (W1-15) plus shallow dict copies on every position read (W1-01). Highest-payoff
   SAFE fixes in the review.
3. **Stringly-typed closed vocabularies and one inconsistent enum form.** `OrderStatus`/
   `OrderCommand` are functional-form auto-int enums while every other enum is class-based with
   string values (W2-01); `operation_type`, `triggered_by`, and `ErrorEvent.severity` carry
   closed vocabularies as free-form `str` (W2-05/06/07).
4. **Two locked-decision violations, both S-effort:** `float`-typed money parameters on the
   `modify_order`/`cancel_order` public API (W4-01) plus a real latent `Decimal < float`
   `TypeError` on the exchange validation path (W2-10); and `uuid.uuid4()` as a second ID scheme
   for correlation IDs (W4-08 ≡ W1-06).
5. **One encapsulation gap leaks into production code:** `execution_handler.py:109` mutates
   `SimulatedExchange._supported_symbols` directly because no public API exists; the e2e conftest
   does the same and documents the fragility (W3-04).
6. **Six drifts are correctly owned by backlog item 999.5-(b)** (composition/config interface)
   and must NOT be fixed standalone: W4-02/03/05/06/07 and the `create_order` bypass W4-09
   (999.5-(d)). Coordinate, don't duplicate.

**Risk distribution:** 36 SAFE · 7 BEHAVIOR-SENSITIVE · 3 FRAGILE (W1-11, W2-02 — both on the
`order_manager.py` reconcile/admission path — and any future touch of that path per CONCERNS.md).

## 2. Type-Modeling Decision Rubric (proposed team standard)

Derived from the codebase's own exemplar patterns — adopt as the written standard:

| Use… | When… | Exemplars |
|---|---|---|
| `@dataclass(frozen=True, slots=True, kw_only=True)` | The object is an **immutable fact or value object** — constructed complete, no lifecycle mutation, no behavior beyond factory classmethods. Anything placed on the queue MUST be this. | All `Event` subclasses, `core/bar.py::Bar`, sizing policies in `core/sizing.py`, `SignalRecord` |
| **Pydantic `BaseModel`** | The object is **validated external/configuration input** needing field coercion, cross-field validators, or env-layer overrides. Frozen (`ConfigDict(frozen=True)`) when it must not mutate post-construction. | `SystemConfig`, `ExchangeConfig`, `PortfolioConfig`, `Settings` |
| **`Enum` in `core/enums/`** | The value is a **closed vocabulary crossing a domain boundary**. Class-based form, explicit UPPERCASE string values, `_missing_` for case-insensitive parse, plus a `<domain>_<type>_map` companion dict. | `OrderType`, `FillStatus`, `Side`, `CashOperationType` |
| **`str, Enum` in `config/`** (documented exception) | The closed set is consumed **only** by its co-located Pydantic config model and never by events/handlers directly. Moving it to `core/` would gain nothing; keeping Pydantic string serialization matters. | `FeeModelType`, `SlippageModelType`, `PortfolioType` (see W2-13) |
| **Plain class** | The object is a **stateful domain entity** with behavior and lifecycle transitions. | `Order`, `Portfolio`, `MatchingEngine`, the managers |

**Litmus tests:** an inline comment listing allowed values (`# ERROR, CRITICAL, WARNING`) means
"this should be an Enum". A `.value` comparison at a dispatch site means "compare enum members
instead". A mutable `@dataclass` returned from a pure function means "add `frozen=True`".
A `float` annotation on anything money/quantity-shaped is a defect (locked decision).

docstrings explicitly cite D-12 ("sink read, NOT a cross-domain handler call"). The placement is
also a prerequisite for the 999.5-(b) ScenarioSpec promotion. **No move needed.** Similar cases
hunted: W4-02/03/06/07 above (two ambiguous, one drifted-but-999.5-owned, one justified).

  `Portfolio.update_config` (`portfolio.py:166-192`): it does piecemeal `setattr` on the live
