### Root cause (correct fail-safe, not a code defect)

The D-04 post-reconcile baseline guard (landed this phase in 05.1-08) compares the venue base-asset
balance against the engine's post-reconcile believed position (0 at a fresh session). The demo
sub-account is **not flat** — it carries a pre-seeded base-asset balance:

```

  USD:  100000.0     USDG: 100000.0     USDC: 99944.23790928

```

The wired pair is `BTC/USDC`; the guard sees venue `BTC ≈ 1.0009` vs engine believed `0` → unexplained
residual → latched HALT (real via the 05.1-05 HALTED latch) → `start()` refuses RUNNING.

This matches the documented **accepted behavior** (05.1-CONTEXT.md lines 100/141): *"until Wave 2's
reconcile sync arm lands, a restart while holding a position halts at the baseline guard — correct
fail-safe."* It also reflects the ARCH-2 precondition (line 99): *"one dedicated OKX demo sub-account
per live system ... balance ≡ strategy position by construction"* — i.e. the dedicated sub-account is
expected to start **flat**. The current demo sub-account (OKX-preseeded / residual from the 05-12 run
on 2026-07-03, when the D-04 guard did not yet exist) violates that precondition.

## Flatten attempt (2026-07-04) — demo BTC is UNSELLABLE; account cannot be flattened autonomously

Per the user's "flatten all / close all positions" instruction, an attempt was made to sell the
pre-seeded base assets so the account starts flat. Findings:

- **No open derivative positions** (`fetch_positions()` → none with contracts).
- **BTC is in the trading/spot/unified account** (funding empty) — so it is nominally tradeable-account BTC.
- **EEA/MiCA quote restriction:** `BTC/USD`, `ETH/USD`, `XRP/USDT`, `ETC/USDT` all rejected with
  **sCode 51155 / 51138 / 54092** (RestrictedLocation / TradFi-required). Only **USDC-quoted** pairs are
  permitted for this regional entity. So BTC can ONLY be sold on `BTC/USDC`.
- **`BTC/USDC` sell-price floor blocks all sells:** every BTC market sell (1 BTC down to 0.001 BTC)
  **canceled with `filled=0`**; a marketable limit sell was rejected with
  **sCode 51138: "The lowest price limit for the sell leg is 60,160."** The market imposes a **minimum
  sell price of $60,160**, but the entire bid book tops out at **~$59,342** (best bid) — the sell floor is
  *above* the highest bid, so no sell can ever execute. (Market BUYS fill normally — the test's 0.0001
  buys succeeded — the block is sell-side price-band specific.)
- **ETH** partially sold (0.2367 of 10 on `ETH/USDC`, ~$411 USDC credited) before its book/band stopped
  further fills; XRP/ETC unsellable (USDT-only → MiCA-blocked). BTC total unchanged at 1.00094001904.

**Conclusion:** the pre-seeded ~1 BTC on this OKX EEA demo sub-account is **not sellable** under the
current venue price-band + MiCA-quote constraints, so the account cannot be brought flat autonomously.
The D-04 baseline guard therefore keeps (correctly) halting `start()`. Completing the online settlement
proof requires one of:
1. A **freshly-provisioned flat dedicated demo sub-account** (ARCH-2 "balance ≡ strategy position by
   construction") — the design's intended precondition.
2. Manually clearing the BTC via the OKX web UI / a venue path not subject to the 51138 sell floor
   (e.g. once the demo book's price band moves so bids rise above $60,160).
3. **Recording this as the Wave-1 empirical result** — the guard correctly halts a non-flat account —
   and deferring the real-fill settlement proof to a clean-baseline re-run.

All offline code deliverables (D-01..D-05, D-19, D-20 Task 1) are complete and verified; only the online
real-fill proof (settlement assertions 1–2 + ARCH-3 points 3–4) remains, blocked by demo-venue state.

## Alternate-ticker attempt (2026-07-04) — validates the approach, blocked by MiCA whitelisting

Per the user's suggestion ("run the test with another ticker" since a flat account can't be provided),
the settlement proof was re-run against a USDC pair whose base the account holds zero of, by overriding
`_OKX_STREAM_SYMBOL` (engine) + `_OKX_SYMBOL`/`_BASE_CCY`/`_MIN_DEMO_QTY` (test) to **SOL/USDC**.

- **The approach is validated:** with a flat base (SOL=0), the D-04 baseline guard PASSED and `start()`
  reached RUNNING and submitted the order — proving the alternate-ticker path defeats the halt correctly.
- **But the order is MiCA-blocked:** the SOL/USDC min-size market BUY was rejected with **sCode 51155
  (RestrictedLocation)** — despite SOL/USDC showing `active=True` in `load_markets`. The OKX EEA entity
  only permits a *whitelist* of USDC pairs for trading; SOL/DOGE/LTC/XRP/ETC are all restricted.
- **The tradeable set = {BTC/USDC, ETH/USDC}, and both are held (non-flat).** ETH was re-attempted as a
  flatten target (it partially filled once earlier), but ETH/USDC market sells now also **cancel with
  filled=0** — the same sell-price-floor-above-best-bid dynamic as BTC. Neither held major can be sold to
  flat.

**Definitive conclusion:** on this OKX EEA demo sub-account, the intersection {EEA-MiCA-tradeable} ∩
{flat base} is **empty**, and the two tradeable pairs cannot be sold down to flat (sell-price floors
above best bid). The online real-fill settlement proof is **not achievable on this account/region in this
session** by any autonomous path. Completing it requires a suitable account (a freshly-provisioned flat
BTC/USDC or ETH/USDC demo sub-account, or a non-EEA demo account without the MiCA whitelist).
